SUBCHAPTER I.  INTRODUCTION TO TIMS


14.01  TERMINOLOGY AND ACRONYMS

Most of these terms can be found in M22-4, Part 1, Chapter 2 and Part XII, Chapter 2. They are repeated here for continuity.

ARCHIVE BATCH: A batch that has been processed and stored.

BATCH: A group of organized documents that are scanned into TIMS.

BATCH CATEGORY: A manual process in which a category is assigned to a group of organized documents. As an example, a batch category can be a group of 1999s, 1999bs, Originals, or Mixed.

BATCH COVER SHEET: A page that is physically placed on top of a batch to identify the documents before they are scanned. The cover sheet lists the scanning attributes assigned in the Define Batch module.

BATCH ID: A system-generated number used to track an electronic batch throughout its processing.

BATCH LIMIT SETTING: A number specifying how many batches will be displayed in the batch window. This applies to the batch, fix index, and Index/QC windows.

BATCH PRIORITY: Each batch has a number between 1 and 9 (lowest to highest) assigned to it which sets the level of priority for that batch. If 9 is selected, the Expedite button is automatically selected. (NOTE: The default is 4)

BATCHER: A person who creates, defines, and groups documents into organized batches.

BIRLS: Beneficiary Identification and Records Location Subsystem.

BINQ: BIRLS inquiry in BDN.

BUPD: BIRLS update in BDN.

CLAIM TOKEN: The combination of newly received documents and historical documents from a claimant's folder attached to support the resolution of outstanding work on a claim.

CLAIMANT: This is the person who applies for benefits. This individual may either be the Veteran or designated beneficiary.

CLAIMANT FOLDER: The storage location of all documents relating to a particular Veteran or designated beneficiary.

COMMIT: The process which makes permanent changes to data (inserts, updates, deletes) in the database.

COMMIT DATE: The day the document information is sent to the index database and the image is sent to storage.

CREATE BY: A field in which the name of the person who created the batch (i.e. the batcher) is placed.

DEFINE BATCH: The specifications that are assigned to each batch that is prepared.

DOCUMENT DATE: The day the document was date stamped received or if there is no date stamp, it is the date the document was scanned into the system..

DOCUMENT LENGTH: A number which specifies how many pages should be counted before a new document is started. Select this option and specify the number of pages in the block provided.

DOCUMENT SEPARATOR PAGE: A piece of coded paper that divides the documents in a paper batch.

DOCUMENT TYPE: A pre-defined code that TIMS uses to determine the type of processing a document requires.

DOUBLE FEEDS: A scanner device error in which two pieces of paper are fed through the scanner device at the same time resulting in the loss of information.

DUPLEX: The process of scanning a page on both sides.

ELECTRONIC BATCH: The electronic grouping of images from a scanned batch of paper documents.

ELECTRONIC CERTIFICATION OF AUTOMATED PROCESS (ECAP): A computer program which attempts to process certifications received electronically. When ECAP is unable to process a certification, it routes the document to Fix Index. These documents are similar to the documents routed by Indexers.

EXPEDITE: To cause a batch or claim token to receive highest priority (9) in TIMS. Any document in an expedited batch with priority 9 rises to the top of the batch or claim token queue from which VCEs work.

FINISH (BATCH): Finishing a batch in the Index/Quality Control (QC) process releases the batch for committal and allows TIMS to separate the batch into its component documents in various claimants' folders.

FIX INDEXER: Operators, who perform the same tasks as Index/QC operators, but have additional duties including correcting document indexes and other problems with documents.

FLUNK: To delete a document that is not readable or has a poor image quality from the current electronic batch. If the document is valid, it must then be re-batched and scanned as part of a new batch.

INDEX/QC: The process of verifying the image quality and entering or editing document information.

INDEXER: An operator who performs the manual indexing process by:

  • Viewing the document image on a workstation screen, and
  • Entering the appropriate index values based on an analysis of the document's content.

The operator is also responsible for performing QC by:

  • Viewing the document images on a workstation screen, and
  • Validating data obtained through the automatic data recognition processes.

INFORMAL CLAIM: Usually any claim filed without a VA Form 22-1990, Application for VA Education Benefits or VA Form 5490, Application for Survivors' and Dependents' Educational Assistance.

INFORMAL CLAIM QUEUE: A queue established in TIMS to receive folders that do not have the CEST Completed box marked.

LOC: BDN Screen for Location of TIMS folder.

MINQ: BDN command for Master Record inquiry.

MANUAL DOCUMENT SEPARATION: The indexer separates the documents manually or leaves them under the same Document Identification number (DOC ID). All documents in the batch would appear as a single document under the same DOC ID.

MIXED: A batch of different documents. (See Mixed Batch)

MIXED BATCH: A batch that contains multiple document types.

ORIG: A batch of original documents.

OFF-RAMPED: Electronic documents that cannot be matched to an existing TIMS folder.

PAYEE NUMBER CODE: A two-digit code identifying the type of beneficiary to whom the payment record pertains. BDN generates this code.

QUALITY CONTROL (QC): A manual operation where an operator assesses the image quality of captured documents, and verifies the results of automatic data recognition.

SCAN DROP DOCUMENT BOX: A configurable box in the Define Batch module which enables users to indicate that a batch should not form a claim token.

SCANNING (BATCH SCANNING): The act of capturing the electronic images of a group of documents into TIMS.

SCANNING PAGE OPTIONS: A TIMS function which allows simplex (one-page) or duplex (two-page) scanning choices.

SENSITIVITY LEVEL: Any number between 0 (the least sensitive) and 4 (the most sensitive). (NOTE: The default is 0). A guide to follow is:

  • 0 = BDN sensitive levels 0 to 6.
  • 1 = No longer used.
  • 2 = No longer used.
  • 3 = BDN sensitive level 7.
  • 4 = BDN sensitive level 8 or greater, or any indication of fraud.

SEPARATION OPTIONS: A TIMS function which allows manual, document separators, or document length separation choices.

SEPARATOR SHEET: An 8½ X 11 bar coded sheet of paper that tells the scanner to create a new document. Insert separator sheets during batching.

SET BOX NUMBER: A number used in archiving batches to identify the box in which the batch is physically sorted. All batches are physically sorted in boxes and each box is assigned a number. That number is entered in the Set Box Number field.

SIMPLEX: The process of scanning a page that has data on one side of the page.

SINQ: BDN command for Status Inquiry.

THE IMAGE MANAGEMENT SYSTEM (TIMS): A computer application utilized by VA in making and maintaining electronic folders. The electronic folders consist of electronic images of the documents used for education claims.

VETERAN FILE: The means of identifying a Servicemember in TIMS and associating that individual with any existing claimant folder.

VID: BDN Screen displaying Veteran Identification Data.

14.02  PURPOSE AND BENEFITS OF TIMS

The purpose of TIMS is to make electronic images of the documents used for education claims.

a. Benefits. The benefits of using TIMS include:

  • Decreasing claims processing time,
  • Organizing work,
  • Improving workflow,
  • Viewing the same file by multiple users simultaneously, and
  • Transferring the electronic files between stations.

14.03  WORKFLOW FOR UPSTREAM PROCESSING

a. Stream Process. TIMS is known as a stream process because one step in the process flows into the next step. This provides an advantage in simplicity and makes it easier to find errors. This may also make correcting the errors faster and easier.

b. Upstream Process. The front end of the TIMS operation is the beginning of the stream which is why it is known as "Upstream". Sometimes it is also known as capture, since this is where the images are moved from paper to electronic media.

c. Parallel Flow of Information. There are two parallel flows of information that occur in the upstream end of TIMS. They are:

  • The handling and routing of physical documents, and
  • The images that are created from them in TIMS.

The two flows must be synchronized as much as possible throughout the upstream process. When synchronization breaks down, the whole process breaks down.

a. Batch Processing. This process includes several actions, generally performed by the RPO's Upstream team(s). Generally the Upstream team has employees known as Program Support Assistants, TIMS Clerks, or Upstream Clerks. In this chapter, the acronym PSA represents this position. Duties of this position include:

  • Receiving the mail,
  • Breaking down the mail to prepare batches,
  • Ensuring every document has the correct claim number and/or Social Security Number (SSN) and;
  • Assigning a batch ID number to the paperwork.

b. Batch Processing Category. The batch processing category attaches tracking numbers to documents through batch identification numbers. These provide preliminary details to TIMS for creating electronic images (also known as defining the batch). This is the first step in the process of creating images from paper documents.

c. Categories for Batching. There are 6 main categories which exist for batch processing:

  • 1999s.
  • Originals.
  • 1999bs.
  • Mixed.
  • 1990E (TOE).
  • 5490s (FRY and Chapter 35).

14.06  MAIL SCREENING

a. Mail Analysis. The processing of education claims or issues begins with the analysis of the incoming mail at a RPO. The overall mail management program is the responsibility of the Support Services Division (SSD), under M23-1, Part 1, Chapter 1. All mail received from the public must be properly analyzed to determine the appropriate action. It is essential that education mail be analyzed by a person qualified to determine the proper action since there are:

  • Numerous VA education benefits,
  • Multiple computer databases,
  • Several types of claims folders, and
  • Variable RPO jurisdictions over claims.

Failure to properly analyze education mail may result in unnecessary delays and mistakes if it is misrouted or mishandled.

b. Education Mail. Education mail is any form with a 22 prefix or a document which pertains to a benefit covered by this manual (M22-4). Some education mail constitutes a formal or informal claim for benefits. Refer to Chapter 3 for further discussion on formal Vs. informal claims.

c. Receiving the Mail. Paper mail is usually received, opened, and date-stamped in the mailroom.

NOTE: The PSA verifies the mail has been date-stamped. If the mail is NOT date-stamped, it is the PSA's responsibility to ensure that it is date-stamped before further processing.

d. Date-Stamping Mail. Mail should be date-stamped so that the date-stamp does not cover any written text that the claimant has entered or pre-printed text, and is always on the front side of the documents.

NOTE: If it is not possible to avoid date-stamping on text, it is acceptable to place a date-stamp over pre-printed text.

SUBCHAPTER IV.  INDEX AND QUALITY CONTROL (QC)

 

14.18  INTRODUCTION TO INDEXING AND QUALITY CONTROL

a. Indexing. Indexing is a task that assigns the correct Social Security Number (SSN) to each document. The employee that completes this task is called an Indexer. All records in TIMS are based on the SSN of the claimant.

It should be noted documents sent electronically may go to Fix/Index and will need an Indexer to "index" the document to the appropriate folder. This is not the same as correcting SSNs or names which is the true function of Fix/Index. This function of Indexing is required by all PSAs, regardless of where the document is found.

b. Quality Control. Quality Control (QC) is a task that ensures the quality of the electronic image is acceptable. This is also completed by the Indexer. The Indexer matches the hard copy documents with the scanned images.

In this role, the Indexer makes sure that both the front and back of a document is scanned. The Indexer also verifies that the documents that are being indexed match the scanned image. The Indexer also assigns the type of document and benefit.

14.21  CREATING A RECORD IN TIMS

a. Verify Jurisdiction. Indexers should be familiar with RPO jurisdictions. If the claim is NOT in the indexer's jurisdiction, the PSA should still scan the document into TIMS and create a folder if necessary. The VCE will review the claim and return to Upstream if the file needs to be transferred to another RPO. The correct procedures involve transferring the document to the correct RPO through TIMS.

Figure 14.22 describes the rules for jurisdiction as it applies to certain situations. These situations are based on the information provided on the Veteran's application.

14.22 [FIGURE] Jurisdiction over Special Situations

Situation

Jurisdiction
14.22 [FIGURE] Jurisdiction over Special Situations

School Known

If the claim is for enrollment in a specific school or training establishment, the RPO with jurisdiction over the school has jurisdiction over the claim.

Exception to School Known

The jurisdiction rule for School Known does not apply to Correspondence training, or licensing and certification tests.

Correspondence School

If the claim is for enrollment in a correspondence or combination correspondence and residence school, the RPO with jurisdiction over the claimant's place of residence will have jurisdiction over the claim.

If no EDU folder exists, then the RPO receiving the application will have responsibility of the folder.

Licensing and Certification Tests

The RPO with an existing EDU folder has jurisdiction over the claim for licensing and certification tests.

If no EDU folder exists, then the indicator is based on the claimant's place of residence.

Tuition Assistance Top-Up

The RPO with jurisdiction over the school has jurisdiction over the claim.

14.29  OUT OF RPO QUESTION BOX

During Indexing or performing Index QC, the PSA may receive a message which says:

"This Claimant Folder (V:xxxxxxxxx/C:xxxxxxxxx) has been transferred out of the RPO. Please request the folder from other RPO before fixing the document."

This message is intended to inform the PSA that the document they are trying to index cannot be placed into a TIMS folder because the active folder is being worked in another RPO. If the PSA continues with the action, and indexes the documents to the RPO where the folder is not, it could corrupt the folder. This means the electronic folder may have an open token in multiple places.

The PSA must broker (point) to the RPO where the folder is and place the document in the correct folder.

 SUBCHAPTER VI.  MISCELLANEOUS ACTIONS


14.32  KNOWING HOW TO IDENTIFY A POA

During all reviews of documents, whether batching or indexing, the PSA should watch for VA Form 21-22, Appointment of Veterans Service Organization as Claimant's Representative, or VA Form 21-22a, Appointment of Individual as Claimant's Representative. The claimant's record is usually annotated with the POA during the establishment of the claims file.

a. To add a POA to the VID Screen:

1.  Do a BUPD command on READY Screen.

2.  This will open the VID Screen.

3.  Tab to the Power of Attorney field and enter the POA code (Part Part 2: Chapter 9- Code Tables section 9.17).

4.  Review the rest of the VID screen. Ensure there are no duplicate entries in all columns. Those will have to be deleted from the screen before completing your action.

5.  Enter UPD in next screen field and will update the POA on the VID screen.

6.  Do a SINQ command to bring up updated VID screen showing POA code and capture to the TIMS folder. In remarks, the PSA should indicate "adding POA and their initials".

b. To change a POA:

1.  Use the CORR command on READY Screen.

2.  This will open the VID Screen.

3.  Using the tab, delete the "old" POA code, and then input the new code.

4.  Hit Enter to complete the action.

5.  Do a SINQ command to bring up updated VID screen showing POA code and capture to the TIMS folder. In remarks, the PSA should indicate "adding POA, and their initials".

c. To update POA to a PIF. In order to add a Power of Attorney to a Pending Issue only:

1.  Using the PCHG command on READY Screen, input the file number, benefit type, and the end product code. Press Enter

2.  This will open the 101 pending issue control screen.

3.  On the 101 screen, tab to the Power of Attorney field and enter the POA code.

4.  Press Enter to update the field.

5.  Do a SINQ command to bring up updated PO1 screen showing POA code and capture to the TIMS folder. In remarks, the PSA should indicate "adding POA, and their initials".

14.33  CHANGING A NAME

During the review of documents, whether batching or indexing, the PSA should watch for documents which could indicate a name change for a Veteran or student. The Veteran or student may not include a letter of explanation with the request for a name change. Claimants may change their name to whatever they choose as long as there is supporting documentation to support the change. If there is a question regarding the name change, the PSA should discuss with their immediate supervisor. Documents requesting the change of name should be scanned and indexed into TIMS. If the document should be returned for clarification, the VCE should do the development.

a. To change a name on the NAM screen:

1.  On the READY Screen, use the BUPD command, put NAM in Screen name. Be sure to put in the correct file number and first, middle and last name in the appropriate fields. Hit Enter to continue.

2.  On the NAM screen, tab to the "Change" field, enter an "X".

3.  Tab to the first name field, enter an "X".

4.  Tab to the bottom fields for name, middle, last and enter the correct information.

5.  Tab to next screen at the bottom of the screen and type UPD. Press Enter.

6.  Do a SINQ command and capture a copy of the VID screen showing the changed name. In REMARKS, put the message "changed name, date and your initials".

b. To change a name on a PIF:

1.  Enter CNAM on the READY Screen. Be sure to include the file number, payee number, and end product code.

2.  The 601 Screen will open. On this screen, tab to the Name field and/or "Name Person Ent" field.

3.  Make the appropriate changes. Press Enter to complete the action.

4.  Using SINQ, and capture a copy of the VID screen showing the changed name into TIMS. In REMARKS be sure to indicate changed name and your initials.

NOTE: PSAs must ensure the name and the stub name match.

SUBCHAPTER VII.  FOLDER TRANSFER PROCEDURES


14.36  GENERAL

a.  While there are two different procedures used to transfer folders within TIMS, the version known as Mass Folder Transfer provides the quickest process for not only the office requesting the folder but for the office receiving it.

(1) Each RPO should identify individuals who should have access to the other RPOs' TIMS database for the purposes of folder transfer procedures. The TIMS System Administrator at each RPO must receive the user names, social security numbers, and role information before granting access.

(2) Each station's System Administrator must create two roles in the TIMS database; "TRANSFER" and "SUPVTRANSF". The TIMS System Administrator at each RPO will assign the TRANSFER role to transfer clerks at the other RPOs. This role will allow an individual to Folder Processing in the Benefit Processing module and Folder Transfer Processing in the System Administration module. Privileges should be granted as outlined below.

(3) The TRANSFER role should be limited to:

i.  FPROC: add and/or remove documents from the claim token, and suspend.

ii.  FXFER: "Run" allows the transfer to happen.

(4) The TIMS System Administrator at each RPO will assign the SUPVTRANSF role to Upstream or other Supervisors at the other RPOs. This role will include all modules and privileges assigned to transfer clerks as well as the privilege to finish claim tokens.

(5) RPOs requesting the TIMS folder should do a BUPD in BDN. It is no longer necessary to do a BFLD command in BDN to transfer the folder. However, BDN must be updated regarding the location of the Education folder.

(6) Supervisors at each RPO are responsible for resolving any errors on their RPO's transfer log. The RPO may use e-mail to facilitate the process as well as to facilitate the transfer of problematic folders.

14.37  MASS FOLDER TRANSFER PROCEDURES

(a) The PSA should open Notepad and create a list of the folders needed to transfer out.

(b) The list must have the Veteran's SSN and the claimant's SSN only (no dashes between numbers).

(c) When listing them on the Notepad use the below format. You must have the first line below in that exact format. You change the sending and receiving RPO's per your request and add the number of records. When listing the SSNs, put Veteran's SSN then (comma, space) claimant's SSN

14.38  REGULAR FOLDER TRANSFER PROCEDURES

In situations when Mass Folder Transfer is not utilized, RPOs should follow these instructions whether they are transferring folders out or in.

a. Specific Procedures to Transfer a Folder In. Follow these instructions if a claim is received at an RPO but there is already a TIMS folder established at another RPO.

1.  Review BDN LOC Screen to determine which RPO has jurisdiction.

2.  Open your TIMS.

3.  Access Folder Processing and search for an existing folder.

4.  The PSA should now exit their TIMS.

5.  Access TIMS of the RPO where the folder exists.

6.  Under Folder Processing, search for the folder which needs to be transferred. If not found, you should repeat the process in the TIMS of the other RPOs.

7.  Once the folder has been located, review for an existing claim token and/or pending end product. If there is an active claim token and/or pending end product, do not continue with the transfer. Refer the folder transfer request to the Upstream Supervisor for review and further action.

NOTE: If there is an active claim token, the Supervisor should review the folder to determine the proper disposition of the claim token and any existing end products. If the folder should be transferred, the Supervisor will finish the claim token before transferring the folder. The Supervisor should ensure the claim token is reestablished at the receiving station and routed to the appropriate queue, if necessary. Communication between the two Supervisors, each in their own RPOs, will be critical to ensure minimal delays when a claim in transit needs to be worked by the receiving RPO.

8.  If there is no active claim token or pending end product, review the folder for a flash. The PSA must annotate the information in the FLASH into a NOTE and then delete the FLASH before continuing.

9.  Exit Folder Processing.

10.  Access System Administration - Folder Transfer Processing and search for the folder you wish to transfer.

11.  Single click on the folder to select it (highlight it).

12.  Select folder from the tool bar.

13.  Select Transfer Selected Folders from the tool bar.

14.  Select receiving station from the list of RPOs and click OK.

15.  Click Yes to transfer the folder.

16.  Select View Electronic Transfer Requests from the tool bar.

17.  Single click on the folder to select it (highlight it).

18.  Click on the icon labeled Commit Claimant Folder(s) for Transfer

19.  Click on Yes to commit folder to transfer.

20.  Exit the sending station's TIMS.

21.  Re-log into the original TIMS. Transferred folders may not be immediately accessible. Consequently, transfer clerks must maintain a list of transferred folders and review TIMS the next day to determine if the folder was successfully transferred.

22.  Access Folder Processing and search for the folder just transferred.

23.  Review the folder to ensure all documents are accounted for - the total number of documents should include documents previously contained in your TIMS database, if any, plus the number of transferred documents. If the folder does not contain the appropriate number of documents, it should be referred to a Supervisor.

24.  Process a BUPD to change the folder location in BIRLS.

SUBCHAPTER VIII.  ARCHIVING


14.39  ARCHIVING PAPER DOCUMENTS

All paper documents scanned into TIMS must be boxed and archived for an appropriate time frame based on the RCS Schedule. Generally these documents should be saved for a minimum of 6 months following their scan date. For more information, contact your immediate supervisor.

a. Preparing the Boxes for Archived Documents:

1.  Boxes should be assembled and a destroy date label placed on them.

2.  Each day, the oldest batches completed (i.e. indexed) are placed in the archive box.

NOTE: The batches indexed on the 2 previous dates are left on the archive table for SQC purposes.

3.  Once the box is full, close the top. Do not tape the top of the box. The batches still need to be archived.

4.  Information to write on the box:

  • On the bottom left corner, MM-DD-YY, write the date of the oldest indexed batch that is in the box (it should be the batch on top).
  • On the bottom right corner, write the box number. Insure that there is not another box with the same date on the staging shelves. If there is already a box with the SAME date, the next box would be numbered 2, etc...
  • On the top left corner, write the date as YYYY-MM-DD.
  • On the top right corner, write the box number.
  • Write the destroy date on the destroy date label. It will be 12 months from the date written on the top and bottom left of the box. In this example it would be 09-15-16.

 

5.  Place labeled box at the end of the archive table.

Go to Top

14.40  ARCHIVING INTO COMPUTER

a. Log into TIMS.

b. On TIMS launcher: Select Category "Batch Processing" and Module "Archive Batch".

 

The archive batch module will open.

c. Select any batch from the box and type the batch ID number in the "Batch ID" box and select "Search". The batch number is then shown in the large box at the bottom of the window.

 

d.  Click on the "Set Box Number" button (i.e. 201509151 in the pop-up box) and select "OK". This is the last time you will have to type the box number.

 

e.  Click and highlight the last batch number in the "BATCH ID" box, type the next batch number and hit Enter. This puts the new batch number in the white box.

f.  Double click the batch number in the batch ID line (large white box). The archive Batch box will pop up with the box number. Select "OK" or Enter.

g.  Repeat steps 5 and 6 for each batch in the box.

h.  After you have archived all the batches in the box. Close the top, sign your name on the destroy date label (legibly) and place the completed box on the staging shelf.

NOTE: IF THE BATCH NUMBER DOES NOT SHOW UP IN THE LARGE BOX, THE BATCH SHOULD BE TURNED IN TO YOUR IMMEDIATE SUPERVISOR.
