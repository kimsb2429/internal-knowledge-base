Sub-Chapter 5. Work Product History LTS Screen

Sub-Chapter 5. Work Product History LTS Screen

a. General

When the Claimant Biography is successfully created, navigate to the Work Product History screen.

b. New Claimant

New claimants will not have any Work Products at this time. Click the Add Work Product button on the right-hand side of the screen.

The user will be taken to the Entitlement Source screen.

c. Entitlement Source Screen

On the Entitlement Source screen, the user will select the source for CH33 benefits. “Entitlement Source” refers to the active duty service from where the entitlement will be drawn:

  • Claimant’s own service (SELF)
  • Fry Scholarship (FRY)
  • Transfer of Entitlement (ToE)

Note: A ToE claimant’s biographical record must be created first before proceeding with a ToE claim. After this has occurred, a veteran’s or service member’s record must then be created and authorized to proceed with source selection for the ToE claimant.

 

A claimant can have more than one entitlement source to draw from. However, LTS doesn’t have the functionality to deal with multiple entitlement sources, so more than one entitlement source may not selected.

 

Click the Select button next to the entitlement source to continue.

**For display purposes, we have selected SELF

The user will be taken to the Work Product Creation Screen

 

d. Work Product Creation Screen

On the Work Product Creation screen, the user will enter descriptive data for the claimant. The user will enter the following fields:

  • Date Received (date application was received)
  • Description (what is the purpose for the award, such as payment for a school term or semester)

Also on the screen will be Work Product Type. This is auto-filled based on claimant information:

  • Original Award - if this is claimant’s first award
  • Supplemental Award – if this is a subsequent award

Once the appropriate data is filled in, the user should click the Create Work Product button.

If you would like to cancel this record, click the Cancel button.

 

NOTE:

When the Work Product History has been successfully created, the read-only banner that features the claimant file number and name will display a summary of the claimant’s entitlement. The following information will be featured on the banner of each subsequent screen in LTS.

  • Eligibility Date
  • Benefit Level
  • Delimiting Date
  • Original Entitlement
  • Used Entitlement
  • Remaining Entitlement
