Sub-Chapter 1. Search Chapter 33 - LTS Data Entry Rules

This section describes data entry rules for the Search Chapter 33 LTS screen in the Long Term Solution (LTS) system.  You will find data entry rules to help you with the following:

1.01 Search LTS

1.02 Search VADIR

1.03 Wildcard Search

1.04 Record Locked

 

1.01 Search LTS

a. Search

The search for a claimant’s record always begins in the CH33 LTS database.  To start a search, locate the claimant’s record in the LTS.  Begin the search by entering any of the following in the corresponding fields on the screen:

  • Social security number (SSN) or File #,
  • Last Name, First Name, or
  • Date of Birth

 

b. Social Security Number (SSN)

When searching by Social Security Number (SSN) enter nine numeric digits without dashes

 

c. File #

When searching for an eight numeric digit File Number, enter the eight numeric digits, no dashes or spaces. For example, if the file number is 77777779, you will enter 77777779 and then LTS will automatically populate the “C” in the number to look like C77777779.

If File number has nine numeric digits, it must match the SSN. 

File number searches only apply to CH33 LTS, not VADIR (VA/DoD Identity Repository). VADIR is.

 

d. First and Last Name

When searching by First Name, use a minimum of one character and a maximum of 20. You cannot search by First Name only.

Searching by Last Name requires a minimum of two characters and maximum of 26. Do not use special characters, hyphens and spaces. If you do, they will be ignored for comparison purposes.

Search is not case sensitive, so you do not have to start with a capital letter.

 

e. Date of Birth

Search by Date of Birth using the format MM/DD/YYYY.

 

 

When you enter the SSN or File number the Search Chapter 33 button is enabled on the lower right-hand corner, highlighted in yellow.

If records are found, they can be viewed in the Results table. This means the claimant had a prior claim processed.

If no records are found after searching the CH33 database, the “No Results Found” message will be displayed on the Search Results of Chapter 33 screen.

1.02 Search VADIR

a. VADIR

If no results are found in the Chapter 33 (CH33) LTS, you will get a message stating, No Results Found. You will see that a “Search VADIR” button becomes available on the far right of the screen. 

 

Perform the search using the data entry rules described above, but this time you will click the “Search VADIR” button. If no results are founds in VADIR, the user will get the following warning message:

b. No association of [Claimant] in VADIR.

If the claimant is in VADIR, most of the personal identifying information will auto-fill in LTS. The user will not need to key in additional information, but should compare and verify results to the application including address, phone number, e-mail address, etc.

 If no matches were found in either CH33 LTS or VADIR, you have 2 options to continue the claim:

  • Start a New Chapter 33 Search
  • Continue the claim by clicking the Create New Chapter 33 Record button, which navigates you to the Claimant Biography screen and create a new record within the CH33 LTS system.

1.03 Wildcard Search

a. Wildcard

A wildcard search is used when you do not have complete information, for example, a poor quality copy from The Imaging Management System (TIMS) [Hyperlink] where the last name is partially legible.

Wildcard searches require that you place a percent (%) sign at the end of the data string you enter. Wildcard searches can be used with the following fields:

  • SSN or File number
  • Last Name

For example, a wildcard search using a partial SSN of 444 would be entered as 444%.

If no matches are found using any of the strategies described above, the user will need to manually create a new CH33 record. Click the Create New Chapter 33 Record button on the far right screen to navigate to the Claimant Biography screen.

1.04 Record Locked

a. Record in Use

If a record exists and is currently opened by another LTS user, the Record Locked message will be displayed.

If a record is locked, you can view the record in read-only mode until the record is released.

You will not be able to access the record unless the user:

  • Releases the lock
  • Logs out of LTS, or

There is 15 minutes of inactivity
