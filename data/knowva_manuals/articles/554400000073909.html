4.03   CHAPTER 1607 STATUS AND AWARD HISTORY SCREEN (M21)

The M21 screen can be accessed using the CADJ (Claim Adjudicate), CDAT (Claims Data Entry), CORR (Correct Master Record), STOP, SUSP (Suspend), RESU (Resume), MINQ and CNUM (Change of Number) commands. 4.12   PINQ PROCESSING

a. The PINQ (Pending Issue Inquiry) command accesses the pending issue file located at the BDNRS (Benefits Delivery Network Regional System). The PIF (Pending issue File) contains data entered by the operator, data from BIRLS (Beneficiary Identification & Records Locator Subsystem) and data from the claimant's master record if one exists. The pending issue remains accessible until the issues are resolved and the EPC (End Product Code) taken or cancelled. The S02 (Payee Selection) screen will display all pending issues if more than one EPC is pending. See Part II, Chapter 6.

b. To access the P01 (Pending Issue File Claim Status) screen through the PINQ command, the following entries are required on the READY screen:

(1) Command PINQ

(2) Operator's password.

(3) File Number

(4) Press ALT/ENTER.

NO BENEFIT is necessary since PINQ processing accesses all systems. .

c. The The P01 screen displays the status of the claim under CLAIM HISTORY. The CLAIM LOCATION FIELD shows location (e. g., ADJUDICATION) and action (e.g., GAP-DATA ENTRY). Messages at the bottom of the screen give additional information (e.g., PENDING ISSUE CLEARED, "12B" PENDING ACCEPTANCE). Access is limited to screens appropriate and related to that pending issue. "SCREEN INAPPROPRIATE FOR ISSUE" is displayed when entry of a screen that could not be used for that claim is made.

4.13   SINQ PROCESSING

a. The SINQ (Status Record Inquiry) command is a powerful inquiry command. SINQ will display a list of all education master records PROVIDED there are no pending issues at the same BDNRS. (See part II, par. 3.02.)

b. To process a SINQ inquiry, the following entries are required on the READY screen:

(1) Command SINQ

(2) Operator's password.

(3) File Number

(4) Press ALT/ENTER.

c. The SINQ command is really a combination of the PINQ, MINQ and BINQ (BIRLS Inquiry) commands. The SINQ command processes through the screens that these commands would access and shows those screens in the following order:

(1) SINQ first searches for a Pending Issue File. If it finds a Pending issue on the File Number entered, SINQ acts just like the PINQ command. If more than one pending issue is found, the S02 selection screen is shown

(2) SINQ next searches for a Master Record if it does not find a Pending Issue. If it finds a Master Record with the File Number entered, SINQ acts just like the MINQ command. An S02 screen is shown if two or more master records exist.

(3) SINQ next searches the BIRLS database if it does not find a Master Record. If it finds BIRLS record with the File Number entered, SINQ acts just like the BINQ command. It shows the BIRLS VID (Veteran Identification Data) screen with the correct file number. "NO RECORD" is shown on the ready screen if no BIRLS record is found.
