SUBCHAPTER I.  GENERAL

6.01  WEAMS – Web Enabled Approval Management System

a.  General. ELRs (Education Liaison Representatives) use WEAMS to maintain information on approved Educational and Training Institutions (ETI) programs.  ELRs are responsible for adding records of newly approved ETIs and revising records upon receipt of information from SAAs (State Approving Agencies).

b.  Update Access. ELRs and other designated VACO Oversight and Accountability Division employees have update access.  ELRs are not limited to maintaining WEAMS records for an RO (Regional Office) or a state.  CELOs (Chief Education Liaison Officers) may designate ELRs to process approval actions for multiple states, either permanently or temporarily.

c.  Inquiry Access. RPO (Regional Processing Office) and RO (Regional Office) Veterans Benefits Counselors, VCEs (Veterans Claims Examiners), ECMs (Education Case Managers), VSRs (Veterans Service Representatives), VR&E (Veteran Readiness and Employment), Office of Inspector General Agents, and Finance employees have inquiry access to the WEAMS records at all ROs.

d.  National Database. WEAMS records are stored in a national database.  Users have the option of searching for approval information for records maintained by a specific RO or searching all the records in the database.  Facilities of all types are included in the database.  It is not possible to filter searches to a particular type of training or search for a particular program.

e.  Design. WEAMS  is a VA intranet application.  Users have the option of using drop down menus or combinations of key strokes to start a process.  For example, hold down the ALT key and press the letter F to select File on the Main Menu screen.  Use the up and down arrow keys to highlight the desired entry and press the Enter key.  On the screens, users can use the mouse to move the cursor to the desired field or press the Tab key to scroll through fields.  Processes are started by clicking the appropriate button on the screen.  Users may minimize, maximize, or close WEAMS by clicking the buttons in the upper right-hand corner of the screen.

f.  Screen Entries. To make entries easy to read, all information appears in all capital letters.  Drop down boxes are provided with standardized entries.

g.  Security. Access to WEAMS is controlled by the CSS (Common Security Services) module.  RPO Education Officers and CELOs and RO Service Center Managers and VR&E officers need to notify the RPO's or RO's security officer of each employee's level of access.

(1) WEAMS Level 1 allows the user to perform all functions to include adding, modifying, and deleting records.  This level is restricted to ELRs, Education Compliance Survey Specialists, and clerical personnel assigned to VACO Oversight and Accountability Division. 

(2) WEAMS Level 2 allows the user view access only and is generally assigned to all other users who need the information to process claims for education benefits or to answer inquires from persons eligible for education benefits.

h.  Help. WEAMS has a help file with additional information that is located within the application.

SUBCHAPTER II.  INSTITUTIONS

 6.04  FINDING EDUCATIONAL AND TRAINING INSTITUTIONS (ETI)

a.  General.  WEAMS was developed to allow the user to search ETI’s using many options and filters. Users will commonly not know the exact facility code, they may need to look up multiple facility codes for a specific institution located in multiple states, or they may need to know all facility codes based on a type of program offered. The following options identify ways a user can search for ETIs or to quickly prepare a report to answer an inquiry.

b.  Finding ETI Options:  There are many ways to find an ETI or group of ETIs within WEAMS. 

(1)  Facility Code Search. Users may enter the full known facility code, or they can enter partial (minimum 3 characters) to locate ETIs. enter all or part of the facility code in the Facility Code field.  Note, when 3 to 7 characters are used, the search results will include the matching entry located anywhere within the established facility code.

(2) Search Tab. The search tab when selected will open and allow for additional options to search for ETI(s).

  1. Organization Name: Enter all or part of the name of the ETI name. Note, when using partial name entry, the search results will include the matching entry located anywhere within the established WEAMS organization name. If there is any difference between the way  the name is shown in WEAMS, the program may not be found.   Using this option solely will return matching name entry submitted for any state in the selected Physical Address Country. The default setting is null value.
  2. Physical Address State: Select from drop down menu the specific state that support required search. Using solely this option will return all facility codes established with the physical address state in the selected Physical Address Country. The default setting is null value.
  3. Physical Address Country: Select from drop down menu the country that supports the required search. The default selection is USA.
  4. Advanced Search Button: This button provides specific users with the capability to add more filters to the required search. Each filter is self-explanatory for the user to apply as many filters to support the search request.
  5. TIN Search Button: Allows user to enter a Tax Identification Number to look up one or multiple ETIs.

c.  No results matched.  If no matching records are found, a message will appear showing “No records found” and the “Search Criteria” used.  The user will need to alter the search criteria used if necessary to locate an ETI that has been established in WEAMS.

  • **If the campus is a “Main” campus, the Main Campus, all Branches that are children of the “Main Campus” and all “Extensions” that are children of the “Main (Parent) Campus” will be displayed.
  • For a “Branch” campus, the Main Campus, Branch Campus, and all Extension campuses that are children of that Branch Campus will be displayed. Branch campus’s also have centralized certification capability.

    c.   Link Main Campus

The Link Main Campus screen allows the user to view all facility codes associated with the organization, and depending on the user's permissions and if the campus is the Main Campus, associate a branch campus to this Main campus, or delete a branch campus from this Main Campus.

The screen displays a list of facility codes currently associated with the selected organization.  A “Y” in the column under the header “Main Campus Indicator” denotes whether the facility is a Main Campus. 

When a new organization is created using the New Organization Profile screen (link), one of the required fields to be entered is a unique facility code. The facility code, which is entered during the original creation of the organization, will always be marked as the Main Campus.  However, it can be ‘adopted’ as a Branch Campus by another Main Campus from the “Link Main Campus” page.

Add New Branch

In order to add a new branch campus, the current campus must be the Main campus (in fact, the “Add New Branch” and “Delete Branch” buttons won’t appear unless the current campus is a Main campus).  If the current facility is a Main campus, to add a new branch campus for this facility, click on the “Add New Branch” button.   A blank facility code field is displayed to the user. Enter the facility code for the Branch Campus you wish to associate with this Main campus, and click  “Add New Branch” (or "Cancel" to discard the changes). The user is taken back to the Link Main/Branch Campus page, where the modified facility code list will be displayed.

Delete Branch Campus

In order to delete a branch campus, the current campus must be the Main campus (in fact, the “Add New Branch” and “Delete Branch” buttons won’t appear unless the current campus is a Main campus).  To delete a facility code as a branch from the current Main campus, click on the "Delete Branch Campus" button at the top of the screen.

A blank facility code field is displayed to the user. After typing in the facility code for the branch campus you wish to disassociate from the current Main campus, click the "Delete Branch Campus" button (or "Cancel" to discard the changes). The user is taken back to the Facility Code list.  The user will note that the facility code list no longer contains the facility code of the recently disassociated organization.

6.11  CERTIFYING OFFICIAL / POC SCREENS

a.  General A Certifying Official is the contact point for the Organization's programs. A school certifying official (SCO) with the primary responsibility for certifying Veteran-beneficiary enrollment at the educational institution. The SCO is delegated the authority to sign enrollment certifications, other certification documents and reports relating to VA Education Benefits.

This page displays a list of all certifying officials associated with the Organization's programs and an option to add a new official.

View/Modify Existing Organization Official **

To view the Organization Official details, click on the Name in the List of Officials. The following fields are displayed:

  • Legacy Name - The name will only appear if there is a Legacy Name in the DB, and no “First Name”, “Last Name” of the officials in the DB.  A Legacy Name is a ‘carry over’ name from the old applications.
  • First Name
  • Middle Name
  • Last Name
  • Priority - the official's rank as compared to other Certifying Officials for this organization's programs.
  • Title
  • Official Type - Designated, Read Only
  • Address One, Two, and Three
  • City
  • State
  • Zip Code
  • Country
  • Phone
  • Fax
  • Email
  • Status - Approved, Disapproved, Suspended, or Withdrawn
  • Status Date - Automatically populated when Status is changed.

Note: SCO address field is not required if it is the same as the Organizational Profile address. The name, priority, phone number and email fields are required, as well as the type, status, and status date.

Modify any of the relevant fields and click "Save".

Clicking "Reset" clears all fields, and "Cancel" returns the user to the previous screen.

Add a New Organization Official **

Click on the "Add New" button at the top of the screen.

Fill in at least the required fields: First Name, Last Name, Status, and Status Date.

SUBCHAPTER III.  PROGRAMS

6.12  PROGRAM SCREENS

a.  General. The Search Program and Program Detail screens allow the user to perform the following functions:

  1. Search for specific programs associated with the organization based on various criteria
  2. Search Results
  3. View and Modify existing programs
  4. Add new Programs
  5. Aircraft information
  6. Suspend programs
  7. Lift program suspensions
  8. Withdraw programs

1. Search Programs - (criteria)

The user can search for existing programs using the following criteria:

  • Program Type - choose one from the list (blank for all types):
    • Institutions of Higher Learning (IHL) – Credit bearing post baccalaureate certificate programs are listed in the WEAMS IHL program section for processing purposes, but do not lead to a degree; therefore, they are not “deemed approved.”
    • Non-College Degree (NCD)
    • Correspondence Course (COR)
    • Flight Training School (FLT)
    • On the Job Training/Apprenticeship (OJT)

NOTE: Users should not add a new program if there are “X” or “Duplicate program” program fields that can be overwritten with current data. These can be located by checking the “Search programs that were withdrawn more than a year ago” box. Users should add new programs by using the X or Duplicate programs first before adding a new line item.

  • Program Code - a unique identifier for each program. Please refer to M22-4 Part 2, 9.08
  • Description
  • Search programs that were withdrawn more than a year ago (checkbox)

2. Search Results

The Search Results will display the following:

  • Description
  • Program Code - a unique identifier for each program
  • Type - Program Type (IHL, NCD, COR, FLT, or OJT)
  • STEM - Indicator to tell if program is a STEM program
  • Status

Tip: The Search Results can be sorted by clicking on any column header of the “Search Results”.

3.  View and Modify Existing Programs **

The Search Results list displays a list of programs based on the search criteria selected. To view or modify a program, click on its Description to bring up the Program Detail screen for that program. For additional information on the fields related to the specific Program Type click on the corresponding link below:

  • Institutions of Higher Learning (IHL)
  • Non-College Degree (NCD)
  • Correspondence Course (COR)
  • Flight Training School (FLT)
  • On the Job Training/Apprenticeship (OJT)

4.  Add New Programs **

To add a new program, click on the corresponding "Add" button below the search criteria for the particular program you want to add. Click on the link below for more information.

  • Institutions of Higher Learning (IHL)
  • Non-College Degree (NCD)
  • Correspondence Course (COR)
  • Flight Training School (FLT)
  • On the Job Training/Apprenticeship (OJT)

**Note: If the institution displayed is an Extension Campus (as noted by the 3rd character of the Facility Code being an “X”), then information is identical to the parent, and is displayed is directly from the Parent institution.

5.  Aircraft Information

If a “Flight” program exists for the institution, then on the “Search Programs” screen, there will be an additional link to the right of it called “Aircraft Information”. Click here for more about  how to Add and Update Aircraft information.

6.13 L&C SPECIFIC ORGANIZATION INFORMATION 

a.  General.  If the Organization offers Licenses and/or Certifications, the L&C Specific Organization Information needs to be entered.

By default, when an organization is created it will have a Status of ‘Created’. An organization cannot be approved until it has at least one License/Certification and one Certifying Official approved.

If the user has ELR privileges, all the of the following fields are displayed. If the user is an SAA, only the bolded fields are displayed:

  • Original Approval Date – the effective date on which the organization was originally approved.
  • ELR Remarks - Enter any relevant remarks made by the ELR regarding the L&C Organization
  • Organization Type - Choose from L&C Governmental Organization or L&C Non-Governmental Organization.
  • Status (defaults to Created) - Choose from one of the following statuses:  Disapproved, Approved, Suspended, Withdrawn.

Note: The Status cannot be changed to Approved unless an approved L&C official has been created.

  • Status Date - automatically populated when the status is changed.
  • Concur Date - the date on which an ELR initially concurred
  • Concur Name - the WEAMS username of the ELR who originally concurred
  • List of SAA Modifications

See the Certifying Official for L&C help topic (link) for more information.

6.18  VIEW TRANSACTION HISTORY

The Transaction History screen displays the Organization’s Transaction History. The user can view changes made to identified data fields and statuses. The Transaction History tracks the old value of the data field, the new modified value, by whom the change was made and the date/time when the change was made.

Note: The Transaction History screen is read-only - no records can be modified.

You can perform a search on all Transactions within the Start and End date parameters on the top of the screen.  The date range cannot exceed 30 days from the current date.
