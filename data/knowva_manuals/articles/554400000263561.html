Introduction

This section discusses the Entitlement and Kickers screen of DGI.

II.2.D.11.a

Entitlement Used section

   

This section of the Entitlement and Kickers screen allows the user to add, edit, and delete any education entitlement used under a different benefit type.

Figure: Entitled Used section 

Adding used entitlement

Select the Add Entitlement button to open the Add/Edit Entitlement Used window which displays the following fields for entry.

(1)  Type – this field is required.  Select the type of benefit used.

(2)  Entitlement Used – this field is required.  Enter the amount of entitlement used by the beneficiary for the selected benefit type in a MM-DD format.  Processors must enter used chapter 30, chapter 32, chapter 34, chapter 35, chapter 1606, and chapter 1607 as shown in Benefits Manager (BM) or the BM Historical Reporting report for previously used benefits.

(3)  Eligible checkbox – this checkbox appears if the Type is chapter 30, chapter 1606, or chapter 1607.  Select this checkbox during original award processing if the beneficiary decides to make a relinquishment.

Note: If the beneficiary is eligible for chapter 30, chapter 1606, or chapter 1607 at the time of application for chapter 33 benefits, but has not used any entitlement, add an entitlement used of “00-00” to allow for proper benefit relinquishment, if necessary.

See (F) 2024 13 June: Procedural Advisory: Changes to Original Claims Processing After Rudisill v. McDonough Decision for details on the various relinquishment scenarios. 

(4)  Apply the 81-Month Rule checkbox – this checkbox appears if the Type is chapter 35.  Select this checkbox if the beneficiary is eligible for the 81-Month Rule.  See M22-4, Part XII, 2-3.08 for details on the 81-Month Rule.

Select the Save button for the entry to be saved.  Once Entitlement Used has been entered, it can be edited by selecting the Edit Entitlement button or deleted by selecting the associated checkbox and selecting the Delete button.

Note: If a beneficiary has used or is eligible for more than one non-chapter 33 benefit, the Add Entitlement button can be used to add multiple lines of entitlement used.

Figure: Add/Edit Entitlement Used window

See M22-4, Part XII, 2-3.08 for details on the 48-month rule.

  

II.2.D.11.b

Benefit Relinquished section

  

This section of the entitlement and kickers screen allows the processor to add or edit a benefit relinquishment to establish chapter 33 eligibility (if one is needed). 

See (F) 2024 13 June: Procedural Advisory: Changes to Original Claims Processing After Rudisill v. McDonough Decision for details on the various relinquishment scenarios. 

To do so, select on the Add Benefit Relinquished button which opens the Add/Edit Benefit Relinquished window with the following fields for entry.

(1)  Election Date – enter the relinquishment election date from the application or the appropriate alternative election date. 

(2)  Benefit Relinquished – select the appropriate benefit to relinquish.

Note: Only benefits added in the Entitlement Used section and marked as Eligible will display in the dropdown menu. 

Note: If chapter 1607 is relinquished, the Colmery Section 106 box must be checked.  See M22-4, Part XII, 2-2.06 for details on Section 106.


  Figure: Colmery Section 106 

(3)  Chapter 30 Remaining – this field only displays if chapter 30 is being relinquished.  Enter the amount of chapter 30 entitlement remaining at the time of relinquishment in a MM-DD format.

(4)  Alternative Election Made checkbox – Select this if an alternative election was made on behalf of the beneficiary.  This will add alterative election information to the letter generated by DGI.

Once a relinquishment has been added, the VCE can edit the relinquishment by selecting the Edit Benefit Relinquished button.

Figure: Benefit Relinquished example

Figure: Add/Edit Benefit Relinquished window

 

II.2.D.11.c

Chapter 33 Start section

This section displays the date that the beneficiary’s chapter 33 benefits start.  VCEs can adjust the chapter 33 start date to a date after the Election Date by entering a date in MM/DD/YYY format and selecting the Submit button.

Note: This date should be adjusted only in specific circumstances that require it. 

 

Figure: Chapter 33 Start section

II.2.D.11.d

Chapter 33 Delimiting Date section

This section displays the beneficiary’s delimiting date (if applicable) and allows the VCE to make changes to the delimiting date as appropriate.  To adjust the Chapter 33 Delimiting Date Override field, enter a new date in MM/DD/YYYY format and select the Submit button.

Figure: Chapter 33 Delimiting Date section

II.2.D.11.e

Periods of Chapter 33 Non- Participation section

This section displays any periods of non-participation for chapter 33.  VCEs can add a non-participation period by selecting Add Non-Participation Period to open the Add/Edit Period of chapter 33 Non-Participation window with the following fields for entry.

(1)  Reason – select the appropriate reason.

(2)  Start Date – enter the start date of the non-participation period.

(3)  End Date – enter the end date of the non-participation period.  If the non-participation period is ongoing, this field remains blank.

(4)  Notes – this field is not required and is a free text field for any notes.

Select the Save button for the entry to be saved.  Once period of non-participation has been entered, it can be edited by selecting the Edit Period button or deleted by selecting the associated checkbox and selecting the Delete button.

 Note: This field does not stop or adjust payments or entitlement.

Figure: Add/Edit period of chapter 33 non-participation

II.2.D.11.f

Kickers & Supplementals section

This section allows the VCE to manage kickers and supplemental education assistance.

 

Figure: Kickers & supplementals section

To add a kicker, select Add kicker/supplemental which opens the Add/Edit kicker/supplemental window with the following fields for entry.

(1)  Type – select the appropriate type of education assistance.  Options are:

•   Chapter 1606 kicker

•   Chapter 30 kicker

•   Chapter 33 kicker

•   Chapter 33 supplemental

(2)  Start Date – enter the date the kicker/supplemental is available for use in chapter 33.  It is a required field, is entered in MM/DD/YYYY format, and cannot be earlier than August 1, 2009.

(3)  End Date – enter the date the kicker/supplemental is no longer available for use in chapter 33.  It is a required field if an end date is listed in VIS and is entered in MM/DD/YYYY.  No kickers or supplementals will be paid after this date.

(4)  $/Month – the user enters the dollar amount per month for full-time training.  This is a required field.

(5)  Initial – the user enters the initial kicker/supplemental entitlement at the time of application for chapter 33.  This is a required field and is entered in MM-DD format.  The value may not exceed 36-00.

Note: To account for kicker rate changes, VCEs must enter an end date for the old rate and input a new kicker line with the new start date and new rate.

Select the Save button for the entry to be saved.  Once a kicker has been entered, it can be edited by selecting the Edit kicker button or deleted by selecting the associated checkbox and selecting the Delete button.

Figure: Add/Edit Kicker/Supplemental window 

II.2.D.11.g

Entitlement Transferred Out section

This section allows a VCE to manage transferred entitlement.

  

Figure: Entitlement Transferred Out section 

To transfer entitlement, select the Add transferee button to display the Transfer out entitlement window with the following entry fields.

(1)  SSN – enter the SSN for the dependent receiving the transferred entitlement.  A DGI BIO screen must be established for each SSN being any transfer is possible.

(2)  Ch33 entitlement – enter the amount of entitlement being transferred to the dependent in MM-DD format.  It is a required field, and this value must be less than or equal to the transferor’s remaining entitlement.  If transferred entitlement has been revoked and none was used, enter 00-00.

(3)  Begin date – enter the date the transferred entitlement is available to the dependent.

(4)  End date – enter the date that entitlement is revoked, if applicable.  It allows for a future date but may be left blank if no end date is provided.

(5)  Chapter 30 kicker – displays only if the transferor has a chapter 30 kicker added to DGI.  Enter the amount of chapter 30 kicker transferred to the dependent in MM-DD format.  It must be less than or equal to the amount of chapter 30 kicker remaining.  If transferred entitlement has been revoked and none was used, enter 00-00.

(6)  Chapter 1606 Kicker –displays only if the transferor has a chapter 1606 kicker added to DGI.  Enter the amount of chapter 1606 kicker transferred to the dependent in MM-DD format.  It must be less than or equal to the amount of chapter 1606 kicker remaining.  If transferred entitlement has been revoked and none was used, enter 00-00.

(7)  Relationship – select the appropriate relationship between the transferor and the dependent (either child or spouse).

Select the Save button for the entry to be saved.  Once the transfer of entitlement has been entered, it can be edited by selecting the Edit button.

 

Figure: Entitlement Transferred Out window

II.2.D.12.a

Enrollment Manager Inbox

The Enrollment Manager Inbox (formerly the VA Online Certification of Enrollment (VA-ONCE) Inbox) section is used to display enrollment and change in enrollment information submitted by the school via Enrollment Manager.

(1)  If there are no Enrollment Manager submissions for the claimant, the message “The claimant has no Enrollment Manager records at this time” will be displayed.

Figure: Enrollment Manager Inbox – No records example

(2)  If new information has been sent through Enrollment Manager, the submissions will be displayed in the Enrollment Manager Inbox.

 

Figure: Enrollment Manager Inbox – Submissions displayed example 

(3)  The Decline, Add, and Correct buttons will become active depending on the specific circumstances when an Enrollment Manager submission is selected.

(4)  School information submitted on an electronic VA Form 22-1999 will be displayed with Enrollment as the Type.

(a)  If the begin date, end date, and facility code of the Enrollment Manager submission do not match an existing enrollment in the Enrollment Periods section, this is considered a new enrollment and the Decline and Add buttons become active.

•  Selecting the Decline button will prompt the VCE to confirm the decision.  If the submission is continued, the enrollment will be removed from the Enrollment Manager Inbox. Otherwise, choose Cancel to return to the Enrollment screen.

•  Selecting the Add button will launch the Add Enrollment window. The data entry fields will be auto-populated with the Enrollment Manager data, as well as the information from Web Enabled Approval Management System (WEAMS) and may be modified if necessary. If the Accept button is clicked, the information will be displayed in the Enrollment Periods section. Clicking the Cancel button returns the VCE to the Enrollment screen and keeps the enrollment in the Enrollment Manager Inbox.

(b)  If the begin date, end date, and facility code of the Enrollment Manager submission match an existing enrollment in the Enrollment Periods section (such as a correction to the tuition of fee charges), the Decline and Correct buttons become active

•  Selecting the Correct button will launch the Correct Enrollment window. The data entry fields will be auto-populated with the Enrollment Manager data and may be modified if necessary.

•  Selecting the Accept button results in the information being displayed in the enrollment periods section. Selecting the Cancel button returns the VCE to the enrollment screen and keeps the enrollment in the Enrollment Management Inbox.

(c)  If the school is correcting one or both of the term dates (the begin date or end date of the Enrollment Manager submission match an existing enrollment in the enrollment periods section), the processor must manually associate the Enrollment Manager submission by selecting on the appropriate original enrollment being amended from the enrollment periods section. 

(5)  School information submitted on an electronic VA-Form 22-1999B will generally be displayed with Amendment as the Type.  If there is no amendment with the same effective date in the enrollment periods section, the Decline and Add buttons will become active.

•  Selecting the Decline button prompts the processor to confirm the decision. If the submission is continued, the enrollment will be removed from the Enrollment Manager Inbox. Otherwise, select Cancel to return to the Enrollment screen.

•  Selecting the Add button will open the Add Amendment window. The data entry fields will be auto-populated with the Enrollment Manager data, including attendance and mitigating circumstance information in the change profile section. It may be modified if necessary. Selecting the Accept button results in the information being displayed in the Enrollment Periods section. Selecting the Cancel button returns the VCE to the enrollment screen and keeps the enrollment in the Enrollment Manager Inbox

(6)  If there is an amendment with the same effective date in the enrollment periods section (because this is a corrected amendment), the Decline and Correct buttons will become active.

Selecting the Correct button launches the Correct Amendment window. The data entry fields will be auto-populated with the Enrollment Manager data, including attendance and mitigating circumstance information in the change profile section, and may be modified if necessary. Selecting the Accept button results in the information being displayed in the enrollment periods section. Selecting the Cancel button returns the VCE to the enrollment screen and keeps the enrollment in the Enrollment Manager Inbox.

(7)  If the school information submitted on an electronic VA Form 22-1999B that does not match the facility code, begin date, and end date of an original enrollment in the enrollment periods section will be displayed with orphan amendment as the Type.

The VCE must manually enter and/or associate the orphan amendment with an original enrollment to activate the Add or Correct buttons. The Decline button will already be active. 

(8)  An increase in enrollment, a reduction in enrollment, or a correction to the residence or distance hours effective on the first day of the term will be displayed with Amendment—First day of term as the Type.  

If the Amendment—First day of term is selected, this will activate the Decline and Correct buttons.

Selecting the Correct button opens the Correct Enrollment window. The data entry fields will be auto-populated with the Enrollment Manager data. Selecting the Accept button results in the information being displayed in the Enrollment Periods section. Selecting the Cancel button returns the VCE to the Enrollment screen and keeps the enrollment in the Enrollment Manager inbox.

(9)  Certain types of Enrollment Manager submissions will not be associated with enrollments or amendments in the Enrollment Periods section. The record can be associated to a record in the Enrollment Periods section by clicking on the row. This will highlight that enrollment or amendment and activate the Correct button.

All Enrollment Manager Inbox records must be processed for a claim to be authorized.

 

 

 

 

II.2.D.12.b

Enrollment Section - General

The Enrollment section of the Enrollment screen of DGI displays all previously authorized, in-progress, and deleted terms.  VCEs can expand each enrollment to view the details as necessary. 

(1)  The following fields are visible for each enrollment.

(a)  Status – displays in small text above the start date.  Possible values are:

•  Authorized,

•  In Progress, or

•  Deleted

(b)  Dates – displays the date range of the enrollment.

(c)  Training Type – displays the type of training for the enrollment.  Possible values are:

•  Grad,

•  Undergrad,

•  NCD,

•  Flight,

•  App,

•  OJT, or

•  Corr.

(d)  Facility Code – displays the facility code associated with the enrollment.

(f)  Facility Name – displays the facility name associated with the enrollment as listed in WEAMS.

(g)  Program Code – displays the program code of the program in which the beneficiary is enrolled.

(h)  Program Name – displays the program name of the program in which the beneficiary is enrolled.

 

Figure: Enrollment section example

(2)  Further information can be viewed by selecting the blue arrow at the right of the enrollment row.  The following fields display for graduate, undergraduate, and non-college degree (NCD) programs.

(a)  Effective Date – displays the effective date of the enrollment (i.e., the start date of the term) or of the amendment (i.e., the date the change went into effect).

(b)  Type – displays the type of enrollment or amendment.  Possible values include:

•  Original Enrollment,

•  Reduction – Protected,

•  Reduction – Non-Protected,

•  Termination – Protected,

•  Termination – Non-Protected, or

•  Unchanged.

(c)  Hour Type – displays the hour type of the enrollment.

(d)  STEM – indicates whether the enrollment is using STEM entitlement.

(e)  FT Hrs – displays the hours considered full time for the enrollment.

(f)  Tuition, Fees, and Yellow Ribbon – display the amount entered for each value from the enrollment certification.

(g)  Res. Hrs and Dist. Hrs – display the number of residence and distance hours in which the beneficiary is enrolled.

(h)  Vacation Days – displays the number of vacation days reported for the enrollment.

(i)  Apply 6X Exclusion

(j)  Notes – displays information on any checkboxes selected by the SCO during the amendment process.

 

Figure: Expanded Enrollment section – Graduate/ Undergraduate/NCD example

(b)  Hours/Week – displays the number of hours per week worked.

(c)  Starting Step – displays the step the OJT or App started on.

(d)  Elapsed Time – displays amount of time that has passed during the OJT or App at the time of initial enrollment certification.

Figure: Expanded Enrollment section – OJT/APP example

(a)  Type – displays the type of enrollment.  It will display Original Entitlement for Flight enrollments.

(b)  Dual Hours – displays the submitted dual hours from the enrollment certification.

(c)  Dual Simulator Hours – displays the submitted dual simulator hours from the enrollment certification.

(d)  Solo Hours – displays the submitted solo hours from the enrollment certification.

(e)  Ground Hours – displays the submitted ground hours from the enrollment certification.

(f)  Pre/Post Hours – displays the submitted pre/post hours from the enrollment certification.

(g)  Other Hours – displays any other submitted hours from the enrollment certification.

(h)  Submitted Charges – displays the dollar amount of the submitted charges from the enrollment certification.

Note: Processors can use the View Certs button to Add or Edit flight certifications (Certs).

(b)  Submitted Lessons – displays the number of lessons submitted.

(c)  Non-Reimbursable Lessons – displays the number of lessons that are not reimbursable.

(d)  Submitted Rate – displays the dollar amount of the rate per lesson submitted.

II.2.D.12.c

Managing Graduate Enrollments

In this section, the training type Graduate or Advanced Professional (Grad) is discussed.

(1)  Adding a new grad enrollment.  

Select the Add Enrollment button to open the Add Enrollment window which requires the VCE to select a Training Type.  Upon selecting Graduate or Advanced Professional, the Add Graduate or Advanced Professional window opens with the following entry fields.

(a)  Training Type – this field defaults to Graduate or Advanced Professional for this window.  Changing the type will change the window.

(b)  Begin Date – enter the first day of the term as shown on the enrollment certification.

(c)  End Date – enter the last day of the term as shown on the enrollment certification.

(d)  Facility Code – enter the 8-character (alphanumeric) code that corresponds with the institution the beneficiary is attending.  The facility code must match the codes found in the WEAMS.

(e)  Objective – select the appropriate option.  Options are:

•  031 MA

•  032 MS

•  033 MD-NEC

•  041 DRPHILOSP

•  042 DR-NEC

•  043 PSTDR-NEC

(f)  Program Code – enter the 3-digit code associated with the enrollment and shown on the enrollment certification or in WEAMS.

(g)  Program Name – enter the program shown on the enrollment certification or in WEAMS.

(h)  Hour Type – select the appropriate option.  Options are:

•  Semester

•  Quarter

•  Clock (standard)

(i)  FT Hours – enter the FT hours as shown on the enrollment certification.

(j)  Res Hrs – enter the number of residence hours certified on the enrollment certification.

(k)  Dist Hrs – enter the number of distance hours certified on the enrollment certification.

Note: Total residence and distance hours must be greater than zero.

(l)  Vacation Days – this field should not be used for grad enrollments.

(m)  Tuition Charged – enter the amount of the tuition reported on the enrollment certification.

(n)  Fees Charged – enter the amount of the fees reported on the enrollment certification.

(o)  + Vacation Period – this field should not be used for grad enrollments.

(p)  Yellow Ribbon Amount – enter the amount of the Yellow Ribbon contribution reported on the enrollment certification.

(q)  Out of State Tuition and Fees – enter the amount of the out of state tuition and fees reported on the enrollment certification.

(r)  Interval Payments – effective August 1, 2011, interval payments are no longer permitted.  Default value of No Restrictions is maintained for all enrollments.

(s)  Restore Entitlement for Enrollment Period checkbox. – See (J) 2020 30 October: Procedural Advisory: Colmery Section 109 Restoration of Entitlement, LTS update. for more details.

Select Save to add Enrollment.

Figure: Add Graduate or Advanced Professional window

(2)  Correcting a Grad Enrollment

Select the Correction button to open  the Correct Graduate or Advanced  Professional window that displays the data from the original enrollment.  During correction, edited fields are updated in the original record as well as amendments.  Correction functionality should be used in the following circumstances:

(a)  To correct data entry errors before authorization.

(b)  To correct data entered under a new work product from DGI user error, or because of a correction submitted by the school (such as an amended VA Form 22-1999) after the initial award has been authorized.

(c)  To process an increase or reduction in enrollment hours that occurred on or before the first day of the term.

Select the Save button to return to the Enrollment screen.

 

Figure: Correct Graduate or Advanced Professional window 

(3)  Amending a Grad Enrollment

Selecting the Add Amendment button allows VCEs to make changes to a claim when the original award (or any other amendment) has a status of either In Progress or Authorized.  The amendment functionality should be used for any change in hours reported on a VA Form 22-1999B (Notice of Change in Student Status) or provided otherwise, except for an increase or reduction in hours that occurred on or before the first day of the term.  

The left side of the window contains data entered in the original enrollment that is in read-only format.   

(a)  Effective Date – enter the effective date of the change being reported.

(b)  Reason – select from the available options as listed.

  • Increase
  • Prereg never attended
  • Reduction (NCD)
  • Reduction NP
  • Reduction pun
  • Reduction during drop
  • Completed NP
  • End of term or course
  • Graduation
  • Unsat
  • WD before term
  • WD after drop NP
  • WD after drop Pun
  • WD during drop
  • WD or interrupt (NCD)
  • Enrollment Manager termination
  • Hours changed between online & res courses
  • Other

(c)  Enter data as reported on the amendment certification for the following fields.

  • Revised Res Hours
  • Revised Dist Hours
  • Revised Tuition
  • Revised Fees
  • Revised Yellow Ribbon

The column on the right contains the following options:

(a)  Student Did Not Attend Class. Select this box for a termination, which is effective on or before the first day of the term. No other boxes need to be checked for this scenario. Ensure all credit and dollar amount entries are zeroed out.

(b)  Mitigating Circumstances are Required. Select this box if there is a reduction or termination after the school’s drop/add period, non-punitive grades are assigned, and restoration of entitlement does not apply (there is an unprotected change in enrollment). This will automatically allow for the application of the six-credit hour exclusion if it is available.

(c)  Mitigating Circumstances are Provided. Select this box if the Mitigating Circumstances are Required box is checked and mitigating circumstances are received either with the change in enrollment or after development.

(d)  Mitigating Circumstances are Acceptable. Select this box if the Mitigating Circumstances are Required, and the Mitigating Circumstances are Provided box have both been checked. This will tell DGI that mitigating circumstance are acceptable and apply the exclusion to the term.  If the mitigating circumstance is not acceptable, this box remains unselected.

(e)  Restore Entitlement for Enrollment Period. Select this box if there is a reduction or termination and restoration of entitlement rules apply. This will prohibit the application of the six-credit hour exclusion.

(f)  School Closure/Program Disapproval. See (J) 2020 30 October: Procedural Advisory: Colmery Section 109 Restoration of Entitlement, LTS update. for details.

(g)  Continue Monthly Housing Allowance. See (J) 2020 30 October: Procedural Advisory: Colmery Section 109 Restoration of Entitlement, LTS update. for details.

(h)  COVID-19 Negative Impact due to school/program modifications. See (E) 2021 18 May: Procedural Advisory: Section 1102 and 1103 of Public Law (PL) 116-315 for Chapter 33 Processing for details.  

(i)  Do Not Apply 6X. Select this box if the six-credit hour exclusion is available but should not be applied. The selection is not available if there is a 6X date entered on the Claimant Biography screen.  This is applicable in the following situations:

  • An enrollment certification is received and a VA Form 22-1999B for an unprotected reduction/termination is submitted before authorization of the original award. In this case, both the original enrollment, and amended enrollment should be entered under the same work product. The boxes for Mitigating Circumstances are Required and Do Not Apply 6X should be checked.
  • A VA Form 22-1999B is received indicating completion of the term with nonpunitive grades, and mitigating circumstances are not reported. In this case, the boxes for Mitigating Circumstances are Required and Do Not Apply 6X should be checked.
  • A VA Form 22-1999B is received, indicating completion of the term with nonpunitive grades, and mitigating circumstances are reported. In this case, the boxes for Mitigating Circumstances are Required, Do Not Apply 6X, and Mitigating Circumstances are Provided should be checked. If applicable, the Mitigating Circumstances are Acceptable box should also be checked. 
  • Mitigating circumstances are received pursuant to development where benefits were not paid, or the term was completed with nonpunitive grades. In these cases, the Amended Award check boxes should be modified under the correction functionality. The Mitigating Circumstances are Provided box should be checked. If applicable, the Mitigating Circumstances are Acceptable box should also be checked. 

Once all the appropriate boxes have been selected, select the Save button to continue processing the claim, or select the Cancel button to return to the Enrollment Screen.

Figure: Add Graduate or Advanced Professional Amendment window

See M22-4, Part IV, 11 for details on the six-credit hour exclusion, mitigating circumstances, enrollment adjustments, and restoration of entitlement.

 

 

 

 

II.2.D.12.d

Managing Undergraduate Enrollments

II.2.D.13.a

General

DGI automatically generates letters when possible. The Letter Information screen will indicate which letter(s) were generated due to the data entered in the current work product. This screen may also have prompts for the VCE to enter additional information, based on the letter that is being generated. A Comments section displays on the screen.  Entering a comment is not required, but anything entered will display in the Comments section of the WPS Screen (see Topic 14: Work Product Summary Screen) after the VCE selects the Next button.

If a POA is listed on the BIO screen, letters produced for that beneficiary will generate with a carbon copy (cc) line for the POA. A copy of the letter will also be generated. When letters are automatically sent to the print vendor for mailing, the POA copy of the letter will be shipped to the appropriate RPO for distribution.

The Station Number indicated on the BIO screen determines the return address displayed on the top of system generated letters. If the beneficiary is under 18 years of age, their letters will generate using the information entered in the Parent/Guardian Information section of the BIO screen.

DGI cannot generate letters for VET TEC, and no letters are be produced when VET TEC is the benefit source. System-generated letters are also filtered out of the following work product types: FETImport, WorkProductZero, Reconciliation.

The WPS screen allows the VCE to suppress system-generated letters and upload modified letters. More information can be found in Topic 14: Work Product Summary Screen.

Note: All system generated letters should be released unless the information is incorrect, requires editing, or there is specific guidance to suppress.

 

 

 

 

II.2.D.13.b

Certificate of Eligibility

Generally, a certificate of eligibility (COE) letter will generate when the beneficiary is eligible and no award actions (payments and/or debts) result in the current work product.

This may include scenarios where an enrollment or amendment has been added to DGI without a payment or debt being scheduled or created (for example, a flight or correspondence course was entered without any certs). When this occurs, the COE letter will contain a paragraph explaining that the program is approved, but no payments have been made because nothing is due. In this case, the Payment Letter will also generate, and is the correct letter to send for the scenario. A future system release will include the deletion of the COE from these work products. Until that time, the COE should be suppressed by the user.

II.2.D.14.j

Net Awards section

The net awards section displays the details of all payments or receivables calculated based on the actions taken in the current work product.

Recurring payments: Before August 1, 2021, future housing and kicker payments were issued using recurring (06H) payments.  The payments began on the first day payment was due and ended on the last day payment was due.  The monthly housing rate was displayed in the housing field, and the monthly kicker rate was displayed in the appropriate kicker field, if one was due.  Each month, a full file pass was run in BDN to issue the actual amount due for that month.  Each time a housing or kicker payment was released by BDN, the Date Last Paid (DLP) was updated.  Any work product created while a recurring award was running would stop on the DLP and created a new payment for the DLP until the end of the term.

Because enrollment verification is now required for chapter 33, recurring payments are no longer used.  Housing and kicker payments are now issued using lump sum housing (06I) payments that can be put on hold if a beneficiary does not verify their enrollment for more than two months.  See M22-4, Part II, 2.D.5 for details.

Figure: Recurring payments section

Lump sum payments: Lump sum payments include both payments and debts being created in the current work product.  The payments can be scheduled for future release, as applicable. 

Housing and/or kicker payments are broken down into monthly ranges and are scheduled for release at the end of the corresponding month.  These payments are placed on hold if the beneficiary does not verify their enrollment for more than two months (see M22-4, Part II, 2.D.5 for details).  If a beneficiary’s payment status is “hold” for failing to verify their enrollment, this status reflects on the Enrollment Verification and Payment Instruction screens but will not display on the WPS screen.

Retroactive housing and/or kicker payments are combined into one 06I lump sum transaction that is separated from any current or future month(s) housing and/or kicker payments scheduled.  Retroactive payments are defined as lump sum payments for periods prior to the current month.

Payments of books and supplies, tuition and fees, and Yellow Ribbon are generally scheduled to be released two weeks prior to the start of an enrollment period. 

Payments created by manual or miscellaneous awards are released as lump sum payments.

Distinct BTs are maintained for different award types and different recipients.  For example, separate tuition and fees BTs are created when terms at different facilities are processed in one work product for one beneficiary.

Currently, STEM payments are not automatically sent to eMPWR for release.  They display in the Lum Sum Payment section and are sent to the Payment Instructions screen (see Topic 15: Payment Instructions Screen) as payments needing manual release in eMPWR using current STEM procedures found in (H) 2020 04 August: Procedural Advisory: STEM Scholarship Processing in LTS – Buffalo STEM Team.

The manual awards section of the WPS page displays manual awards made in previously authorized work products and allows the creation of new manual awards.  Previously authorized manual awards display with a status of Authorized and cannot be edited or deleted. 

Manual awards can be used to add or charge entitlement.  They can also be used to create payments or debts for tuition and fees, Yellow Ribbon, books and supplies, housing, and/or kickers.  All manual awards display in the lifetime awards section, and all manual awards and debts created in the current work product display in the net awards section.

Note: Manual awards should be used only when necessary or when guidance directing their use in a specific scenario has been issued.  Work products containing manual awards require two signatures. 

Figure: Manual awards section

(1)  Creating a Manual Award

Selecting Add Manual Award will display the Add/Edit Manual Award window. The fields in this box change based on the type of manual award being entered. Fields with an asterisk are required, and the description field is available to describe the reason for the award. 

Select Save after all required information has been entered to add the manual award with a status of in-progress. Selecting the Edit button for an in-progress manual award will reopen the Add/Edit Manual Award window.

Selecting the checkbox on the far left of an in-progress manual award allows the use of the Delete Manual Award button, which deletes all checked awards. 

Available types of manual award action include the following.

(a)  Adding entitlement: Using the Add Entitlement manual award type will subtract the amount entered from the entitlement used under chapter 33, and it will increase the entitlement remaining by the entered amount. There is no system limitation to prevent adding more entitlement than was previously used. When this occurs, the used entitlement in the beneficiary information banner will reflect as a negative number.

(b)  Charging entitlement: Using the Charge Entitlement manual award type will add the amount entered to the entitlement used under chapter 33, and it will decrease the entitlement remaining by the entered amount. Charging entitlement requires an effective date, and it can cause the entitlement to exhaust. The charge will be applied to the record on the effective date and will not affect any terms that end before that date. Entitlement exhaust rules will be applied to any terms ending after the effective date.

(c)  Manual TOE entitlement adjustments: Before May 18, 2015, only whole months of entitlement could be transferred to dependents on the Entitlement and Kickers screen. If transferred days were required, it had to be done through manual awards to charge or add. When the functionality to transfer months and days to a dependent was added, it was to be used for new TOE claims only. If there have ever been manual awards to add or charge days on a TOE claim for the dependent or the Veteran, the days transferred will always have to be adjusted using manual awards. Simply adding an opposite manual award to cancel the old one may cause erroneous entitlement calculations when days are then transferred using the Entitlement and Kickers screen.

(d)  Manual payments and debts: Manual payments are created by choosing the type of manual award needed. If a negative amount is entered, a debt is created. All non-STEM manual payments or debts will be sent to eMPWR when the work product is authorized.

(e)  STEM manual awards: If a STEM manual award is used for an eligible STEM beneficiary, the payment will be sent to the Payment Instructions screen for manual entry into the eMPWR payment system. If a STEM type manual award is used for a beneficiary who is not eligible for STEM payments, the award will display in the Manual Awards section, but it will not be included as a net award or sent to the Topic 15: Payment Instructions Screen) when authorized.

Miscellaneous benefits section

The miscellaneous benefits section of the WPS page displays previously authorized miscellaneous awards and allows the creation of new awards or the correction of authorized awards.  All miscellaneous benefit awards display in the lifetime awards section and all awards in the current work product also display in the net award section.

(1)  Creating miscellaneous benefits

Select Add Benefit to display the Add/Edit Miscellaneous Benefits window with an option to select the type of benefit with the details below.  Select Save after all required information has been entered to add the award with a status of in-progress.

Figure: Miscellaneous benefits section

Select Edit for an in-progress award to reopen the Add/Edit Miscellaneous Benefits window. Selecting the checkbox on the far left of an in-progress award allows the use of the Delete Benefit button, which deletes all checked awards. 

Available types of miscellaneous awards include the following.

(a)  Licensing and Certification (LAC)

  • See M22-4, Part XII, 4-2.03 for background on corresponding entitlement charge.
  • Amount Claimed field – enter the amount claimed by the beneficiary.  The maximum enterable valuable = $9,999.99.  Maximum net award created will not exceed $2,000.00 per test.
  • Date field – enter the date of the test being claimed and cannot be a future date.
  • Description – enter a description for all LAC claims. 

Figure: Add/Edit miscellaneous benefits – LAC

(b)  National Exam

  • See M22-4, Part XII, 4-2.03 for background on corresponding entitlement charge.
  • Amount Claimed field – enter the amount claimed by the beneficiary.  The maximum enterable valuable = $9,999.99.  Maximum net award created will not exceed $2,000.00 per test.
  • Date field – enter the date of the test being claimed and cannot be a future date.
  • Description – enter a description for all National Exam claims.

Figure: Add/Edit Miscellaneous Benefits – National Exam

(c)  Refund of CH30 Contributions.  

i.  This option is only available for WPs satisfying all the following conditions:

  • An entitlement of Self, and
  • Chapter 30 benefit was relinquished, and
  • An entitlement exhaust date is populated, and
  • Any entitlement transferred to dependents was used prior to entitlement exhaust date, and
  • A housing award exists on or spanning the entitlement exhaust date.

ii.  CH30 Contribution amount field – enter a value up to $1,200.00 as determine by the total amount of the participation contribution paid by the beneficiary.

Figure: Chapter 30 eligibility data section of VIS Education screen

iii.  Refund amount field: Calculated by rules based on chapter 30 contribution amount entered by VCE and the remaining chapter 30 entitlement at the time of relinquishment.  See M22-4, Part XII, 4-6.01 for rules details.

Note: This value will populate after the user saves the chapter 30 contribution amount. 

iv.  Refund issue date field: Automatically populated based on the following rules.

  • If the exhausting term has completed at the time of processing, the refund date is the date of authorization of the work product.
  • If the exhausting term has not been completed, the refund issue date equals the first day of the month after the term end date.  This date acts as a maturation date, so the refund will be issued at that time unless interim action causes it to be no longer applicable.

v.  The chapter 30 contributions refund is only payable one time.  An error displays if more than one chapter 30 contributions refund is entered.

Note: Before issuing a chapter 30 refund via DGI, VCE must ensure that no such refund was paid via BDN.

Important: Work products containing chapter 30 refunds procedurally require second signature authorization following standard second signature procedures.  No chapter 30 refund should be authorized with only one reviewer.

Figure: Add/Edit miscellaneous benefits – Refund of chapter 30 contributions

(d)  Rural Relocation

i.  When a beneficiary is found eligible for a payment of the rural relocation benefit (see M22-4, Part XII, 4-6.04 for eligibility details), the VCE enters the following information.

  • Date field – enter the date claimed for the rural relocation.  It cannot be a future date.
  • Award Amount field – enter $500.00 which is always the value for Rural Relocation.
  • Description field – enter a description for all rural relocation claims.

ii.  The rural relocation benefit is only payable one time.  An error displays if more than one rural relocation benefit is entered.

 

Figure: Add/Edit miscellaneous benefits – Rural relocation

(e)  Tutorial Assistance

When a beneficiary is found eligible for the tutorial assistance (TA) program (see M22-4, Part XII, 3-10 for eligibility details), the VCE enters the following information.

i.  Previous TA Paid OOS field – enter the amount of any TA (Non-chapter 33 or chapter 33) paid outside of the DGI system.  This value cannot exceed a lifetime value of $1,200.00.

ii.  Begin Date and End Date fields – enter the dates of the TA claimed. For each TA entered, the dates must be of the same month and year and cannot be future dates.

iii. Amount Remaining field – enter the amount claimed by the beneficiary (or $100.00, if amount claimed is greater) for the month being entered.  The value cannot exceed $100.00

iv.  Description field – enter a description for all tutorial assistance claims.

Figure: Add/Edit miscellaneous benefits – Tutorial assistance

(f)  Work Study

When a beneficiary is found eligible for the work study program (see M22-4, Part XII, 3-9 for eligibility details), the VCE enters the following information to issue a payment.

i.  Begin Date and End Date fields – enter the dates of the work study period being claimed.  These values cannot be future dates.

ii.  Hours field – enter the number of hours claimed.  This field accepts quarter hour increments entered as a decimal (.25, .50, .75, or .00

iii. Amount Paid field – enter the amount due.  This must be no greater than $1,500.00.

iv.  Description field: – enter a description for all work study claims.

Figure: Add/Edit miscellaneous benefits – Work study

(2)  Correcting Miscellaneous Benefits

(a)  Previously authorized miscellaneous awards display with a status of authorized. 

(b)  Select Correction to open the Add/Edit Miscellaneous Benefits window.

(c)  Selecting the Save button when nothing was changed will display the award with a status of in-progress.

(d)  Select Edit for an in-progress award to reopen the Add/Edit Miscellaneous Benefits window.

(e)  Select the checkbox on the far left of an in-progress award to use the Delete Benefit button, which deletes all checked awards.

(f)  If the Amount Claimed is increased during a correction, a payment will be created for the difference between the amount previously paid and what is due now. The entitlement charged will also increase for those miscellaneous benefits that charge entitlement.

(g)  If the Amount Claimed is decreased during a correct, a debt will be created for the difference between the amount previously paid and what is due now. Entitlement will be restored according to the new amount paid for those miscellaneous benefits that charge entitlement.

(h)  A previously authorized miscellaneous benefit can be deleted by returning it to an in-progress status using the Correction button. Deleting a miscellaneous benefit will remove it from the miscellaneous benefits section, restore any entitlement previously charged for the payment, and create a debt for the amount previously paid.

(3)  Letters for Miscellaneous Benefits

Generally, miscellaneous benefits will be included in the Payment Letter, if they create a positive (payment) or negative (debt) net award. 

 

 

 

 

II.2.D.14.n

Letters section

The Letters section displays the system generated letters.  It includes three buttons.

(1)  Send system generated letters: This is the default option.  All letters are displayed as PDF documents and release to the print vender upon WP authorization.

(2)  Suppress letter(s): This option turns all letters to Word documents.  No letters are released upon authorization.

(3)  Allow manual letters upload: This option turns all letters to Word documents.  When using this option, selecting the authorize button displays a box that allows the upload of modified letters and the suppression of individual letters.

Figure: Letters section of the Work Product Summary page

(4)  Individual letters are suppressed by selecting the Do Not Send checkbox.

(5)  Selecting Choose File allows the upload of modified letters in PDF format. Only one file can be uploaded at a time. To upload an additional letter, select Choose File again. Uploaded letters cannot be suppressed, so they should be deleted if incorrect.

Note: To avoid print failure, uploaded letters must be a PDF document, and must be created from the DGI letter template. Making any alterations to the seal or address may cause the failure of all letters in a print batch, not just the ones with invalid formatting. System validations have been added to verify the document type and location of the seal and address. If a letter fails the validation, selecting Choose File again will allow the user to upload a corrected document.

(6)  Selecting the Authorize button on this screen should only be done when all information has been reviewed because this action cannot be undone. The work product will be authorized, letters with Do Not Send indicated will be suppressed, any uploaded and unsuppressed system-generated letters will be sent, and any payments generated by the work product will be sent to eMPWR based on the Payment Option button.

(7)  After the work product is authorized, all letters will display as PDF documents. These letters serve as the record of what was sent to the beneficiary, and there is no need to capture the letter to TIMS. Beginning July 30, 2022, a NOT SENT message displays over the PDF icon for a letter that was suppressed in the work product.

(8)  The displayed letters are recreated based on the system rules in place on the day the letter is being viewed. This means that when viewing historical letters, they will have present day language and forms.

Example: Historical letters from work products that were authorized before the Appeals Modernization Act will display with the new appeal rights paragraph and VA Form 20-0998.

 

 

 

 

II.2.D.14.o

Comments, navigation, and authorization section

The comments box displays any comments that were added on previous screens. It also allows the addition of a new comment. Comments that are added on this screen are saved if Done or  Authorize is selected after typing them. If the user navigates away from the WPS screen without selecting one of these buttons, any new comments added will be removed. Saved comments display on the work product after it is authorized.

Selecting the Previous button navigates back to the Topic 13: Letter Information Screen. Selecting the Done button will save the changes made to the Work Product Summary screen and navigate to M22-4, Part II, 2.D.6.

The text on the Authorize button changes based on Payment Options button selected at the top of the screen. Selecting this button allows manual letter upload if that option is selected in the Letters section of the screen. Selecting the Authorize button on this screen should only be done when all information has been reviewed because this action cannot be undone. The work product will be authorized, letters will be suppressed or sent based on user selection, and any payments generated by the work product will be sent to eMPWR based on the Payment Option button. If the work product being authorized contained payments, the user is navigated to the Topic 15: Payment Instructions Screen when authorization is successful.

Figure: Comments and authorization section of the Work Product Summary page

II.2.D.15.a

General

The Payments Instructions (PI) screen is mostly a read-only screen that shows the status of transactions sent automatically to eMPWR.

At the top of the PI screen, relevant beneficiary information is displayed including the following.

  • Comment Box – allows VCEs to enter any relevant notes.  Select Save to save any entries.
  • Beneficiary Name field – displays the name of the beneficiary for the claim in first name – middle initial – last name format.
  • Beneficiary File Number field – displays the file number of the beneficiary.
  • Payee Number field – displays the payee number assigned to the beneficiary.
  • Veteran File Number field – displays the file number for the source of entitlement.  If the beneficiary is the source of his or her own entitlement, this field matches the beneficiary file number field.  If the beneficiary is using entitlement transferred from a spouse or parent or entitlement under the Fry program, this field displays the file number of that spouse or parent.
  • Authorization Date field – displays the date the work product was authorized.

Note: Before the conversion to eMPWR, the PI screen showed a “Stub Name” field that showed the beneficiary’s first initial, middle initial, and first five characters of the last name since this value was used by BDN.  eMPWR does not use the stub name, so it no longer displays.  For WPs authorized on or before June 26, 2024, the stub name continues to display.
