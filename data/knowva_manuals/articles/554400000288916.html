Topic 1: General information

Introduction

This section discusses the general layout of the Enrollment screen of BM.

Change Date

October 18, 2024

II.3.F.1.a

General

The Enrollment screen of BM displays the following sections.

  1. Enrollment verification information
  2. Enrollment periods
  3. Manual awards

II.3.F.1.b

Enrollment verification section

The Enrollment verification section of BM displays the following information.

Verified through date:  This field displays the current enrollment verified through date for the benefit type of the claim.  This field cannot be edited here by users and displays in MM/DD/YYYY.

See M22-4, Part II, 3.C for more details on enrollment verification.

Figure: Enrollment verification example

II.3.F.1.c

Enrollment periods section

The Enrollment periods section of BM allows the user to add, correct, or amend an amendment and displays the following information.

  1. Begin date: This field displays the begin date of the given term.
  2. End date:  This field displays the end date of the given term.
  3. Training type:  This field displays the type of training.
  4. Facility code and school name:  This field displays the facility code and name of the training institution.
  5. Objective:  This field displays the objective of the term.
  6. Program name:  This field displays the name of the program.

The user can expand the basic enrollment period information to display the following information depending on the training type.

  1. If the Training type is non-college degree (NCD), the following fields display:
    1. Hours type:  Possible values include:
      1. Semester,
      2. Quarter, or
      3. Clock.
    2. Full-time (FT) hours:  This field displays the number of hours considered full-time at the school.
    3. Approved hours:  This field displays the approved hours for the program.
    4. Vacation days:  This value displays the number of vacation days during the term.
  2. If the Training type is Graduate, the following fields display:
    1. Hours type:  Possible values include:
      1. Semester,
      2. Quarter, or
      3. Clock.
    2. FT hours:  This field displays the number of hours considered full-time at the school.
    3. Vacation days:  This value displays the number of vacation days during the term.
  3. If the Training type is Undergraduate, the following fields display:
    1. Hours type:  Possible values include:
      1. Semester,
      2. Quarter, or
      3. Clock.
    2. FT hours:  This field displays the number of hours considered full-time at the school.
    3. Vacation days:  This value displays the number of vacation days during the term.

If the enrollment period has any associated amendments, the following information displays based on training type.

  1. If the Training type is NCD, the following fields display:
    1. Effective date:  This field displays the effective date of the amendment in MM/DD/YYYY format.
    2. Type:  This field displays the amendment type with the following possible values:
        1. Increase,
        2. Reduction (NCD),
        3. Reduction non-punitive (NP),
        4. Reduction punitive,
        5. Reduction during drop,
        6. Completed NP,
        7. End of term or course,
        8. Graduation,
        9. Unsatisfactory progress,
        10. Pre-registered never attended,
        11. Withdrawal (WD) before term,
        12. WD after drop NP,
        13. WD after drop pun,
        14. WD during drop,
        15. Withdrawal (NCD),
        16. Leave of Absence (LOA) termination (NCD), or
        17. Other.
  2. Enrolled hours:  This field displays the number of hours in which the beneficiary is enrolled.
  3. Tuition & fees:  This field displays any tuition & fees certified with the enrollment in $XX.XX format.
  4. Details:  This field displays a summary of the reason for the amendment with the following possible values:
    1. Do not apply six credit hour exclusion,
    2. Incarcerated,
    3. Increase – LOA,
    4. Mitigating circumstances required,
    5. Mitigating circumstances received,
    6. Mitigating circumstances acceptable,
    7. Restore entitlement,
    8. Revised end date,
    9. Termination – LOA, or
    10. Top-up.

Topic 2: Adding Enrollments

Introduction

This section discusses how to add enrollments in BM.

Change Date

August 30, 2024

II.3.F.2.a

Adding graduate enrollments

To add a new graduate enrollment, the user selects the Add new enrollment button to open the Add a new enrollment window.  The user then chooses Graduate or advanced professional from the Training type dropdown menu and selects the Continue button to advance to the Add graduate or advanced professional enrollment window.  Users also can use the Cancel or X button to close the window without taking any actions.

In the Add graduate or advanced professional enrollment window, users enter the following information.

  1. Begin date:  This field is required.
    1. Must be in MM/DD/YYYY format.
    2. The user can either manually type in the date or select the appropriate date from the calendar icon.
    3. The date cannot be more than 180 days in the future.
  2. End date:  This field is required.
    1. Must be in MM/DD/YYYY format.
    2. The user can either manually type in the date or select the appropriate date from the calendar icon.
  3. Facility code:  This field is required.  Must be in X-X-XXXX-XX or XXXXXXXX format. The user can enter with or without hyphens.
  4. School name:  This field is not editable.  It is automatically filled in based on the entered Facility code.
  5. Objective:  This field is required.  The user selects one of the following options from the dropdown menu:
    1. Master of Arts,
    2. Master of Sciences,
    3. Master’s degree,
    4. Doctor of Philosophy,
    5. Doctor’s degree, or
    6. Postdoctoral.
  6. Program name:  This field is required.  The user selects one of the options from the dropdown menu that displays all Institution of Higher Learning (IHL) programs for that school.
    1. The user can start typing the program name from the enrollment certification to filter the programs listed.
    2. If the enrollment certification reports a non-matriculated term, the user can enter “non-matriculated” in this field to process the claim.
  7. Hours type:  This field is required and is automatically filled based on the entered Facility code.  The user can adjust the selection to the following options from the dropdown menu:
    1. Semester,
    2. Quarter, or
    3. Clock
  8. Undergraduate full-time hours:  This field is required and is automatically filled in based on the Facility code with the corresponding value in WEAMS.  The user may adjust this value, if appropriate.
  9. Enrolled hours:  This field is required.
    1. The user enters a positive value in hours and fraction of an hour in XX.XX format.
    2. This field must be greater than 00.00 hours.
    3. This field is automatically filled if the enrollment certification is received from Enrollment Manager (EM) with the sum of residence and distance credit hours.  The user may adjust this value, if appropriate.
  10. Tuition & fees:  This field is required, if provided. 
    1. This value must be between $0.00 and $99,999.99, inclusive.
    2. If no tuition & fees value is provided on the enrollment certification, the value is $0.00.
    3. Must be provided in the following situations:
      1. The Top-up checkbox is selected,
      2. The Incarcerated checkbox is selected,
      3. The training time is less than ½ time or ¼ time, or
      4. The beneficiary is on active duty during the term.
  11. Vacation period:  These fields are required only if a vacation period of 7 days or longer is certified.  The user enters the following information.
    1. Start date:  The user enters the start date of the vacation period in MM/DD/YYYY format.
    2. End date:  The user enters the end date of the vacation period in MM/DD/YYYY format.
    3. The user selects the Add another vacation period button if more than one vacation period is needed.
  12. Certified TT (training time):  This field is required for graduate enrollments.  The user selects one of the following options from the dropdown menu:
    1. Full-time,
    2. ¾ time,
    3. ½ time,
    4. Less than ½ time, or
    5. ¼ time.
  13. Top-up:  The user selects this checkbox if the certification is for a Top-up enrollment. See M22-4, Part II, 3.A.3.b for details on Top-up enrollment and kickers.
  14. Incarcerated:  The user selects this checkbox if the certification is for an incarcerated beneficiary.

The user selects the Add enrollment button to save the new enrollment.  Users also can use the Cancel or X button to close the Add graduate or advancement professional enrollment window without taking any actions.

Figure: Add graduate enrollment

  5. End date,

II.3.F.4.a

Amending graduate enrollments

To amend a previously entered graduate enrollment, the user selects the ellipses icon at the right of the enrollment and then selects Add amendment to open the Edit enrollment period window.

Note: If an existing amendment already exists but needs changed, the user can edit the correction by selecting the pencil icon to open the Edit enrollment period window.

The Edit enrollment period window contains a Current enrollment period information section that displays the current information for the given enrollment in a read-only format with the following information:

  1. Facility code and school name,
  2. Objective,
  3. Program name,
  4. Begin date,
  5. End date,
  6. Hours type,
  7. Enrolled hours,
  8. Undergrad FT hours,
  9. Vacation days, and
  10. Certified TT.

The following user enters the following information to add an amendment.

  1. Effective date: This field is required. 
    1. Must be in MM/DD/YYYY format.
    2. The user can either manually type in the date or select the appropriate date from the calendar icon.
  2. Amendment Reason:  This field is required.  The user selects one of the following options from the dropdown menu:
    1. Increase,
      1. Displays Revised enrolled hours, Revised certified training time, and Revised tuition & fees fields.
      2. Enrolled hours are increased on the effective date.
    2. Completed NP,
      1.  Displays Revised enrolled hours, Revised certified training time, Revised tuition & fees fields, mitigating circumstances (MITC), Restore entitlement, and Do not apply six-credit hour exclusion checkboxes.
      2. No adjustment to the enrolled hours with an effective date of the last day of the term.
      3. Six-credit hour exclusion does not apply.  MITCs can impact award depending on approval.
    3. Reduction pun,
      1. Displays Revised enrolled hours, Revised certified training time, Revised tuition & fees fields, MITC, Restore entitlement, and Do not apply six-credit hour exclusion checkboxes.
      2. Enrolled hours are reduced on the effective date.
    4. Reduction during drop,

II.3.F.5.a

Deleting an unauthorized enrollment

To delete an unauthorized enrollment, correction, or amendment, the user selects the trashcan icon in the corresponding line which opens a window with the question Are you sure you want to delete this ?

The user selects the Yes, delete button to delete the enrollment, correction, or amendment.  Users also can use the No, go back or X button to close the window without taking any actions.

Figure: Deleting an enrollment

II.3.F.5.b

Deleting an authorized amendment

Users cannot delete previously authorized enrollments or corrections.  If an enrollment or correction was authorized in error, the user must process an amendment of type Other with Revised hours of 0 and Revised tuition & fees of $0.00.

To delete an authorized amendment, the user selects the trashcan icon in the corresponding line which opens a Delete amendment window with the message “Deleting this item may adversely affect awards adjudication or impact benefits.  Would you like to continue?”

The user selects the Yes, delete button to delete the enrollment, correction, or amendment.  Users also can use the No, go back or X button to close the window without taking any actions.

Deleting an authorized amendment may result in impacts on previously authorized awards and/or entitlement charges.

Managing national exams

To add a national exam manual award, the user selects the Add manual award button to open the Manual award window.  The user then selects National exam from the dropdown menu and selects Continue to proceed to the Add national exam window.

The user then adds the following information:

  1. Facility code:  This field is required.  Must be in X-X-XXXX-XX or XXXXXXXX format. The user can enter with or without hyphens.
  2. Test date:  This field is required. 
    1. Must be in MM/DD/YYYY format.
    2. The user can either manually type in the date or select the appropriate date from the calendar icon.
    3. The date cannot be in the future.
  3. Amount paid:  This is a required field.  This value must be at least $0.01.
  4. Description:  This field is required if there are any notes needed.  The user may enter up to 200 characters.

Note: BM rules calculate the amount of entitlement used for a national exam manual award based on the amount paid.  No user action is needed.

The user selects the Add button to save the national exam manual award.  Users also can use the Cancel or X button to close the Add national exam window without taking any actions.

Figure: Adding a national exam manual award

Selecting the pencil icon next to a national exam manual award opens the Edit manual award window which allows the user to change any of the fields available in the Add national exam window.  The user selects the Save button to save the edits.  Users also can use the Cancel or X button to close the Edit manual award window without taking any actions.

Selecting the trashcan icon next to a user-added VA eligibility status opens a message reading, “Are you sure you want to delete this manual award?”  The user selects either Yes, delete or No, go back.

II.3.F.7.g

Managing OJT enrollments

To add an OJT manual award, the user selects the Add manual award button to open the Manual award window.  The user then selects On-the-job training from the dropdown menu and selects Continue to proceed to the Add OJT window.

The user then adds the following information from the appropriate job aid listed under attachments on this page:

  1. Facility code:  This field is required.  Must be in X-X-XXXX-XX or XXXXXXXX format. The user can enter with or without hyphens.
  2. Begin date:  This field is required. 
    1. Must be in MM/DD/YYYY format.
    2. The user can either manually type in the date or select the appropriate date from the calendar icon.
    3. The date cannot be in the future.
  3. End date:  This field is required. 
    1. Must be in MM/DD/YYYY format.
    2. The user can either manually type in the date or select the appropriate date from the calendar icon.
    3. The date cannot be in the future.
  4. Basic amount:  This is a required field.  This value must be at least $0.01.
  5. Basic entitlement charge:  This is a required field.  The user enters entitlement amount in MM-DD format.  The minimum entry is 00-00.
  6. Kicker amount:  This field is required only if the beneficiary is using a kicker.  This value must be at least $0.01.
  7. Kicker entitlement charge:  This is a required field if the beneficiary is using a kicker.  The user enters entitlement amount in MM-DD format.  The minimum entry is 00-00.
