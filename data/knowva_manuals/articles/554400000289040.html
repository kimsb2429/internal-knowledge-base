Topic 1: Prior used entitlement

Introduction

This section discusses the Prior used entitlement section of the Entitlement and kickers screen.

Change Date

August 29, 2024

II.3.E.1.a

General

The Prior used entitlement section of the Entitlements and kickers screen allows users to view and manage other EDU entitlement used under different chapters that could impact total entitlement.

The section shows the following information.

  1. Benefit:  This field indicates which other benefit was used.  Possible values include:
    1. Chapter 1606,
    2. Chapter 1607,
    3. Chapter 30,
    4. Chapter 32,
    5. Chapter 33,
    6. Chapter 34,
    7. Chapter 35, and
    8. Other.
  2. Used entitlement:  This field displays the amount of entitlement used for the given Benefit in MM-DD format.

Figure: Prior used entitlement example

II.3.E.1.b

Managing prior used entitlement

The user selects the Add entitlement button to open the Add entitlement window which allows the user to add the following information.

  1. Benefit:  This field is required.  The user selects one of the following options from the dropdown menu:
    1. Chapter 1606,
    2. Chapter 1607,
    3. Chapter 30,
    4. Chapter 32,
    5. Chapter 33,
    6. Chapter 34,
    7. Chapter 35, or
    8. Other.

Note:  If the beneficiary has used multiple other benefits, the user adds multiple lines of prior used entitlement.

Note: The benefit type of the claim being worked does not display as an option.

  1. Was 48 months exhausted before 10/1/2013?:  This checkbox displays if the Benefit selected is Chapter 35.  If the user selects the checkbox, the 81-month rule for chapter 35 will not be applied.
  2. Description:  This field displays and is required if the user selects a Benefit of Other.  The user must enter free text with between one (1) and 200 characters, inclusive.
  3. Used entitlement:  This field is required for all Prior used entitlement entered.  The user enters entitlement used in MM-DD format.  The minimum entry is 00-01.

The user selects the Add button to save the prior entitlement used.  Users also can use the Cancel or X button to close the Add entitlement window without taking any actions.

Figure: Add entitlement window

Selecting the pencil icon next to a line of entitlement opens the Edit entitlement window which allows the user to change any of the fields available in the Add entitlement window.  The user selects the Save button to save the edits.  Users also can use the Cancel or X button to close the Edit entitlement window without taking any actions.

Selecting the trashcan icon next to a line of entitlement opens a message reading, “Are you sure you want to delete this entitlement?”  The user selects either Yes, delete or No, go back.

Topic 2: Manual entitlement adjustments

Introduction

This section discusses the Manual entitlement adjustments section of the Entitlement and kickers screen.

Change Date

October 18, 2024

II.3.E.2.a

General

The Manual entitlement adjustments section of the Entitlement and kickers screen allows users to add or charge entitlement to a beneficiary’s file when appropriate.  Manual entitlement adjustments always adjust the entitlement used for the benefit type of the claim.

Figure: Manual entitlement adjustments example

II.3.E.2.b

Managing entitlement adjustments

The user selects the Adjust manual entitlement button to open the Adjust manual entitlement window which allows the user to enter the following information.

  1. Effective date:  This field is required.
    1. Must be in MM/DD/YYYY format.
    2. The user can either manually type in the date or select the appropriate date from the calendar icon.
    3. The date cannot be in the future.
  2. Adjustment type:  This field is required.  The user must select either the Add or Charge option.
  3. Entitlement type:  This field is required.  The user selects one of the following options from the dropdown menu:
    1. Basic entitlement,
    2. 1606 kicker entitlement,
    3. 30 kicker entitlement,
    4. 32 kicker entitlement, or
    5. TATU entitlement.
  4. Entitlement amount:  This is a required field. 
    1. The user enters entitlement amount in MM-DD format.  The minimum entry is 00-01.
    2. If an Adjustment type of Add was selected, the Entitlement amount is added to the beneficiary’s entitlement.
    3. If an Adjustment type of Charge was selected, the Entitlement amount is deducted from the beneficiary’s entitlement.
  5. Description:  This field is required.  The user enters free text with between one (1) and 200 characters, inclusive.

The user selects the Submit adjustment button to save the manual entitlement adjustment used.  Users also can use the Cancel or X button to close the Adjust manual entitlement window without taking any actions.

If more than one manual entitlement adjustment is needed, the user can add multiple lines.

Figure: Adjust manual entitlement window

Selecting the pencil icon next to an entitlement adjustment opens the Edit manual entitlement window which allows the user to change any of the fields available in the Adjust manual entitlement window.  The user selects the Save button to save the edits.  Users also can use the Cancel or X button to close the Edit manual entitlement window without taking any actions.

Selecting the trashcan icon next to an entitlement adjustment opens a message reading, “Are you sure you want to delete this manual adjustment?”  The user selects either Yes, delete or No, go back.

Topic 3: Kickers

Introduction

This topic discusses the Kickers section of the Entitlement and kickers screen.

Change Date

October 18, 2024

II.3.E.3.a

General

The Kickers section of the Entitlement and kickers screen allows users to view and manage chapter 30, chapter 32, and chapter 1606 kickers.   

Figure: Kickers section

Note: The kicker section does not display for chapter 35 beneficiaries

II.3.E.3.b

Chapter 30 kickers

The Chapter 30 subsection of the Kickers section displays the following information which can either be pulled from the VA/DoD Identity Repository (VADIR) or entered manually.  The following information displays:

  1. Kicker information
    1. Kicker code:  This field displays the two-character kicker code associated with the beneficiary’s kicker.  
  2. Entitlement information
    1. Original entitlement:  This field shows the original amount of kicker entitlement in MM-DD format.  This field is generated by rules and cannot be edited by the user.
    2. Entitlement used:  This field shows the amount of kicker entitlement used in MM-DD format.  This field is generated by rules and cannot be edited by the user.
    3. Kicker remaining:  This field shows the amount of kicker entitlement remaining in MM-DD format.  This field is generated by rules and cannot be edited by the user.
    4. Exhaust date:  This field shows the date of kicker exhaustion in MM/DD/YYYY format.  This field is generated by rules and cannot be edited by the user.
  3. Status
    1. Effective date:  This field shows the effective date of the kicker in MM/DD/YYYY format.  This field is generated by rules and cannot be edited by the user.
    2. Kicker status:  This field shows the status of the kicker.  This field is generated by rules and cannot be edited by the user.
    3. Kicker rate:  This field shows the monthly kicker rate in $XX.XX format.  This field is generated by rules and cannot be edited by the user.

Figure: Chapter 30 kicker example

To add a chapter 30 kicker, the user selects the Add chapter 30 kicker button to open the Add chapter 30 kicker window which allows the user to enter the following information.

  1. Kicker code:  This field is required.  The user selects one of the options from the dropdown menu.  See Appendix for a full list of possible selections.
  2. The user selects the Add button to continue adding chapter 30 kicker information.  Users also can use the Cancel or X button to close the Add chapter 30 kicker window without taking any actions.

Figure: Add chapter 30 kicker – step 1

  1. This kicker was used under other benefits:  The user selects this checkbox if the beneficiary has used their chapter 30 kicker under another benefit.
    1. When this checkbox is selected, a Benefit field displays with Chapter 33 populated.  The user cannot change this value.
    2. When this checkbox is selected, an Amount field displays and is required.  The user enters entitlement amount in MM-DD format.  The minimum entry is 00-01.

The user selects the Save button to save the chapter 30 kicker information.  Users also can use the Cancel or X button to close the Add chapter 30 kicker window without taking any actions.

Figure:  Add chapter 30 kicker – step 2

Once a chapter 30 kicker has been added, the Add chapter 30 kicker button no longer displays.  Users can edit a kicker to override information pulled from VADIR.

Selecting the pencil icon next to the Kicker code field opens the Edit chapter 30 kicker window which allows the user to change the kicker code.  The user selects the Save button to save the edits.  Users also can use the Cancel or X button to close the Edit chapter 30 kicker window without taking any actions.

Selecting the pencil icon next to the Entitlement information fields opens the Edit chapter 30 kicker entitlement information window which allows the user to change the amount of chapter 30 kicker entitlement used.  The user selects the Save button to save the edits.  Users also can use the Cancel or X button to close the Edit chapter 30 kicker entitlement information window without taking any actions.

Selecting the trashcan icon next to the Chapter 30 subsection opens a message reading, “Are you sure you want to delete this kicker?”  The user selects either Yes, delete or No, go back.

II.3.E.3.c

Chapter 32 kickers

When a beneficiary’s earliest Enter on Duty (EOD) date between January 1, 1977, and June 30, 1985, then a Chapter 32 subsection displays in the Kickers section of the Entitlement and kickers screen.

The Chapter 32 subsection of the Kickers section displays the following information which can either be pulled from the VADIR or entered manually.  The following information displays:

  1. Kicker information
    1. Kicker code:  This field displays the two-character kicker code associated with the beneficiary’s kicker.  Possible values include:
      1. 4A,
      2. 4C,
      3. 3A,
      4. 3B, or
      5. 3C.
    2. DOD contributions:  This field displays the amount of contributions made by the Department of Defense (DOD) towards the chapter 32 kicker.
  2. Entitlement information
    1. Original entitlement:  This field shows the original amount of kicker entitlement in MM-DD format.  This field is generated by rules and cannot be edited by the user.
    2. Entitlement used:  This field shows the amount of kicker entitlement used in MM-DD format.  This field is generated by rules and cannot be edited by the user.
    3. Kicker remaining:  This field shows the amount of kicker entitlement remaining in MM-DD format.  This field is generated by rules and cannot be edited by the user.
    4. Exhaust date:  This field shows the date of kicker exhaustion in MM/DD/YYYY format.  This field is generated by rules and cannot be edited by the user.
  3. Status
    1. Effective date:  This field shows the effective date of the kicker in MM/DD/YYYY format.  This field is generated by rules and cannot be edited by the user.
    2. Kicker status:  This field shows the status of the kicker.  This field is generated by rules and cannot be edited by the user.
    3. Kicker rate:  This field shows the monthly kicker rate in $XX.XX format.  This field is generated by rules and cannot be edited by the user.

Figure:  Chapter 32 kicker example

To add a chapter 32 kicker, the user selects the Add chapter 32 kicker button to open the Add chapter 32 kicker window which allows the user to enter the following information.

  1. DOD contributions:  This field is required.  The user enters a dollar amount between $0.01 and $40,000.00, inclusive.
  2. The user selects the Add button to save the chapter 32 kicker information.  Users also can use the Cancel or X button to close the Add chapter 32 kicker window without taking any actions.

Figure:  Add chapter 32 kicker

Once a chapter 32 kicker has been added, the Add chapter 32 kicker button no longer displays.

Selecting the pencil icon next to the DOD contributions field opens the Edit chapter 32 kicker window which allows the user to change the amount of DOD contributions.  The user selects the Save button to save the edits.  Users also can use the Cancel or X button to close the Edit chapter 32 kicker window without taking any actions.

Selecting the trashcan icon next to the Chapter 32 subsection opens a message reading, “Are you sure you want to delete this kicker?”  The user selects either Yes, delete or No, go back.

II.3.E.3.d

Chapter 1606 kickers

The Chapter 1606 subsection of the Kickers section displays the following information which can either be pulled from the VADIR or entered manually.  The following information displays:

  1. Entitlement information
    1. Original entitlement:  This field shows the original amount of kicker entitlement in MM-DD format.  This field is generated by rules and cannot be edited by the user.
    2. Entitlement used:  This field shows the amount of kicker entitlement used in MM-DD format.  This field is generated by rules and cannot be edited by the user.
    3. Kicker remaining:  This field shows the amount of kicker entitlement remaining in MM-DD format.  This field is generated by rules and cannot be edited by the user.
    4. Exhaust date:  This field shows the date of kicker exhaustion in MM/DD/YYYY format.  This field is generated by rules and cannot be edited by the user.
  2. Status
    1. Effective date:  This field shows the effective date of the kicker in MM/DD/YYYY format.  This field is generated by rules and cannot be edited by the user.
    2. Kicker status:  This field shows the status of the kicker.  This field is generated by rules and cannot be edited by the user.
    3. End date:  This field displays the end date of the kicker in MM/DD/YYYY format.
    4. Kicker rate:  This field shows the monthly kicker rate in $XX.XX format.

Figure: Chapter 1606 kicker example

To add a new chapter 1606 kicker, the user selects the Add chapter 1606 kicker button to open the Add chapter 1606 kicker window which allows the user to enter the following information.

  1. Kicker status:  This field is required.  The user selects either Eligible or Eligibility retained.
  2. Kicker rate:  This field is required.  The user selects one of the following options from the dropdown menu:
    1. $350,
    2. $200, or
    3. $100.
  3. Effective date:  This field is required. 
    1. Must be in MM/DD/YYYY format.
    2. The user can either manually type in the date or select the appropriate date from the calendar icon.
    3. The date cannot be in the future.
  4. The user selects the Add and edit entitlement button to continue adding chapter 1606 kicker information.  Users also can use the Cancel or X button to close the Add chapter 1606 kicker window without taking any actions.

Figure: Add chapter 1606 kicker – step 1

  1. This kicker was used under other benefits:  The user selects this checkbox if the beneficiary has used their chapter 1606 kicker previously.
    1. When this checkbox is selected, a Select Benefit field displays and is required.  The user selects one of the following options from the dropdown menu:
      1. Chapter 1606,
      2. Chapter 1607,
      3. Chapter 33, or
      4. Other.
        1. If Other is selected, a Description field displays and is required.  The data entry rules are as follows:
          1. The user enters free text with between 1 and 200 characters, inclusive.
      5. If the chapter 1606 kicker was used under more than one benefit, the user can enter multiple lines of chapter 1606 kicker entitlement information.
    2. When this checkbox is selected, an Amount field displays and is required.  The data entry rules are as follows:
      1. The user enters entitlement amount in MM-DD format.  The minimum entry is 00-01.

The user selects the Save button to save the chapter 1606 kicker information.  Users also can use the Cancel or X button to close the Add chapter 1606 kicker window without taking any actions.

Figure:  Add chapter 1606 kicker – step 2

Selecting the pencil icon next to the Entitlement information opens the Edit chapter 1606 kicker entitlement information window which allows the user to change the amount of chapter 1606 kicker entitlement used.  The user selects the Save button to save the edits.  Users also can use the Cancel or X button to close the Edit chapter 1606 kicker entitlement information window without taking any actions.

Selecting the pencil icon next to the Status opens the Edit chapter 1606 kicker status window which allows the user to update the kicker status. The user can edit the following information:

  1. Kicker status:  This field is required.  The user selects one of the following options from the dropdown menu:
    1. Eligible,
    2. Eligibility retained,
    3. Not eligible,
    4. Terminated,
    5. Suspended, or
    6. Reinstated.
  2. Kicker rate:  This field is required.  The user selects one of the following options from the dropdown menu:
    1. $350,
    2. $200, or
    3. $100.
  3. Effective date:  This field is required.
    1. Must be in MM/DD/YYYY format.
    2. The user can either manually type in the date or select the appropriate date from the calendar icon.
    3. The date cannot be in the future.
  4. This is a previous kicker status:  The user selects this checkbox if this kicker status has ended.
    1. If selected, an End date field displays and is required. 
      1. Must be in MM/DD/YYYY format.
      2. The user can either manually type in the date or select the appropriate date from the calendar icon.
      3. The date cannot be in the future.

The user selects the Save button to save the edits.  Users also can use the Cancel or X button to close the Edit chapter 1606 kicker status window without taking any actions.

Figure:  Edit chapter 1606 kicker status

Selecting the trashcan icon next to a chapter 1606 kicker opens a message reading, “Are you sure you want to delete this kicker?”  The user selects either Yes, delete or No, go back.

II.3.E.3.e

Navigation buttons

Selecting the Back button at the bottom of the Entitlement and kickers screen returns the user to the Eligibility screen for the corresponding benefit type. 

Selecting the Continue button at the bottom of the Entitlement and kickers screen advances the user to the M22-4, Part II, 3.F.

Selecting the Return to beneficiary information button at the bottom of the Entitlement and kickers screen returns the user to the Beneficiary information screen.

Selecting the Return to top button brings the user back to the top of the Entitlement and Kickers screen.

Selecting the Search beneficiaries buttons brings the user to the .search screen (M22-4, Part II, 3.A).
