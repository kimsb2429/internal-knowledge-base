Topic 1: Claim Summary Screen

Introduction

This section discusses the Claim summary screen of BM.

Change Date

September 5, 2024

II.3.G.1.a

General

The Claim summary screen of BM displays the following sections.

  1. Eligibility step,
  2. Entitlement and kickers step,
    1. For chapters 30 and 1606 only.
  3. Entitlement step,
    1. For chapter 35 only.
  4. Enrollments step,
  5. Awards summary,
  6. Letters, and
  7. Claim Authorization.

II.3.G.1.b

Error and warning messages

When certain circumstances affect a claim, various messages will display on the Claim summary screen under the claim banner.

  1. Authorization errors:  If a claim cannot be authorized, an authorization error message appears under the claim banner.  Once the error has been resolved, the message will no longer display and the claim can be authorized.
  1. Warning messages:  If certain circumstances affect a claim a warning message appears under the claim banner.  Warnings do not prevent claim authorization but serve to caution the user to validate information given the specific situation.

II.3.G.1.c

Payment address display

The most recent payment address, either direct deposit or physical address displays in a banner on the Claim summary screen.  This banner is for user information and convenience.  This provides a final opportunity to confirm any payments are released to the correct location before authorization.  If any changes are needed, the user navigates to the Beneficiary information screen to correct or update the payment address.

Topic 3: Awards Summary

Introduction

This section discusses the Awards summary section of the Claim summary screen.

Change Date

September 5, 2024

II.3.G.3.a

General

The Awards summary section of the Claim summary screen displays the information for both the Net awards generated in the current claim and the Lifetime awards for the beneficiary and benefit type.

Figure: Awards summary section

II.3.G.3.b

Net awards section

The Net awards section contains the following information for the business transactions (BTs) generated in the current claim.

  1. Award type: This field displays the type of award generated.  Possible values are:
    1. 1606 basic,
    2. 30 basic,
    3. 35 basic,
    4. 1606 kicker,
    5. 30 kicker, or
    6. 32 kicker.
  2. Manual award type:  This field displays the type of manual award generated.  Possible values are:
    1. Accelerated payment,
    2. Apprenticeship, (APP),
    3. Correspondence (COR),
    4. Flight (FLT),
    5. Licensing & certification, (LAC),
    6. National exam,
    7. On-the-job training (OJT),
    8. Tutorial assistance,
    9. Work study, or
    10. N/A (if not a manual award).
  3. Start date:  This field shows the start date of the payable period for the BT in MM/DD/YYYY format.
  4. End date:  This field shows the end date of the payable period for the BT in MM/DD/YYYY format.
  5. Amount:  This field shows the amount of the BT, either positive or negative.
    1. If the BT establishes a payment, the Amount will display as $0.00.
    2. If the BT establishes a debt, the Amount will display as ($0.00).
  6. Transaction code:  This field shows the code for the type of BT generated.  Only the code displays.  Possible values are as follows.

II.3.G.3.c

Lifetime awards section

The Lifetime awards section contains the following information for all BTs generated for a beneficiary.

  1. Award type: This field displays the type of award generated.  Possible values are:
    1. 1606 basic,
    2. 30 basic,
    3. 35 basic,
    4. 1606 kicker,
    5. 30 kicker, or
    6. 32 kicker.
  2. Manual award type:  This field displays the type of manual award generated.  Possible values are:
    1. Accelerated payment,
    2. Apprenticeship, (APP),
    3. Correspondence (COR),
    4. Flight (FLT),
    5. Licensing & certification, (LAC),
    6. National exam,
    7. On-the-job training (OJT),
    8. Tutorial assistance,
    9. Work study, or
    10. N/A (if not a manual award).
  3. Start date:  This field shows the start date of the payable period for the BT in MM/DD/YYYY format.
  4. End date:  This field shows the end date of the payable period for the BT in MM/DD/YYYY format.
  5. Amount:  This field shows the amount of the BT, either positive or negative.
    1. If the BT establishes a payment, the Amount will display as $0.00.
    2. If the BT establishes a debt, the Amount will display as ($0.00).
  6. Training time:  This field displays the certified training time value with the possible values of:
    1. Full-time,
    2. ¾ time,
    3. ½ time,
    4. Less than ½ time, or
    5. ¼ time.

Figure: Lifetime awards section

II.3.G.3.d

Enrollment verification information

This section will display the current Verified thru date for the beneficiary. 

This section also allows the user to update the Verified thru date within the claim workflow. To update the Verified thru date, select the pencil icon. In the “Edit verified through date” modal, the user may enter a new date or use the date picker by clicking the calendar icon. 

The date entered cannot be prior to the previous Verified thru date, must be greater than the current processing date, and must be equal to the end date of a prior month or end date of a past award period.

Topic 4: Authorization in Benefits Manager

Introduction

This section discusses how to authorize a claim in BM.

Change Date

October 18, 2024

II.3.G.4.a

General

In the Authorization section at the bottom of the Claim summary page, the user has several action options depending on the scenario.

  1. Selecting the Back button will return the user to M22-4, Part II, 3.F.
  2. Selecting the Submit for authorization button records the user as the adjudicator of the claim and exits the claim.  Users will use this button whenever a claim requires a second signature.
  3. Selecting the Authorize button authorizes the claim and releases any letters and matured payments associated with the claim.  If a second signature is required by the system, this button cannot be selected.
  4. Selecting the Exit the claim button exits the user from the claim.

II.3.G.4.b

Second signature requirements

The BM system is programmed to require second signatures for several situations.  In these situations, the adjudicator and the authorizer must be different users.

  1. Original eligibility decisions all require a second signature.
  2. First-time payments all require a second signature.
  3. Big payment scenarios require a second signature.  Big payment functionality activates in the following scenarios.
    1. For chapter 1606, if the total net award amount is equal to or greater than $8,000.00.
    2. For chapter 30, if the total net award amount is equal to or greater than $20,000.00.
    3. For chapter 35, if the total net award amount is equal to or greater than $20,000.00.
  4. Manual awards require a second signature, when included as part of the following scenarios:
    1. Original eligibility decision,
    2. First-time payment,
    3. Big payment,
    4. Accelerated payment,
    5. Correspondence (COR),
    6. Flight (FLT),
    7. Tutorial assistance, or
    8. Work study.

II.3.G.4.c

Procedural requirements

Some situations require a second signature but are not systemically programmed in BM.  The user must recognize when their claim meets one of these scenarios.  Examples include:

  1. Awards for claimants who are VA employees and relatives of VA employees,
  2. Awards resulting from administrative decisions,
  3. Change to eligibility,
  4. Awards involving incarcerated claimants,
  5. Awards involving entitlement exhaustion,
  6. Awards involving extensions of delimiting dates,
  7. Any time changes are made to VADIR data, and
  8. Any time changes are made to rules-calculated data.
