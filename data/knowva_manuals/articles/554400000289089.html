Topic 1: General Letters Information

Introduction

This topic provides general information about system-generated letters in BM.

Change Date

October 18, 2024

II.3.H.1.a

General

BM automatically generates the following letters:

  1. Chapter 30 decision,
  2. Chapter 1606 decision,
  3. Chapter 35 payment,
  4. Chapter 30 payment, and
  5. Chapter 1606 payment.

The initial release of BM does not include a Letter Information screen that allows the user to modify system generated letters.

If a Power of Attorney (POA) is listed on the Beneficiary information screen, letters produced for that beneficiary will generate with a carbon copy (cc) line for the POA. A copy of the letter will also be generated. When letters are automatically sent to the print vendor for mailing, the POA copy of the letter will be shipped to the authorizing Regional Processing Office (RPO) for distribution.

The Station indicated on the Biographical information screen will determine the return address displayed on the top of system generated letters.

If the beneficiary is under 18 years of age, their letters will generate using the information entered in the Parent/Guardian Information section of the Biographical information screen.

Current System Alert: In the initial release of BM, there are known defects and functionality gaps throughout the system. As a result, all letters should be reviewed and modified/corrected, as needed, until the system is further refined, and all bug-fixes can be implemented. This alert will be removed when the system has been updated.

II.3.H.1.b

Section overview

The Claim summary screen allows the user to suppress system-generated letters and upload modified letters. It also allows the user to generate a new decision letter on demand and create a blank letter template using the beneficiary’s information.

The Letters section is located near the bottom of the Claim summary screen. Before a claim is authorized, this section allows the user to view both PDF and Word versions of system generated letters.

The Letter source column of the table indicates whether a letter was generated by the system or uploaded by a user. System-generated letters can be suppressed individually by checking the Suppress letter checkbox.

After authorization, the section indicates whether a letter was sent or suppressed, and it displays PDF versions of the sent letters. BM stores letters, and acts as the system of record for the notice provided to the beneficiary.

If a letter requires editing, it must be manually uploaded. The system cannot save edits made to system-generated letters.

                                      Figure: Letters section

II.3.H.1.c

Manual upload

To upload a manually edited letter, the user selects the Upload letter button. Only PDF documents can be uploaded.

All uploaded letters should have the return address, claimant address, and VA logo in the correct location and in the proper format.  The system will reject improperly formatted letters, and an error message explaining the problem displays. For best results, users should only use the blank letter template provided by the system, and nothing except the body of the letter should be modified. If an address update is required, it should be done on the Biographical information screen and not manually manipulated.

Manually uploaded letters cannot be suppressed. They must be deleted using the trashcan icon to remove them from the system. All manually uploaded letters that are in the system will be sent to the print vendor upon authorization.

II.3.H.1.d

Generate new decision letter

If the claim being worked requires a decision letter, and one is not automatically generated, the user selects the Generate new decision letter button. This will force a decision letter to generate as a system-generated letter. It can then be suppressed if it requires editing and manual upload.

II.3.H.1.e

Blank letter template

Selecting the Download letter template(.doc) link will provide the user with a fully editable Word document with all required pre-formatted header and address information.

The template can be used to manually upload letters that are normally produced outside the system. The user should copy and paste the body of the letter without manipulating the pre-populated information.

This document contains both the “We Are Here to Help” enclosure and the VA Form 20-0998. If appeal rights are not needed for the letter being created, the form should be removed from the letter and the enclosures list.

Topic 2: Decision Letter

Introduction

This section provides information regarding system-generated decision letters.

Change Date

October 18, 2024

 

II.3.H.2.a

General

Current System Advisory: At this time, no chapter 35 decision letter has been implemented. Certificate of Eligibility (COE) and denial letters for that benefit must continue to be created using the Personal Computer-Generated Letters (PCGL) system. However, they can be sent through BM using the manual upload functionality discussed in Manual Upload. The addition of a chapter 35 decision letter is planned for a future release. This alert will be removed when this update has been made.

For chapters 30 and 1606, BM will generate decision letters. Generally, a decision letter is used to explain an eligibility and/or entitlement decision to a beneficiary. A decision may be positive, limited, or negative.

The decision letter is required upon initial eligibility determination and when any changes to eligibility or entitlement are made to the claim. The decision letter should be sent even if a payment letter is also generated and sent. If a decision letter is required and fails to automatically generate, the user should manually generate it by using the Generate New Decision Letter button.

A new decision letter will not generate if the only current VA status is No longer eligible. Reaching a delimiting date or exhausting entitlement is not a new decision and does not warrant new appeal rights.

The user should create a CO INF-15 letter using PCGL and the Blank Letter Template. Then it should be manually uploaded and sent through BM. However, if the user changed eligibility or entitlement, causing a change to the delimiting date or amount of original entitlement, a new decision letter should be generated manually.

II.3.H.2.b

Positive decisions

A positive decision letter is a COE. Generally, a COE will generate when the beneficiary is eligible, and no enrollment or amendment information is added to the claim.  

There is no current functionality to produce a specific COE for approved programs or a Bad Course COE for unapproved programs. If a specific COE is required, the user should suppress the system-generated letter and manually edit it to include the approved or unapproved program.

II.3.H.2.c

Limited eligibility

When a beneficiary’s only current VA Status is Suspended, a Limited COE will generate. This letter explains that the beneficiary is still eligible for benefits under the program, but they cannot currently be paid, based on the reason for the VA Status.

Current System Alert: BM was built with Delimiting date gap as a reason for the VA Status to be Suspended for chapter 1606. New instructions from the Department of Defense indicate that there is no valid scenario for the use of this status under that benefit. The status should not be used, and the Limited COE should not be generated for a delimiting date gap. A future release will remove the option from the system, and this alert will be removed when it has been updated.

II.3.H.2.d

Negative decisions

When a beneficiary’s current VA Status is Denied, a denial decision letter will generate. This letter should explain each reason why the beneficiary is denied.

If a decision letter is manually generated when the only VA status is No Longer Eligible, the denial decision letter will also generate. If eligibility or entitlement changes were made in the claim, the user must manually explain the changes that were made. The corresponding language for the denial reason should not need to be edited.

Topic 3: Payment Letter

Introduction

This topic provides information regarding system-generated payment letters.

Change Date

October 18, 2024

 

II.3.H.3.a

Payment letter

BM will generate a payment letter for chapters 30, 35, and 1606.

Generally, a payment letter will generate when net awards are generated in the current claim. This letter should also generate when new enrollment information is received, but net awards are not generated. When future scheduled net awards are canceled due to a change to eligibility or withdrawal before a term begins, the letter generates even if no debts resulted in the claim.

The payment letter consists of a short letter listing all payments and debts. Additionally, the letter includes a Detailed Payment Letter which appears as an enclosure containing the information regarding the beneficiary’s remaining entitlement, the enrollment information we received, relevant mitigating circumstance information, how to repay debts, and legal references required to submit fully informed appeals.

Manual awards (excluding work study) may be contained in the Payments and Debts section of the letter, but there is not enough information available in the system for the letter to explain them. When Manual awards are used, the user must ensure they are listed in the letter and explain the payment in the Evidence and Findings section of the Detailed Payment Letter.

NOTE: All Work Study letters are generated by the Work Study Management System (WSMS). If any Work Study manual award is entered into a claim, NO letters will generate for that claim. If a VCE is assisting Work Study, they must use one claim to enter only the Work Study award. They must perform any other actions that require a letter in an additional claim.

Current System Alerts:

  1. There is no “5-month free entitlement” language in the chapter 35 payment letter, so the VCE must continue using the existing PCGL letter for applicable claims. VCEs can use the Blank Letter Template to manually upload the letter to BM. This alert will be removed when the letter has been updated to include the 5-month free entitlement language.
  2. The Evidence and Findings section should only contain terms that are being added or amended during the current claim. There is a known issue causing previously processed terms to display in some scenarios. These terms must be removed by the user. This alert will be removed when the system has been updated to prevent unchanged terms from displaying in the letter.

For chapter 1606, the system correctly generates both the Denied – Terminated by DoD and the No Longer Eligible – Past Delimiting Date VA statuses with the same effective date when a beneficiary’s eligibility is terminated by DoD. This causes repetitive language to occur in the Benefit Eligibility section of the Detailed Payment Letter. The language regarding the delimiting date must be removed by the user. This alert will be removed when the system has been updated to prevent the repetitive language.
