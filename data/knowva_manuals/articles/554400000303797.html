Topic 6: Work Product History Screen

 

 

Introduction

This section discusses the Work Product History screen of DGI.

 

 

Change Date

June 27, 2025

 

 

II.2.D.6

General

The Work Product History screen allows the VCE to view and access any previously existing or in-progress work products.  It also allows the VCE to create a new work product.

The header of this screen displays the file number and name of the beneficiary being viewed.

If no work product has ever been created for this beneficiary, the Work Product History section displays “No work products have been created.” 

a.  If any work product has previously been created, the following information will be displayed for each work product.

(1)  Date Received field – displays the date the claim was received.

(2)  Work Product Source – displays the benefit type being used (either CH33 or VET TEC).

(3)  Work Product Type – displays the type of award.  Possible values include:

     •    Original award,

     •    Supplemental award,

     •    VET TEC original award, or

     •    VET TEC supplemental award

(4)  Claimant SSN

(5)  Description – describes the processing actions taken.

(6)  Owner – identifies the user who processed the claim.

(7)  Status – displays the status of the work product.  Possible values include:

    •    Authorized,

    •    Deleted, or

    •    In progress

(8)  Date Authorized – displays the date the WP was authorized.

(9)  Authorized By – identifies the user who authorized the claim.

 

Figure: Work Product history table examples 

b.  Possible actions to be taken include the following.

(1)  Continue – allows the VCE to continue processing an In Progress work product.

(2)  View – allows the VCE the view an Authorized or Deleted work product.

(3)  Edit – opens a window that allows the VCE to edit the information associated with an In Progress work product.  Authorized and Deleted work products cannot be edited. 

 

Figure : Edit a Work Product window

Reference: See Topic 8: Work Product Creation Screen (Self and TOE) and Topic 9: Work Product Creation Screen (Fry) for details on data requirements for the Edit window.

(4)  Delete Work Product(s) – when the checkbox next to an in-progress work product is selected, authorized users can use the Delete Work Product(s) button to delete the work product.  Authorized work products cannot be deleted.

(5)  Add WP 107 – This button creates a Colmery Section 107 work product.

Reference: See (D) 2020 08 April: Procedural Advisory: Colmery Section 107 Retroactive Enrollment Certifications for details on Colmery Section 107.

(6)  Add VET TEC Work Product – This button initiates the creation of a VET TEC work product.

(7)  Add Work Product – This button initiates the creation of a new chapter 33 work product by bringing the VCE to the Topic 7: Entitlement Source Screen.

 

Figure: Create Work Product buttons

Topic 7: Entitlement Source Screen

 

 

Introduction

This section discusses the Entitlement Source screen of DGI.

 

 

Change Date

June 27, 2025

 

 

II.2.D.7

General

On the Entitlement Source screen, the VCE selects the source for chapter 33 benefits. “Entitlement Source” refers to the entity from where the entitlement will be drawn:

•   Beneficiary’s own service (Self),

•   Fry Scholarship (Fry), or

•   Transfer of entitlement (TOE) 

Note: A TOE beneficiary’s biographical record must be created first before proceeding with a TOE claim. After this has occurred, a Veteran’s or Service member’s record must then be created and authorized to proceed with source selection for the TOE beneficiary.

 A beneficiary may have more than one entitlement source. However, DGI does not have the functionality to handle multiple entitlement sources. Therefore, only one can be selected at a time.  See M22-4, Part XII, 2 for additional details.

 Select the Select button next to the appropriate entitlement source to continue.  This will navigate the VCE to the Work Product Creation screen.

Topic 8: Work Product Creation Screen (Self and TOE)

 

 

Introduction

This section discusses the Work Product Creation screen for Self and TOE entitlement sources in DGI.

 

 

Change Date

June 27, 2025

 

 

II.2.D.8

General

On the Work Product Creation screen, the VCE enters descriptive data for the claim.

For Self and TOE entitlement sources, the VCE enters the following fields:

•   Date Received – this indicates the date application was received.

Important – this field is required to match the claim date for timeliness measures.  The VCE must edit this field if it is incorrect.

•   Description – this indicates the purpose for the award, such as payment for a school term or semester.  This can be indicated by inputting the document type being worked in the work product.

Also displayed is the Work Product Type. This is auto-populated based on beneficiary information:

•   Original Award – displays if this is beneficiary’s first award processed in DGI.

•   Supplemental Award – displays if this is a subsequent award processed in DGI.

Once the appropriate data has been entered, the VCE selects the Create Work Product button to complete work product creation.  Depending on the type of claim, the user will be navigated to either the Service Data Screen or the Work Product Summary Screen located in M22-4, Part II, 2.D.14.

If needed, the VCE can cancel this record by clicking the Cancel button.

 

Figure: Work Product Creation screen for Self and TOE sources

Topic 9: Work Product Creation Screen (Fry)

 

 

Introduction

This section discusses the Work Product Creation screen for the FRY entitlement source in DGI.

 

 

Change Date

June 27, 2025

 

 

II.2.D.9

General

On the Work Product Creation screen, the VCE enters descriptive data for the claim.

For FRY entitlement sources, the VCE enters the following fields in the Create a New Work Product section:

•   Date Received – this indicates the date application was received.

Important – this field is required to match the claim date for timeliness measures.  VCEs must edit this field if it is incorrect.

•   Description – this indicates the purpose for the award, such as payment for a school term or semester.  This can be indicated by inputting the document type being worked in the work product.

Also displayed is the Work Product Type. This is auto-populated based on beneficiary information:

•   Original Award – displays if this is beneficiary’s first award processed in DGI.

•   Supplemental Award – displays if this is a subsequent award processed in DGI.

The VCE must also enter the following fields in the Fry Scholarship section:

a.   Veteran SSN: This field is required.  VCE inputs the Veteran’s SSN which must be 9 numerical characters.

Note: This is a new field with the switch over to eMPWR.  Work products authorized before the conversion do not display this information.

b.   Relationship field: This field is required.  VCE selects either Spouse or Parent, as appropriate.

c.   Spouse Remarried Date field: This field displays only if a relationship of Spouse is selected and must be filled in, if applicable.

d.  Veteran First Name field:  This field is required.  VCE inputs the first name of the Veteran.

e.   Veteran Middle Name: This field is required if a middle name is on file for the beneficiary.  If the beneficiary has no middle name listed, this field may be left blank. 

f.   Veteran Last Name field: This field is required.  VCE inputs the last name of the Veteran.

g.   Veteran Date of Death: This field is required if a date of death is on file.  VCE inputs the date of death for the Veteran, if appropriate.

h.   Veteran Line of Duty: This field is required.  VCE indicates if the Veteran’s death was in the line of duty.

i.   Veteran Last Branch of Service: This field is required.  VCE selects the Veteran’s last branch of service.

Once the appropriate data has been entered, the VCE selects the Create Work Product button.

 If needed, the VCE can cancel this record by selecting the Cancel button.

 

Figure: Work Product Creation screen for FRY source

Topic 10: Service Data Screen

 

 

Introduction

The section discusses the Service Data screen of DGI.

 

 

Change Date

June 27, 2025

 

 

II.2.D.10.a

Banner

The Eligibility and Entitlement banner appears on the service data screen and all subsequent screens.  This banner is for information purposes only and cannot be edited.  It updates based on information input on the screens of the work product.

 This banner displays the following information.

(1)   File Number – displays the file number for the current beneficiary.

(2)   Icon – for some beneficiaries, an icon will appear next to the name indicating a special population.  Icons include:

•   Purple Heart,

•   STEM Scholarship, and

•   VET TEC.

(3)  Name – displays the beneficiary’s name.

(4)  6X Availability – displays either as “Available” or the date used.

(5)  Eligibility Date  – displays the first date the beneficiary is eligible to receive chapter 33 benefits based on their service or that of their sponsor.

(6)  Benefit Level – displays the beneficiary’s benefit level as calculated by rules.

(7)  Delimiting Date – displays the beneficiary’s delimiting date, if any.

(8)  Original Entitlement – displays the beneficiary’s original amount of chapter 33 entitlement.

(9)  Used Entitlement – displays the amount of chapter 33 entitlement used by the beneficiary.

(10) Remaining Entitlement – displays the amount of chapter 33 entitlement the  beneficiary has remaining.

Figure: DGI Eligibility and Entitlement Banner example

If the beneficiary has transferred out entitlement or has received transferred entitlement, the following additional information displays.

(11) Entitlement Transferred Out – displays for transferors only.  Shows the amount of entitlement transferred to dependents.

(12) TOE Family Used Entitlement – displays the amount of entitlement used by all members of a TOE family.

(13) TOE Family Remaining Entitlement – displays the amount of entitlement remaining for all members of a TOE family.

Figure: DGI Eligibility and Entitlement Banner – TOE example

References: See M22-4, Part IX, 7 for details on the six-credit hour exclusion and M22-4, Part XII, 7 for details on transfer of entitlement claims.

 

 

II.2.D.10.b

Service History section

This section of the service data screen is used to add, edit, or update service information, training, and excluded periods to establish eligibility and entitlement.

If a record is associated with VADIR, most service data will be automatically populated.  If there is no VADIR record, the VCE must manually enter the data.

Note: Claims impacted by the Rudisill v. McDonough decision may need periods of service populated into DGI adjusted to account for conditional discharges due to immediate re-enlistment.

The service history section contains displays the following information.

  • Enter of Duty – displays the start date of a period of active duty also known as the Entry on Duty (EOD) date.
  • Release from Active Duty – displays the end date of a period of active duty also known as the Release from Active Duty (RAD) date.
  • Branch of Service – displays the branch of service associated with the period of active duty.
  • RNG Activation Statute – displays the Reserve/National Guard (RNG) activation statute for Reserve or National Guard service.
  • PGIB Code – displays the code showing in the Eligibility Code field of the Reserve & National Guard Active Duty Periods table on the Education tab of VIS.
  • Project Code – displays the project code associated with the period of active duty.  This value can be found in the Project field of the Guard/Reserve Active Service (GRAS) table on the Military tab of VIS.
  • Statute Code – displays the statute code associated with the period of active duty.  This value can be found in the Statute field of the Guard/Reserve Active Service (GRAS) table on the Military tab of VIS.
  • Activation Statute – displays the Activation Statute for any entered service.
  • Character of Service – displays the character of service for the period of active duty.
  • Separation Reason – displays the separation reason for the period of active duty.
  • Creditable Service – indicates whether the period of service is creditable towards Chapter 33 or not.
  • Training – displays the number of days deducted for training.
  • Excluded – displays the number of days deducted due to periods excluded from Chapter 33 creditable service.
  • Lost Time – displays the number of days considered lost time.
  • Creditable Days – displays the number of days creditable to Chapter 33 entitlement.
  • Transaction Code – displays the three (3) digit reenlistment code associated with the period of service.

 

 

II.2.D.10.c

Adding and Editing Service section

To add service periods to a DGI record, the VCE selects the Add Service button which brings up the Add/Edit Service Period window.

 The VCE then enters the following information.

(1)  Enter of Duty – the EOD field is required for all service periods.  Enter in MM/DD/YYYY format.

(2)  Release from Active Duty – The RAD is not required for all service periods.  If one exists, it must be entered in MM/DD/YYYY format.  If the beneficiary is still on active duty at the time of processing, the RAD is left blank.

(3)  Branch of Service – The branch of service is required for all service periods.  Select the appropriate branch.

(4)  RNG Activation Statute – This field only displays if a Reserve or National Guard Branch of Service is selected.

(a)  Effective on August 20, 2022, this field is required for any added Guard/ Reserve Active Service (GRAS) periods.

(b)  Processor selects the appropriate option based on information in VIS or other evidence, as appropriate.

Note: GRAS service added prior to August 20, 2022, does not show a value for the RNG Activation Statute. 

(5)  PGIB Code – This field only displays if a Reserve or National Guard Branch of Service is selected.  Processor selects the appropriate option based on information in VIS or other evidence, as appropriate.

(6)  Project Code – This field only displays if a Reserve or National Guard Branch of Service is selected.  Processor enters the code found in the Project field of the Guard/Reserve Active Service (GRAS) table on the Military tab of VIS.

(7)  Statute Code – This field only displays if a Reserve or National Guard Branch of Service is selected.  Processor enters the code found in the Statute field of the Guard/Reserve Active Service (GRAS) table on the Military tab of VIS.

(8)  Separation Reason – The Separation Reason is required for all service periods.  If the beneficiary is still on active duty, the user selects Not Applicable.  Otherwise, the user selects the appropriate option based on information in VIS or other evidence, as appropriate.

(9)  Character of Service – The character of service is required for all service periods.  If the beneficiary is still on active duty, the user selects Not Applicable.  Otherwise, the user selects the appropriate option based on information in VIS or other evidence, as appropriate.

(10) Activation Statute – This field is not required.  It distinguishes between Title 10 and Title 32 service, if selected.

(a)  Title 10 or Title 32 is used for Reserve or Guard service, depending on the information in VIS.

(b)  N/A is kept for branches of the uniformed services other than Reserve and Guard.

(11) Non-Qualifying – Selecting this checkbox makes the service non-qualifying.

(12) Transaction Code – This field is not required but will be automatically filled in when the period of service is pulled in from VADIR and is honorable.

(a)  Processor enters the code found in the reenlistment section of VIS.

(b)  Possible honorable codes are the following:

  • 151 – Active duty (active strength) change, immediate reenlistment,
  • 251 – Active duty (reserve strength) change, immediate reenlistment,
  • 351 – Guard/Reserve Duty change, immediate reenlistment,
  • 381 – Guard/Reserve Duty gain, prior service, delayed reenlistment, or
  • 139 – Guard/Reserve, Gain or Loss.  A distinct break in service to become an officer.

(c)  If the reenlistment code is not one of the five (5) above, the transaction code will not display.

Note: See (G) 2022 14 July: Procedural Advisory: Excessive Service Periods for details on 20 or more periods of service.

Select the Save button to add the service period to the record.

To edit an existing period of service, select the Edit Service button associated with the period of service to bring up the Add/Edit Service Period window.  Follow the directions above to edit service.

Note: If the beneficiary is associated to VADIR and the VCE makes a change to service, the Override DoD Service Data message displays.  If the VCE confirms the warning, the Override DoD Service Data checkbox displaying above the Service History table will be checked.

To add a training deduction or excluded period, select the Add Training/Excluded button which opens the Add New Training/Excluded Period window and enter the following information.

  • Type – select either Training or Excluded, as appropriate.
  • Excluded Period Type – this field only displays if Excluded was selected as the Type.  VCE selects the appropriate type of exclusion.
  • Start Date – processor enters the start date of the training or excluded period.  This is a required field.
  • End Date – enters the end date of the training or excluded period.  This field is not required if training or exclusion is ongoing.

 

Figure: Add New Training Period

Figure: Add New Excluded Period

To edit a training or exclusion period, select the blue arrow at the right of the associated service and selects the Edit Period button which brings up the Edit Training/Excluded Period window.  Type cannot be edited in this window.

 To delete a service, training, or exclusion period, select the checkbox at the right of the appropriate period and select the Delete Service Data box.

 

Figure: Deleting Service or Training/Excluded Periods example

 

 

II.2.D.10.d

Purple Heart section

To add Purple Heart information, selects the Add PH button which opens the Add/Edit Purple Heart window and enter the following information.

  • Award Date – enter the date the Purple Heart was awarded in MM/DD/YYYY format.
  • Branch of Service – select the appropriate branch of service.

 

Figure: Purple Heart section

 

Figure: Add/Edit Purple Heart window 

Figure: Purple Heart Icon

To edit a Purple Heart entry, select the blue arrow at the right of the associated service and select the Edit PH button which brings up the Add/Edit Purple Heart window. 

To delete a Purple Heart, select the checkbox at the right of the appropriate entry and select the Delete PH box.

See (K) 2021 15 November: Procedural Advisory: DGIB – LTS Release DGI-1.1.

 

 

II.2.D.10.e

Comments section

Comments section – if any changes have been made to the Service Data screen, a comment is required to advance to the Entitlement and Kickers Screen as detailed in M22-4, Part II, 2.D.11.
