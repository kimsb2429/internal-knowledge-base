Topic 1: Searching for a beneficiary in BDN Historical Reporting

Introduction

This topic discusses searching for beneficiaries using the BDN Historical Reporting functionality in Benefits Manager (BM).

Change Date

June 27, 2025

II.3.B.1.a

Searching for a beneficiary

The BDN Historical Reporting screen is accessed using a tab at the top of the home screen in BM, which can be accessed via https://iam.education.va.gov/.

Figure: BDN Historical Reporting Search Screen

The user can begin the search by entering the following in the search field on the screen.

  1. File number/ Social Security Number
    1. When searching by File number (such as a “C number”), the user must enter the entire alphanumeric identifier as is in BDN. If the identifier begins a letter, as with VA file numbers, the leading letter must be capitalized.
    2. When searching by social security number (SSN), the user enters nine (9) numeric digits without dashes.

Note: “Wild card” searches are not available in BM.

Note: A chapter 35 record may be located by searching either the Veteran’s file number or the beneficiary’s SSN

If the user wishes to conduct a separate record search, they can either:

  1. Select the File number of SSN previously entered, delete, re-enter, and click Search, or
  2. The user can click Restart search, which erases the prior field entry, and enter a new File number or SSN.

II.3.B.1.b

Search Results

After selecting the Search button, a new section will appear.

  1. Search results allow the user to access record of the beneficiary’s past utilized education benefits by selecting the View Report button.

Figure: Search results example

  1. Multiple results may be presented. The beneficiary and/or their dependents may be displayed with separate and distinct payee codes. The user should select the record which matches their adjudicative need.

An error message will display if the search has more than 100 results requesting the user to refine the search further.

A message will be displayed if no results are returned.

Figure: Error message when no results are returned

An error message will appear if the user does not have the required sensitivity level to access the record.

Figure: Error message when user lacks sensitivity level to access the report

Upon clicking Ok the View Report button will disappear and be replaced with the message Do not have permission to view.

If system connections are down or intermittent, the following error message may be encountered.

Figure: Example of error message when system is down

Topic 2: Reviewing the Reports Generated by BDN Historical Reporting

Introduction

This topic discusses the different reports generated by BDN Historical Reporting.

Change Date

June 27, 2025

II.3.B.2.a

General

When the identified beneficiary has been successfully located and the user clicks View Report, the following options may be presented, based on the beneficiary’s past benefit usage.

  1. Chapter 35,
  2. Chapter 30,
  3. Chapter 1606,
  4. Chapter 1607,
  5. Chapter 32, or
  6. Chapter 34.

Figure: View of reports provided during claimant search

To return to the search screen of BDN Historical Reporting, the user must click the Back to search results button at the top.

Figure: Back to search results button example

II.3.B.2.b

Claimant Details Banner

Upon expanding any of the reports provided, a claimant details banner will be presented at the top (in blue), containing the following information:

  1. Name & Address,
    1. For a beneficiary with a domestic residence,
      1. Displayed as First name, Middle initial, Last name, Street Address/PO Box, City, State
    2. For a beneficiary with a foreign residence, and
      1. Displayed as First name, Middle initial, Last name, Street Address/PO Box, City, Province, Country.
    3. For a beneficiary with a military overseas residence.
      1. Displayed as First name, Middle initial, Last name, Street Address/PO Box, Military PO, and Geographic area of station.
  2. Zip code,
    1. For a beneficiary with a domestic residence,
      1. Displays the first five digits.
    2. For a beneficiary with a foreign residence,
      1. Displays up to five digits.
    3. For a beneficiary with a military overseas residence,
      1. Displays the first five digits.

Note: If an “FZ code” was used in BDN, only the succeeding 3 digits will be displayed, (i.e., “FZ953” in BDN will display as “953” in the report).

  1. Date of birth,
    1. Displayed in the format of MM/DD/YYYY.
  2. Date of death,
    1. If deceased, will be displayed in the format of MM/DD/YYYY, and
  3. Gender,
    1. Display will include either:
      1. M for Male, or
      2. F for Female.
  4. Station,
    1. Indicates the last RO/RPO to process the record, and
    2. Displays as RPOXXX.
  5. Branch of service,
    1. Active-duty branches, and
    2. Reserve/guard units.

Note: Field will NOT be utilized or displayed on Chapter 35 records.

  1. File number,
    1. If a non-SSN VA identifier is provided, this field will display all alphanumeric values, in capitalized form. For example, for a “C” type file, the user would see C12345678 without any dashes or separation,
    2. If the beneficiary is only identified by their SSN, their SSN will appear in this field, without dashes, and
    3. If the beneficiary does not have an SSN, then a system-generated identifier would be produced.
  2. SSN,
    1. If an SSN exists for the beneficiary, it will display as nine (9) numeric digits with dashes, and
    2. If no SSN exists for the beneficiary, the field will display a system-generated identifier.
  3. Date last paid,
    1. Represents the most recent “none” or no-pay date for the beneficiary, and
    2. Displays in the format of MM/DD/YYYY.
  4. Six-credit hour exclusion, and
    1. If previously utilized, the field will display USED, and
    2. If not previously utilized, the field will display UNUSED.
  5. Master record status.
    1. Displays the most recent Master record status from BDN as per Rescinded: M22-4, Part II, 9.28.

Note: If a field does not include a value to present user from BDN, the field will show N/A.

II.3.B.2.c

Chapter 35 Historical Reporting Display

When Chapter 35 information is requested, the following sections/fields will display in the provided report accordion:

  1. Entitlement history,
    1. Default view, and
      1. Original entitlement,
        1. Represents the beginning amount of entitlement for the beneficiary under the benefit, and
        2. Displays in the MM-DD format.
      2. Used entitlement,
        1. Represents the amount of entitlement utilized under the benefit, and
        2. Displays in the MM-DD format.
      3. Remaining entitlement,
        1. Represents the resulting amount of entitlement equivalent to original entitlement minus used entitlement, and
        2. Displays in the MM-DD format.
      4. Prior VA training type, and
        1. Represents the most recent, prior VA education benefit received, if applicable, and
        2. Displays in the format of X.
      5. Prior VA training amount.
        1. Represents the amount of entitlement utilized during prior education benefit usage, and
        2. Displays in MM-DD format.
    2. Expanded view.
      1. Objective-course, and
        1. Represents the objective and course codes for the beneficiary’s enrollment per Rescinded: M22-4, Part II, 9.31 - 9.33, and
        2. Displays in the XXX-XXX format.
      2. Fiduciary file location.
        1. Represents the location of the beneficiary’s fiduciary record, if applicable, and
        2. Displays in the XXX format.

Figure: Example of Ch35 entitlement history section

  1. Important dates and important information, and
    1. Delimiting date,
      1. Represents the first day after a claimant’s period of eligibility expires, and
      2. Displays in the MM/DD/YYYY format.
    2. Date last paid,
      1. Represents the most recent “none” or no-pay date for the beneficiary, and
      2. Displays in the format of MM/DD/YYYY.
    3. Date last certified, and
      1. Represents the day following the latest month, day, and year attendance was verified, and
      2. Displays in the MM/DD/YYYY format.
    4. 89C indicator.
      1. This field indicates if an interest-bearing receivable existed/s in the Centralized Accounts Receivable (CARS) system, which would be collected by EDU prior to any station-created overpayments. Displayed values may include:
        1.  0-9 = Yes, an interest-bearing receivable exists, or
        2. Null = No, an interest-bearing receivable does not exist.

Figure: Example of Ch35 important dates and information section

  1. Awards history.
    1. Default view, and
      1. Begin date,
        1. Represents the start date for a payable period, and
        2. Displays in the MM/DD/YYYY format.
      2. Begin reason,
        1. Represents the reason code associated with the begin date, per Rescinded: M22-4, Part II, 9.14, and
        2. Displays in the format of XX.
      3. No pay date,
        1. Represents the first day after a payable period, and
        2. Displays in the MM/DD/YYYY format.
      4. No pay reason,
        1. Represents the reason code associated with the no pay date, per Rescinded: M22-4, Part II,  9.14, and
        2. Displays in the format of XX.
      5. Rate, and
        1. Represents the monthly calculated amount payable for a period of training, and
        2. Displays in $XX.XX format.
      6. Training time.
        1. Represents the training time code, per Rescinded: M22-4, Part II, 9.23, and
        2. Displays in the format of X.
    2. Expanded view.
      1. Hours,
        1. Represents the number of hours in which the beneficiary is enrolled for that period of training, and
        2. Displayed in the format of XX.
      2. Entitlement charge,
        1. Indicates the rate at which entitlement is charged under the benefit, and
        2. Displays in the X format.
      3. Training type,
      4. Facility code, and
        1. Identifies the training facility at which the training occurred, and
        2. Displays in the format X-X-XXXX-XX.

        2. Displays in the $XX.XX format.

II.3.B.2.g

Chapter 32 Historical Reporting Display

When Chapter 32 information is requested, the following sections/fields will display in the provided report accordion:

  1. Entitlement history,
    1. Default view, and
      1. Original entitlement,
        1. Represents the beginning amount of entitlement for the beneficiary under the benefit, and
        2. Displays in the MM-DD format.
      2. Used entitlement,
        1. Represents the amount of entitlement utilized under the benefit, and
        2. Displays in the MM-DD format.
      3. Remaining entitlement,
        1. Represents the resulting amount of entitlement equivalent to original entitlement minus used entitlement, and
        2. Displays in the MM-DD format.
      4. Prior VA training type, and
        1. Represents the most recent, prior VA education benefit received, if applicable, and
        2. Displays in the format of X.
      5. Dropped entitlement.
        1. Represents entitlement used for award periods not represented in the master record due to system space/record limitations, and
        2. Displays in the MM-DD format.
    2. Expanded view.
      1. Objective-course, and
        1. Represents the objective and course codes for the beneficiary’s enrollment per Rescinded: M22-4, Part II, 9.31 - 9.33, and
        2. Displays in the XXX-XXX format.
      2. Fiduciary file location.
        1. Represents the location of the beneficiary’s fiduciary record, if applicable, and
        2. Displays in the XXX format.

Figure: Example of Ch32 entitlement history section

  1. Important dates and important information,
    1. Delimiting date,
      1. Represents the first day after a claimant’s period of eligibility expires, and
      2. Displays in the MM/DD/YYYY format.
    2. Date last paid,
      1. Represents the most recent “none” or no-pay date for the beneficiary, and
      2. Displays in the format of MM/DD/YYYY.
    3. Date last certified,
      1. Represents the day following the latest month, day, and year attendance was verified, and
      2. Displays in the MM/DD/YYYY format.
    4. Enter on-duty date,
      1. Represents the beneficiary’s first day of active-duty service, and
      2. Displays in the MM/DD/YYYY format.
    5. Release from active-duty date, and
      1. Represents the beneficiary’s last day of active-duty service, and
      2. Displays in the MM/DD/YYYY format.
    6. Disallowance reason 1.
      1. Represents the first reason for the disallowance of benefits under this chapter, and
      2. Displays in the format of XX.

Figure: Example of Ch32 important dates and information section

  1. Contribution history, and
    1. Participant,
      1. Represents the action taken or status of contributions made by the beneficiary, and
      2. Displays in the format of X-Action/Status.
    2. Amount,
      1. Represents the dollar value of the action/status of the DoD’s contributions, and
      2. Displays in the format of $XX.XX.
    3. DoD, and
      1. Represents the action taken or status of contributions made by the beneficiary, and
      2. Displays in the format of X-Action/Status.
    4. Amount.
      1. Represents the dollar value of the action/status of the DoD’s contributions, and
      2. Displays in the format of $XX.XX.

Figure: Example of Ch32 contribution history section

  1. Allotment history.
    1. Year, and
      1. Represents the year in which contributions were made by the beneficiary into the benefit, and
      2. Displays in the format XXXX.
    2. Months of Year(a) Values.
      1. Represents the total contributions made by the beneficiary in a given month, and
      2. Displays in the $XX.XX format.

Figure: Example of Ch32 allotment history section

Note: If a field does not include a value to present user from BDN, the field will show N/A.
