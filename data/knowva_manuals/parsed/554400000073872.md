# Sub-Chapter 2. Claimant Biography - LTS Data Entry Rules

**Sub-Chapter 2. Claimant Biography - LTS Data Entry Rules**

This section describes data entry rules for the Claimant Biography screen in the Long Term Solution (LTS) system. These rules will help you with the following sections of the Claimant Biography screen:

**2.01 Claimant**

---

**a. General**

The Claimant Biography screen will be auto-filled if a claimant s record is in VADIR (VA/DoD Identity Repository) . If there is no record in VADIR, you will need to manually enter the claimant s biographical information. When creating a new chapter 33 record using the Claimant Biography screen, all fields with an asterisk are required. Pre-populated fields such as social security number cannot be edited.

![LTS Chapter 33 Claimant Bio entry](https://www.knowva.ebenefits.va.gov/img/edu/XII62_Claimant_Biography_LTS_Entry_Rules_Img1.jpg) 

**b. Associate (VADIR)**

Users with the VCE Role will have the ability to associate LTS with record from VADIR. This functionality may be used to allow for the population of information such as Date of Birth, Gender, and Service Data.

Once you click on the  Associate button (on the bottom right of the screen), a pop-up window will be displayed requesting you to verify your decision. The choices are to **Cancel** or **Continue.**

 

**c. SSN (Social Security Number)**

The social security number should be auto-filled on the Search screen.

 

**d. Station ID**

User information that is auto-filled from the Login screen.

 

**e. File Number**

The file number is a required field. You must begin it with eight digits or re-enter the SSN. If there is only a file number you may enter that only and then LTS will populate a  C in front of it.

 

**f. Power of Attorney (POA) Code**

The POA field is used if the claimant has a power of attorney form on file or if one is indicated in BDN. The POA codes are the same as those used for Benefits Delivery Network (BDN) [Hyperlink to BIRLS system] claims processing (including 99 or 66).

The POA field is not required. The user can key in the name of a POA code or may select a code from the dropdown.

The POA Name is required if you select either code 66 (Attorney-Exclusive Contact Requested) or 99 (Attorney-Exclusive Contact Not Requested).

 

**g. Six Credit Hour Exclusion (6X) Date**

If the **Six-Credit Hour Exclusion (6X)** was previously granted outside of the LTS, enter that date in the 6X Date field. For example, if the claimant has used the Six-Credit Hour Exclusion (6X) in BDN on a M21 screen for Chapters 30, 1606, or 1607, then input the date the exclusion was used.

**![6 hours exclusion entry](https://www.knowva.ebenefits.va.gov/img/edu/XII62_Claimant_Biography_LTS_Entry_Rules_Img2.png)**

**NOTE:** This will stop the LTS from applying 6X a second time. The 6X Date field may also be modified in a supplemental work product, which may affect lifetime award calculations. A Warning Message will be displayed below the screen header if you modify the 6X date.

**h. Sensitivity**

The Sensitivity field indicates the claimant s level of sensitivity. For example, a senator or high-ranking military officer may have a high level of sensitivity, indicating his/her record may need to be accessed by a claims processor with approved clearance. 

**i. The Fry (Parent LOD) Date**

The Fry (Parent LOD) Date is a field that is auto-filled during a **Fry** claim. It will be discussed at length in the Fry Scholarship Claim section.

**2.02 Personal**

---

In the Personal section of the Claimant Biography screen, the first, middle, and last name; gender; and date of birth are addressed.

 

**a. First Name**

This is a required field. The data entry rules are as follows:

* Must enter 1 20 characters
* The field allows characters Aa Zz, a space, and an apostrophe.
* First letter is automatically capitalized
* When only one character is entered, it will only accept uppercase A Z

 

**b. Middle Name**

Middle Name is not a required field. Data entry rules are as follows:

* Field will accept up to 20 characters.
* The field allows characters Aa Zz, a space, and an apostrophe.
* First letter is automatically capitalized
* When only one character is entered, it will only accept upper case A Z

 

**c. Last Name**

Last Name is a required field. The data entry rules are:

* Must enter 2 to 26 characters.
* The field allows characters Aa Zz, a space, and an apostrophe.
* First letter is automatically capitalized

Rules for adding a **suffix**:

* Suffix is not a required field, but will accept up to 4 characters.
* Allows characters lower and uppercase Aa Zz, numbers from 0 to 9, and also accepts a period

*For example,* Jr., Sr., II, or III

 

**d. Gender**

The Gender section is not a required field. It is a dropdown list that has two options:

* Male
* Female

You cannot edit this field if the record was pulled from VADIR.

 

**e. Date of Birth**

Date of Birth is a required and automatically formatted field (MM/DD/YYYY).

You cannot edit this field if the record was pulled from VADIR.

You can modify the date of birth for a non-associated record in a supplemental work product. Doing so may affect lifetime award calculations. A warning message will be displayed below the screen header if you modify the date of birth. 

**![LTS Chapter 33 Name amd Birthday entry](https://www.knowva.ebenefits.va.gov/img/edu/XII62_Claimant_Biography_LTS_Entry_Rules_Img3.jpg)**

**2.03 Contact Information**

---

The Contact Information section contains the required address, telephone, and email data.

**a. Phone Number**

Phone Number is not a required field but will accept up to 20 digits.

 

**b. Location**

Location is a required field. It is a dropdown box with the following selections:

* Domestic (default)
* Foreign
* Military Overseas

If you select Domestic, City is a required field with a minimum of two characters and a maximum of 20 characters.

If you choose Foreign, the City and State fields will change to the following:

* The City field is changed to State/County/Province.
* The State field is changed to Country.

If the country is not listed in the dropdown box, select ZZ. A required  Other box will appear that will accept 2 to 50 characters.

If Military Overseas is selected from Location, two dropdown boxes appear:

* APO/FPO
* AE/AA/AP

The State field is blank.

 

**c. Email**

An e-mail address is not required, but the field will accept up to 50 characters and must contain the @ symbol.

 

**d. Address**

Address is a required field. It will accept a minimum of 3 characters and a maximum of 40.

Line 2 of the Address field is not required. (See the attached RPO Letter 22-12-16 and USPS Postal Addressing Standards for using multiple address lines.)

Select a **State** from the dropdown box. This will appear only if  Domestic is kept as the default in the Location field above.

**Zip Code** is a required field used to support addresses within the United States, international addresses, and PO/DPO codes.

When the **Location** is **Domestic** or **Military Overseas,** the field accepts a five-digit code or five digits hyphen-four digits (e.g.: 22201-3406).

**![LTS Chapter 33 Contact Information entry](https://www.knowva.ebenefits.va.gov/img/edu/XII62_Claimant_Biography_LTS_Entry_Rules_Img4.jpg)**

**2.04 Comments**

---

The comments box is not required. However, if you try to leave the screen without comments but get an error message stating you must enter comments, this is a bug. You must enter something in order to move on, even if it is only a blank space.

**![LTS Chapter 33 Comment entry](https://www.knowva.ebenefits.va.gov/img/edu/XII62_Claimant_Biography_LTS_Entry_Rules_Img5.jpg)**

**2.05 Disassociate (VADIR)**

---

Users with the Senior VCE Role will have the ability to disassociate the LTS record from the VADIR record, if required. This functionality may be used to allow for the correction of erroneous information such as Date of Birth, Gender, and Service Data.

Once you click on the Disassociate button (on the bottom right of the screen), a pop-up window will be displayed requesting you to verify your decision. The choices are to **Cancel** or **Continue.**

When the LTS record is no longer associated with the **VADIR** record, the **Date of Birth and Gender** fields can now be edited.

 

**2.06 Disassociate (VADIR) VA-DoD Workaround**

---

On October 19, 2015, the Long Term Solution (LTS) Release 6.5.2 was deployed into the production environment. Afterwards, users reported a technical error when trying to create a work product on certain client’s record.

The issue was traced to VADIR and the version that was previously used on the last work product created was not the current version used.

A fix to this issue is being worked by OIT and will be implemented into the LTS once completed. Until the fix is deployed, use the below procedures to process these cases.

**Procedures:**

Once a claim has been identified:

1. A Senior Veteran Claims Examiner (SVCE) should  disassociate the claim from VADIR on the BIO screen in the LTS:

 

Associated claim example:

**![VADIR Search](https://www.knowva.ebenefits.va.gov/img/edu/XII62_Claimant_Biography_LTS_Entry_Rules_Img6.png)**

1. Navigate to the bottom of the BIO page and select the “DISASSOCIATE” button.

 

**![LTS Chapter 33 Disassociate ](https://www.knowva.ebenefits.va.gov/img/edu/XII62_Claimant_Biography_LTS_Entry_Rules_Img7.jpg)**

1. Select Continue:

 

**![LTS Chapter 33 Disassociate conformation screen](https://www.knowva.ebenefits.va.gov/img/edu/XII62_Claimant_Biography_LTS_Entry_Rules_Img8.jpg)**

1. Verify the record is now disassociated on BIO screen.

**![VADIR display](https://www.knowva.ebenefits.va.gov/img/edu/XII62_Claimant_Biography_LTS_Entry_Rules_Img9.png)**

1. Proceed with creating a Work Product and processing of the claim.

 

**Note:** If processing a Transfer of Entitlement (TOE) claim, this process will need to be completed on the dependent and the veteran.

Central Office (CO) is requesting the Regional Processing Offices (RPO’s) to track each instance of this procedure, so when the fix is implemented by OI&T, the effected cases can be re- associated to VADIR.
