**SUBCHAPTER 1. CHAPTER 33 MASTER RECORDS**

**9.01 CREATING A CHAPTER 33 MASTER RECORD**

The following entries are required on the READY screen to access the 309 screen, which will create a Chapter 33 master record:

* Command: CADJ (Claims Adjudication)
* Password
* Screen Number
* File Number
* Payee Number
* Benefit (ch33)
* Stub Name in X X XXXXX format
* Regional Office 3 digit code

A Chapter 33 master record is created only when a 309 screen is completed. The master record will be created or updated by entering UPD (update) in the NEXT SCREEN field.

**9.02 ACCESSING THE MASTER RECORD SCREENS.**All Chapter 33 master record screens can be accessed using the CADJ, Correction (CORR), Master Record Inquiry (MINQ), and Change Number (CNUM) commands. In addition, any screen within the Chapter 33 master record series can be accessed through direct input of the screen number on the Ready screen or through an entry in the NEXT SCREEN field on the M01, M21, M22, M23, M24, or 601 screens. All screens can be accessed for inquiry by the Pending Inquiry (PINQ) if there is a pending issue for the record, and Status Inquiry (SINQ) commands, if there is no pending issue for the record.

Using MINQ command, the operator should enter the following on the Ready screen to access any screen (M21 will be the default screen if none is provided in screen number):

(1) Command: MINQ

(2) Operator's password

(3) File number

(4) Benefit: CH33

**9.11 Chapter 33 Accounts receivable (F25 SCREEN)**

Use the FIST command and the F25 screen to adjust account receivables. These transactions establish, decrease, waive, or write off an account receivable or establish or change the amount of a deduction in regards to a debt. As a reminder, the FIST screen must be captured into TIMS.

File Number  
Payee Number

**9.14 Full File Pass (FFP) Housing**

a. General. A Chapter 33 FFP run will be executed on a monthly basis to generate the recurring Housing Allowance payments as well as perform some routine maintenance. Online processing will pay current month if transaction date is after the 20th of the current month. The FFP job will be scheduled to coincide with the payments being paid by Treasury on the 1st day of the following month. FFP does not generate any

Finance letters because all correspondence related to collections is done by Debt Management Center.

VCEs can view the FFP schedule on the Schedule of Operations through this link:

<https://vaww.vashare.vba.va.gov/sites/HIN-282Ops/ProdAnal/default.aspx>

**EXAMPLE 1:** A 06H housing allowance is entered on the 15th of a month. The beginning date of the award is in the prior month. The FFP for the current month will process around the 22nd day of the month and issue payment on the first day of the following month for the partial prior month and the current month.

**EXAMPLE 2:** A 06H housing allowance is entered on the 22nd of the current month. The beginning date of the award is in the prior month. BDN's online processing will issue an immediate payment for the preceding month.

b. Generating Housing Payments. FFP will pay through the end of the current month, and set DLP to the first of the next month. The monthly housing payments will be recurring and do not require monthly certifications.

c. Master Record Status. The master record type will change from "A" to "E" when a housing award terminates with a new DLP; otherwise the master record type will be "A" if an individual is receiving the monthly housing allowance.

d. Offsetting Receivables. FFP will offset receivables in the following order: 70B, 73B, 72B, 71B, 74B, 90C, 96C, 97C, 61C, 63C. If the housing award is terminating with the current month's run, all of the receivables will be taken, regardless of the collection indicator value except for 75B.

**9.16 GENERAL**

All Chapter 33 BDN fiscal information messages will be obtained through BDN using the FOES (Fiscal Output Exception System) command.  A FOES message is actually a Finance Central System Message.  For the purposes of this chapter, the phrase FOES message will still be used. FOES messages must be requested for each benefit separately (Chapter 30, Chapter 31, Chapter 1606, Chapter 1607, and Chapter 33).  The RPO employee requesting the FOES must be approved to use the BDN command.

Education Officers will designate in writing those individuals who are authorized to review FOES reports for their respective RPOs.  A copy of the delegation will be retained in a locally maintained file marked FOES Reports for Big Pay.

A review of these procedures and requirements will be conducted during annual Appraisal and Assistance Visits.

FOES Reports will be run on a daily basis.  Designated representatives will initiate a review of each FOES report within 2 business days.  For each approved “big pay” FOES report, the designated representative will initial and date the FOES report indicating the payment is approved.  If the “big pay” award is not approved, the designated representative will annotate that “big pay” as disapproved, provide a reason, initial, and date the FOES report.  In cases where a “big pay” is disapproved, the designated representative will ensure the appropriate action(s) is done to correct any errors.

FOES reports for “big pays” will be retained in the same folder as the delegation memorandum and be appropriately stored in a secure location.  The RPO will retain the FOES reports for 3 fiscal years after which time they can be destroyed based on the Records Control System (RCS) schedule.  

a. Processing the FOES command will print all the pending FOES messages currently on the RPO’s file for the benefit selected.  Those messages will be available to be printed again throughout the day; however, they will clear from the queue during BDN’s overnight cleanup.

b. The BDN system can only store messages for three business days.  On the fourth business day, BDN will discard the oldest messages in the queue to make room for new messages. Therefore, the FOES messages should be printed each day.

c. If no FOES messages exist, a message will display on the READY screen when the FOES request is run.

**9.17 RESPONSIBILITY**. FOES messages must be reviewed by the RPO Education Division, the RPO Compliance and Liaison activity (C&L), the Finance activity of the RPO Support Services Division (SSD) and DMC. 

FOES messages automatically print once the FOES Command has been used.  Generally, one FOES print can have up to 3 different messages for 3 different claimants per sheet. 

The Responsible division will maintain the paper printed copies of all FOES messages per the RCS schedule.

**a. EDUCATION DIVISION.**  In most cases, the Education Division will make the initial review and then contact Finance in SSD or the C&L unit.  Follow local RPO procedures regarding how to make contact.  However, using a TIMSC claim token is encouraged whenever possible.  

**EXAMPLE 1:** A Yellow Ribbon payment is returned for a bad address (FOES message 3226). The RPO will contact the appropriate C&L division, who will review the record and update the address in WEAMS then notify the appropriate Finance division to reissue the payment.

**EXAMPLE 2:**  A Yellow Ribbon payment was released to the wrong school (FOES message 3231).  The Education Division corrected the error and a 75B debt was established.  A 06Y payment was reissued to the correct school.  When the payment to the wrong school returns, the Education Division will review the record then send a claim token to Finance to clear 75B debt.  Finance will review the case and take appropriate action.

**b. FINANCE.**  If any fiscal transactions are required, the Education Division or C&L unit will relay that information to Finance through TIMS claim tokens, and Finance will process the needed transactions.  Finance employees are responsible to review the record and to verify that the action(s) requested by the Education Division or C&L is proper and for the correct amount.  If a Finance employee identifies a FOES message that requires action that has not been requested by either the Education Division or C&L, the Finance employee will open a claim token and refer the unprocessed FOES message to Education Division for action. Refer to local RPO guidance regarding the name of the appropriate TIMS queue to route the claim token. Generally, C&L divisions will prefer an email regarding the FOES. Refer to local guidance regarding procedures.

**c. DMC.** DMC employees are responsible for reviewing Chapter 33 FOES messages that are under their jurisdiction, including any improperly routed FOES messages, which are forwarded to them from an RPO.  There may be instances when the DMC will need to work with RPO Finance employees to determine what actions need to be taken to clear the unassociated record.

**9.19 FOES MESSAGE CODES.**

There are other FOES Message Codes in addition to those regarding “big pay”. Refer to the below table for additional information and guidance.  All FOES messages should be initiated within 2 business days.

When a school payment is returned to VA through the Department of Treasury, the return check does not include any school identifying details that would allow BDN to automatically apply the payment to a school debt or create proceeds for the school.  As such, when a school payment is returned, the funds are automatically deposited to the Education Appropriation. The BDN master record is updated with a 21S transaction and SHARE (a Windows application used to access BDN) reflects the return payment amount, issue date, and cancellation date.  A corresponding FOES message is generated to the RPO of Jurisdiction over the claim.  The FOES message should be captured in the claimant’s TIMS file whenever possible.  Remember to ensure only one claimant’s information is on the FOES screen when captured.

When a school payment is returned, the RPO of jurisdiction is required to review the student’s record and determine whether the return payment should be applied to a corresponding school debt (75B).  If a school debt exists, a 08E transaction should be performed against the debt for the amount of the return payment.  If no corresponding school debt exists, the RPO should determine if a corresponding student debt exists for Tuition, Fees, or Yellow Ribbon (74B).  If a student debt exists, a 08E transaction should be performed against the 74B for the amount of the return payment.  If no debt exists, RPOs should review the award to determine if the school has submitted an amended enrollment certification that has not been processed.  When amended awards are processed, RPOs should review the record to determine if the school had previously returned funds to VA and reduce the debt accordingly.  **It is critical that the 08E be performed to reduce the debt to prevent a debt from being erroneously referred to the Treasury Offset Program (TOP).**

**9.20 FOES MESSAGES WHEN ADDRESS DOES NOT COMPLY WITH PAM STANDARDS.**

Chapter 33 BDN will issue payments using the Treasury’s Payment Automation Manager (PAM).  New FOES messages will be generated for records when the address does not comply with PAM standards.

The PAM standard Payment Request format requires what is known as “normalized” address, meaning the city, state, and country name must be sent in separate fields.  Since BDN does not contain separate fields for this information, special programming was completed to parse this data into the existing 6 X 20 address block to properly populate the PAM standard format.  There are conditions where the system is unable to parse this data.  In those instances, a FOES message will be issued to the station of jurisdiction instructing the station to correct the address.  The FOES message will not prevent the payment from being issued; however, ***it is critical the station take corrective action*** as soon as possible to prevent future payments from being misrouted.

The FOES message will read “ADDRESS REQUIRES REVIEW” followed by a bad address reason.  The following table lists the bad address reasons and the required action to correct the bad addresses:

 

|  |  |  |  |
| --- | --- | --- | --- |
| **Reason Number** | **Reason** | **Description** | **Action** |
| 1 | APO/FPO/DPO NOT FOUND | APO, FPO, & DPO addresses require 2 position state code (AE, AP, or AA), and should not include a country name or a FZ code | Correct APO/FPO/DPO address |
| 2 | FOREIGN COUNTRY CODE INVALID | 3 Position FZ code is invalid | Correct 3 position Foreign Zip Code |
| 3 | NAME BLANK | Address Line 1 or Name Line does not contain data | Populate Name Line (address line 1) |
| 4 | STATE CODE NOT FOUND | State code was not identifiable in the address. | Correct State Code -  must use 2 character state abbreviation |
| 5 | TOO FEW ADDRESS LINES | Address is missing critical elements.  This message will be generated when there are 2 or less address lines in the master record. | Add missing address elements.  Address must contain at least 3 lines, including the name line. |

**9.21 TRANSFERS OF DISBURSING AUTHORITY (FOES 3302).** When education benefits are withheld due to a C-type offset, a FOES 3302 Transfer of Disbursing Authority message is produced to notify the appropriate station that funds have been withheld to be applied elsewhere. In order for the funds to be available to the station for disbursement, Finance must notify ALAC to transfer the funds from the benefit appropriation to the station suspense account. 

C-type offsets are not debts.  They are transactions used to withhold benefits due from one benefit system to be applied to a debt or obligation in another system or outside of the BDN and VETSNET (Veterans Service Network) systems. 

It is important to note that 67C offsets are not allowed in the Chapter 33 program.

If a 61C offset is set up in a Chapter 33 record and benefits are withheld, the DMC will be notified through a FOES 3302 message the amount of benefits that have been withheld and are pending appropriate application.  Usually, 61C offsets in education benefits are made to apply funds to a debt in another benefit, such as the Compensation and Pension (C&P) system.  Finance must determine where the funds should be applied.

If the 61C funds are to be applied to a debt, Finance will process a 08B transaction in the appropriate benefit and/or refund any remaining over collection to the payee.

If a 63C offset is set up in a Chapter 33 record and benefits are withheld, the RPO of jurisdiction of that Chapter 33 record will be notified through a FOES 3302 message the amount of benefits that have been withheld and are pending appropriate application.  Normally, stations establish 63C offsets when the funds withheld must be applied to a debt outside of VBA’s systems or when the file number on the offset record does not match the file number of the record with the debt.

Finance will prepare an SF-1047, Public Voucher for Refunds, and submit it to ALAC to return funds to the payee.

**9.22 CLEARING UNASSOCIATED RECORDS.**  The DMC and Finance are responsible to clear all unassociated funds.

**a. Unassociated Funds from Payments Applied to Debts.**  Since most Chapter 33 returned payments either apply to debts or are returned to appropriation 36X0137, there should be no unassociated funds for any reason other than when payments are larger than the debts to which they are applied or if there is, a system error and funds from another benefit are processed into the Chapter 33 program.  RPO employees are not authorized to apply money to debts; therefore, DMC should receive all messages for unassociated funds that must be returned to the payee.

For valid overpayments to debts, the VCE should research the case to see which of the following actions is/are appropriate.  The unassociated funds can be divided and processed using more than one transaction.   
  
**EXAMPLE:**  Part of the unassociated funds can be applied to an existing debt, and the remainder can be refunded to the payee.  Finance must complete one or more of the appropriate action(s).

* Return the funds to the payee using an 06C transaction,
* Apply the funds to another debt using an 06E3,
* Return the funds to the station using a 06C.  Follow up with a second action to make final disposition of the funds, and/or
* Return the funds to appropriation 36X0137 using a 06E2.

**b. Invalid records or system errors.** Occasionally, a returned payment gets classified incorrectly; therefore, payments that are not Chapter 33 payments could “unassociate” in the Chapter 33 system. 

If the DMC gets the unassociated message, they must contact Finance at the RPO of jurisdiction, who will work with Education Division to determine what type of payment it is and what action should be taken. 

If the RPO gets the message, the Education Division must work with Finance to resolve the issue.
