| **II.2.A.1.a:**  **DGIB Managed Service** | The DGIB Managed Service makes it easier to integrate applications, manage them, and increase efficiencies. Previously, each application had a unique purpose and provided a specific service. Most applications were unable to work together and share data. During the development of the Managed Service, many existing legacy applications were replaced and updated with new applications to create the more integrated DGIB Managed Service platform. |
|  |  |
|  | --- |
|  |  |
| **II.2.A.1.b:**  **eMPWR** | eMPWR is a payment processing system operated by the Office of Financial Management (OFM).  Upon the integration of the DGI with eMPWR on July 8, 2024, all Chapter 33 business transactions (BTs) will be transmitted from DGI to eMPWR through a payment microservice that ensures that all required data is sent to eMPWR for payments to be successfully sent out.  “Business transaction” is the general term for any payment-related actions sent from the DGI to eMPWR, including any payments or receivables (also known as debts) created during processing.  The payment microservice holds all BTs until the reach their maturation date.  The BTs are not actually sent to eMPWR until maturation is reached. |
|  |  |
|  | --- |
|  |  |
| **II.2.A.1.c:**  **BDN to eMPWR Transition** | As part of the transition to the DGIB Managed Service, the Benefits Delivery Network (BDN) is being incrementally retired.  Starting on July 8, 2024, all Chapter 33 BTs are be transmitted to eMPWR.  BTs authorized before, but maturing after, June 26, 2024, were converted to eMPWR format as part of the release deployment process. Beneficiaries using the Montgomery GI Bill (MGIB or Chapter 30), Montgomery GI Bill - Selected Reserve (Chapter 1606), Survivors' and Dependents' Educational Assistance (DEA or Chapter 35), and other non-Chapter 33 benefits will have their BTs switched from BDN to eMPWR at a later date.  *Reference:* For more information on non-Chapter 33 payment processing, see Benefits Manager procedures (scheduled for release in December 2024). |
|  |  |
|  | --- |
| **II.2.A.1.d:**  **eMPWR and DGI Integration** | As of July 8, 2024, the DGI system integration with BDN was removed and replaced with eMPWR which handles the:     ·Transmission of Chapter 33 BTs to the Department of Treasury to issue payments,  ·Establishment of debts (receivables), and  ·Notification of BTs to the Debt Management Center (DMC) for debt management activities. |
|  |  |
|  | --- |
| **II.2.A.1.e:**  **Impacts to Processing Actions** | With the transition to eMPWR, there were impacts to processing actions taken by Veterans Claims Examiners (VCEs) before the transition.  These differences include the following actions.     ·All manual fiscal transactions (FIST/FAUT actions) needed for the correction or completion of a Chapter 33 record.  These transactions must be routed for review and processing by the Regional Processing Office’s (RPO’s) Support Services Division (SSD), unless otherwise indicated.  ·The Work Product Summary (WPS) and Payment Instruction (PI) screens of DGI have had significant updates, including authorization options, payment information display, and new Miscellaneous Awards.  ·Payment addresses for Chapter 33 beneficiaries (including direct deposit and physical addresses for paper checks) are now managed on the Claimant Biography (BIO) screen of DGI.  ·Putting a Chapter 33 file into suspense or removing such a suspense for accounts with proceeds on file or potential death claims are managed on the BIO screen of DGI.  ·No BDN Master Records are created for new Chapter 33 beneficiaries, nor are existing master records updated in BDN.  ·No BDN Pending Issue Files (PIFs), including diaries and pending End Products (EPs), can be created for Chapter 33.  ·Review of education benefit payments for Chapter 33 occur in eMPWR for payments issued after June 28, 2024.  ·Notification of payments and debts are managed by one consolidated “Payments Letter” for Chapter 33 beneficiaries.     *References:*See relevant Chapters of this Subpart for further details.
