| **II.2.E.1.a**  **General** | Users who review Education Benefit payments under the Chapter 33 for payments issued on or after July 8, 2024, will do so using the eMPWR user interface in SalesForce.  For details on logging on and accessing eMPWR in SalesForce, please review the Logging into the Application section of the [eMPWR User Guide](https://empwr.va.gov/help/#/user-guide).    Unless additional access is specifically needed for their position or role, individuals will be given access to the eMPWR system with an inquiry (read-only) role. Any fiscal transactions needed for the correction or completion of a Chapter 33 record must be routed for review and processing by the Regional Processing Office’s (RPO) Support Services Division (SSD).    Users can search for a participant (beneficiary) by SSN or file number.    After a successful participant search is conducted from the home screen, the participant record is displayed via Participant Record screen with tabs including Transaction History, Participant Details, Notes, and eFolder.      |  | | --- | |  | |  |  |      Figure: eMPWR Participant Search      |  | | --- | |  | |  |  |     Figure: eMPWR Participant Record screen    *Note*: Due to changes in how relationships are established for TOE beneficiaries, when payments need to be reviewed for a TOE beneficiary who was paid via both BDN and eMPWR, multiple File Number searches will be needed – one using the transferee’s file number (for payments made via BDN) and one using the transferor’s file number (for payments made via eMPWR). |

| **II.2.E.1.b**  **Transaction History Tab** | When processing a claim, responding to questions received via the telephone or written correspondence, Education employees require the ability to search the system to retrieve all transactions associated with the searched participant. The Transaction History tab contains the following information.     1.    **All Transactions**– displays all transaction history records, not just those transactions sent from Education service.  2.    **Deductions**– displays active Benefit Repayment and Offsets deduction records. The processor can change the default filter to display inactive deductions.  3.    **Payments**– displays all payments scheduled for release by treasury and the details of those payment records.  4.    **Receivables**– displays all active receivable records with a balance greater than zero. The processor can change the default filter to display inactive receivables.     After a successful participant search, the system redirects to the **Transaction History** tab on the **Participant Record** page, displaying the **All Transactions** default grid for Accountable Balances.       *Figure: eMPWR Transaction History – All Transactions example*        *Figure: eMPWR Transaction History – Payments example*       *Figure: eMPWR Transaction History – Receivables example*     From this view, the processor also can view the details of a transaction by clicking on the Transaction Id.  Details vary depending upon the transaction but can include the following.     1.    Account Details/Balance Summary (Receivables, Deductions, Accountable Balances)  a.    Status of the Transaction, including outstanding balances (if applicable)  2.    Payment Details (Payment transactions only)  3.    Notes (All transaction types)  a.      Any notes entered during transaction processing.  4.    Historical Transactions against the account (Receivables, Deductions, Accountable Balances)  a.      Any transactions made against the account.  5.    Associated Documents (Accountable Balances)  a.      If any documents were uploaded as part of establishing the account/transaction  6.    Payment Address (Payment transactions only)  7.    Details about payment address or EFT account.    The processor can select the gear icon to export the transaction details to Microsoft Excel CSV format.     |  | | --- | |  | |  |  |     Figure: eMPWR Transaction History – Export    Select the **Close** button to exit the window and return to the Transaction History grid. |

| **II.2.E.2.a**  **General** | While Veterans Claims Examiners (VCEs) can no longer establish (CEST) or adjust (CADJ) Chapter 33 Master Records in BDN upon the transition to eMPWR, VCEs still can view the master record and transaction history from before the conversion in BDN.  To do so, users will follow the processes found in [M22-4, Part XII, 9.02 - 9.06.](https://eg5813aprl-int-nlb.egainstz.cloud/system/ws/v11/ss/article/554400000076905?portalId=554400000001018&usertype=customer&\$lang=en-us&)    *Note*: All active account receivables have been migrated to eMPWR as part of the conversion.  As such, no information will display on the M01 Accounts Receivable screen.    *Note*: No transactions submitted on or after July 8, 2024, will display in BDN.  VCEs must review eMPWR to view these transactions. This means that both eMPWR and BDN will need to be reviewed if a beneficiary has activity both before and after July 8, 2024. |
