| **II.2.D.12.a**  **Enrollment Manager Inbox** | | The Enrollment Manager Inbox (formerly the VA Online Certification of Enrollment (VA-ONCE) Inbox) section is used to display enrollment and change in enrollment information submitted by the school via Enrollment Manager.  (1)  If there are no Enrollment Manager submissions for the claimant, the message “The claimant has no Enrollment Manager records at this time” will be displayed.  This image shows the Enrollment Manager Inbox with no pending records.  *Figure: Enrollment Manager Inbox – No records example*  (2)  If new information has been sent through Enrollment Manager, the submissions will be displayed in the Enrollment Manager Inbox.     *Figure: Enrollment Manager Inbox – Submissions displayed example*  (3)  The **Decline**, **Add**, and **Correct** buttons will become active depending on the specific circumstances when an Enrollment Manager submission is selected.  (4)  School information submitted on an electronic VA Form 22-1999 will be displayed with *Enrollment* as the **Type**.  (a)  If the begin date, end date, and facility code of the Enrollment Manager submission do not match an existing enrollment in the Enrollment Periods section, this is considered a new enrollment and the **Decline** and **Add** buttons become active.  •  Selecting the **Decline** button will prompt the VCE to confirm the decision.  If the submission is continued, the enrollment will be removed from the Enrollment Manager Inbox. Otherwise, choose **Cancel** to return to the Enrollment screen.  •  Selecting the **Add** button will launch the **Add Enrollment** window. The data entry fields will be auto-populated with the Enrollment Manager data, as well as the information from Web Enabled Approval Management System (WEAMS) and may be modified if necessary. If the **Accept**button is clicked, the information will be displayed in the Enrollment Periods section. Clicking the **Cancel** button returns the VCE to the Enrollment screen and keeps the enrollment in the Enrollment Manager Inbox.  (b)  If the begin date, end date, and facility code of the Enrollment Manager submission match an existing enrollment in the Enrollment Periods section (such as a correction to the tuition of fee charges), the **Decline** and **Correct** buttons become active  •  Selecting the **Correct** button will launch the **Correct Enrollment** window. The data entry fields will be auto-populated with the Enrollment Manager data and may be modified if necessary.  •  Selecting the **Accept** button results in the information being displayed in the enrollment periods section. Selecting the **Cancel** button returns the VCE to the enrollment screen and keeps the enrollment in the Enrollment Management Inbox.  (c)  If the school is correcting one or both of the term dates (the begin date or end date of the Enrollment Manager submission match an existing enrollment in the enrollment periods section), the processor must manually associate the Enrollment Manager submission by selecting on the appropriate original enrollment being amended from the enrollment periods section.   (5)  School information submitted on an electronic VA-Form 22-1999B will generally be displayed with *Amendment* as the **Type**.  If there is no amendment with the same effective date in the enrollment periods section, the **Decline** and **Add** buttons will become active.  •  Selecting the **Decline** button prompts the processor to confirm the decision. If the submission is continued, the enrollment will be removed from the Enrollment Manager Inbox. Otherwise, select **Cancel** to return to the Enrollment screen.  •  Selecting the **Add** button will open the **Add Amendment** window. The data entry fields will be auto-populated with the Enrollment Manager data, including attendance and mitigating circumstance information in the change profile section. It may be modified if necessary. Selecting the **Accept** button results in the information being displayed in the Enrollment Periods section. Selecting the **Cancel** button returns the VCE to the enrollment screen and keeps the enrollment in the Enrollment Manager Inbox  (6)  If there is an amendment with the same effective date in the enrollment periods section (because this is a corrected amendment), the **Decline** and **Correct** buttons will become active.  Selecting the **Correct** button launches the **Correct Amendment** window. The data entry fields will be auto-populated with the Enrollment Manager data, including attendance and mitigating circumstance information in the change profile section, and may be modified if necessary. Selecting the **Accept** button results in the information being displayed in the enrollment periods section. Selecting the **Cancel** button returns the VCE to the enrollment screen and keeps the enrollment in the Enrollment Manager Inbox.  (7)  If the school information submitted on an electronic VA Form 22-1999B that does not match the facility code, begin date, and end date of an original enrollment in the enrollment periods section will be displayed with *orphan amendment* as the **Type**.  The VCE must manually enter and/or associate the orphan amendment with an original enrollment to activate the **Add** or **Correct** buttons. The **Decline** button will already be active.  (8)  An increase in enrollment, a reduction in enrollment, or a correction to the residence or distance hours effective on the first day of the term will be displayed with *Amendment—First day of term* as the **Type**.    If the *Amendment—First day of term* is selected, this will activate the **Decline** and **Correct** buttons.  Selecting the **Correct** button opens the Correct Enrollment window. The data entry fields will be auto-populated with the Enrollment Manager data. Selecting the **Accept** button results in the information being displayed in the Enrollment Periods section. Selecting the **Cancel** button returns the VCE to the Enrollment screen and keeps the enrollment in the Enrollment Manager inbox.  (9)  Certain types of Enrollment Manager submissions will not be associated with enrollments or amendments in the Enrollment Periods section. The record can be associated to a record in the Enrollment Periods section by clicking on the row. This will highlight that enrollment or amendment and activate the **Correct** button.  All Enrollment Manager Inbox records must be processed for a claim to be authorized. | | | | |  |

(d)**Facility Code** – enter the 8-character (alphanumeric) code that corresponds with the institution the beneficiary is attending.  The facility code must match the codes found in the WEAMS.  (e)**Objective** – select the appropriate option.  Options are:  •  031 MA  •  032 MS  •  033 MD-NEC  •  041 DRPHILOSP  •  042 DR-NEC  •  043 PSTDR-NEC  (f)  **Program Code** – enter the 3-digit code associated with the enrollment and shown on the enrollment certification or in WEAMS. (h)**Hour Type** – select the appropriate option.  Options are:  •  Semester  •  Quarter  •  Clock (standard)  (i)  **FT Hours** – enter the FT hours as shown on the enrollment certification.  (j)  **Res Hrs** – enter the number of residence hours certified on the enrollment certification.  (k)  **Dist Hrs** – enter the number of distance hours certified on the enrollment certification.  *Note:*Total residence and distance hours must be greater than zero.  (l)  **Vacation Days** – this field should not be used for grad enrollments.  (m)**Tuition Charged** – enter the amount of the tuition reported on the enrollment certification.  (n)  **Fees Charged** – enter the amount of the fees reported on the enrollment certification.  (o)  **+ Vacation Period** – this field should not be used for grad enrollments.  (p)  **Yellow Ribbon Amount** – enter the amount of the Yellow Ribbon contribution reported on the enrollment certification.  (q)  **Out of State Tuition and Fees** – enter the amount of the out of state tuition and fees reported on the enrollment certification.  (r)  **Interval Payments** – effective August 1, 2011, interval payments are no longer permitted.  Default value ofNo Restrictions is maintained for all enrollments. Select **Save** to add Enrollment.  This image shows the Add Graduate or Advanced Professional window.
