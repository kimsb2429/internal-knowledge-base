| **II.3****.C.1.h**  **Payment status** | Since BDN was a processing system as well as a payment system, it was able to stop payments in some circumstances to prevent certain erroneous payments.  eMPWR does not have this ability since it is only a payment system.  The Payment status section of the Beneficiary Information screen allows a beneficiary’s account to be placed into suspense, preventing the release of certain payments until the suspense is resolved.  Reasons for a file to be suspended include the return of payments as undeliverable, report of a date of death, or a potential administrative error. User selects the **Save** button to save the payment status information.  Users can also use the **Cancel** or **X** button to close the Payment status window without taking any actions.  When the situation requires a suspended file to be returned to activestatus, a warning message displays reading “Changing the payment status to active will release all suspended payment transactions.  Confirm you wish to proceed.”  To move forward the user must select the **Proceed with this action** checkbox and select **Save**.  *Note*: Returning the **Payment status** to *Active* from *Suspended* will release all matured payment and debt transactions held up by the suspense.  *Note:* An information message displays on all screens for a beneficiary with a suspended file indicating the file is in suspense and the corresponding reason.  *Working a suspended file*  Although users should resolve a suspense issue before finishing a claim in most cases, it is possible a file will stay in suspense for a longer period and processing will be required while the file is in suspense. BM allows users to create claims for files that are in suspense.  For details on authorizing a claim, see M22-4, Part II, 3.F. |

| **II.3****.C.2.a**  **Enrollment verification methods** | Beneficiaries have several ways to verify their enrollment.   1. **Text messaging**: The preferred method monthly verification is text messaging. Beneficiaries with a U.S. mobile phone number (including Puerto Rico) can use text messaging to verify enrollment. Beneficiaries who provide a valid mobile phone number will receive a text message inviting them to “opt-in” to text message verification. Those who reply *Yes* can then verify their monthly enrollment via text message. Those who reply *No* will be defaulted to email, if a valid email address is in BM. The link within that invitation text message will expire in 14 calendar days.   a.  **Opted-in to text**: Beneficiaries will receive the first text verification message on the last day of the month.   1. If the beneficiary selects *Yes*, their enrollment will be verified, payment released automatically by BM, and an ATTENDY document will be filed in the Image Management System (TIMS). Once verified a confirmation text is sent to the beneficiary. 2. If the beneficiary selects *No*, no payments will be released and an ATTENDN document will be filed in TIMS, creating an active token for user review. 3. If no response is received after three (3) days, a follow-up text message is sent and the link to verify the enrollment will remain open for three (3) additional calendar days.  After seven (7) calendar days, the text conversation is closed, and the system is updated with *No response*.  No payments will be released and an ATTENDR document will be filed in TIMS, creating an active token for user review.   b. **Opted-out of Text**:  Beneficiaries will be automatically enrolled in email verification and will receive an email message as outlined in the subsequent section.   1. **Email**: Beneficiaries can also verify their enrollment information each month through an automated email system. If a beneficiary opts-out of text messages, they will be automatically enrolled into the email system, if a valid email address is in the BM.    1. The TIMS document types for email verification are the same as the text message document types.    2. Beneficiaries will be sent an automated email on the last day of the month that needs to be responded to with either "*Yes, my enrollment is the same"* or "*No, my enrollment has changed"*    3. The beneficiary will be sent a reminder email after seven (7) calendar days with no response.  The conversion will close after 14 calendar days without a response.    4. If the beneficiary selects *Yes*, their enrollment will be verified, payment released automatically by BM, and an ATTENDY document will be filed in TIMS. The beneficiary will be sent to a VA.gov webpage displaying a successful verification message.    5. If the beneficiary selects *No*, no payments will be released and an ATTENDN document will be filed in TIMS, creating an active token for user review. The beneficiary will be sent to a VA.gov webpage asking them to contact their School Certifying Official (SCO) to adjust their enrollment if necessary.    6. After five months without enrollment verification, the beneficiary’s record will be considered delinquent and an ATTENDNR document will be filed in TIMS, creating an active token. All payments will be held until the beneficiary manually verifies their enrollment by calling the Education Call Center (ECC).    7. If the beneficiary clicks an expired link, they will be sent to a VA.gov webpage informing them that no action was taken because the link has expired and directs them to contact the ECC. 2. **Telephone:** The beneficiary may also call the ECC to verify their enrollment status.  ECC staff can verify the enrollment within BM by performing the following steps:    1. Verify the beneficiary’s identity using standard protocol.    2. If the beneficiary verifies that their enrollment is the same (i.e., “yes”), verification may occur.    3. If the beneficiary states that their enrollment has changed (i.e., “no”), additional action is needed.       1. Create a Report of Contact (119E) describing any changes to the enrollment. The 119E must be captured to TIMS.       2. The ECC agent should advise the beneficiary to notify their SCO of the enrollment status change if a 1999B – Enrollment amendment document has not been received.       3. Route claim, following local guidance, to the appropriate supplemental queue for processing. 3. **Verify Your Enrollment (VYE) System**: The beneficiary may also submit their enrollment verification through the VYE system, the replacement for the WAVE system.   *Note*: VYE is only available for chapter 30 and chapter 1606 beneficiaries until further notice.  Chapter 35 beneficiaries must verify their attendance through one of the other options.   4. **Other**:  An email from the claimant received via Ask VA (AVA) or through any VA e-mail system may be used to verify the enrollment status. |

| **II.3****.C.2.b**  **Impact on payments** | Enrollment verification affects EDU payments (i.e., basic and/or kicker payments) made to beneficiaries.  As a result of the enrollment verification requirements, BM creates a separate “35” lump sum business transaction (BT) for each month, where a basic and/or kicker payment is due.   1. Unlike the Post-9/11 GI Bill (chapter 33), these monthly basic and/or kicker payments are **NOT** released automatically by BM for the first two months. 2. The beneficiary must supply a positive (i.e., “Yes, my enrollment is the same”) monthly verification response for payment to be released. 3. If the beneficiary responds “No, my enrollment has changed” or fails to verify for five consecutive months, basic and/or kicker payments will not be released and VCE will need to take appropriate action. |

| **II.3****.C.2.c**  **Processing monthly verifications** | **Claims processing impact and responsibilities.**  Users should continue to process enrollment certifications (VA Form 22-1999) and notices of change in student status (VA Form 22-1999b) submitted using current guidance (see M22-4, Part II, 3.E for details).  If an “unchanged” amendment is received, it should be accepted from Enrollment Manager (EM) or manually entered into BM. The BM claim should be authorized with payments and letter(s) should be released.  BM has been updated to automatically accept text messages which state *Yes*. These affirmative text messages will NOT be sent to TIMS.  BM will automatically update the enrollment verification screen and release payment.  If the beneficiary responds *No* to the text message, BM will automatically hold all basic and kicker payments. An ATTENDN TIMS document will be created as an active claim token and routed to the appropriate supplemental queue for routine processing. When the user receives the ATTENDN document, the following actions will be performed.   1. Review the TIMS folder for any reported changes from the school. 2. Process any pending enrollment certifications or adjustments found, as usual.    1. If there is no evidence of a change submitted by the school in the TIMS folder, the user must contact the school to request any updated enrollment information and update TIMS with a 119E describing any changes or confirmation that the beneficiary remains certified correctly. 3. Process reported changes documented in TIMS. 4. Ensure the BM system-generated letter is not suppressed. 5.
