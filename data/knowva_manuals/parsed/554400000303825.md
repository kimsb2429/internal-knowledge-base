| **Topic 1: Searching for a beneficiary in BDN Historical Reporting** | |
|  | --- |
| **Introduction** | This topic discusses searching for beneficiaries using the BDN Historical Reporting functionality in Benefits Manager (BM). |
|  | --- |
| **Change Date** | June 27, 2025 |
|  | --- |
| **II.3.B.1.a**    **Searching for a beneficiary** | The BDN Historical Reporting screen is accessed using a tab at the top of the home screen in BM, which can be accessed via <https://iam.education.va.gov/>.      *Figure: BDN Historical Reporting Search Screen*    The user can begin the search by entering the following in the search field on the screen.   1. File number/ Social Security Number    1. When searching by File number (such as a “C number”), the user must enter the entire alphanumeric identifier as is in BDN. If the identifier begins a letter, as with VA file numbers, the leading letter **must be capitalized**.    2. When searching by social security number (SSN), the user enters nine (9) numeric digits without dashes.   *Note*: “Wild card” searches are not available in BM.  *Note*:A chapter 35 record may be located by searching either the Veteran’s file number or the beneficiary’s SSN  If the user wishes to conduct a separate record search, they can either:   1. Select the File number of SSN previously entered, delete, re-enter, and click **Search**, or 2. The user can click **Restart search**, which erases the prior field entry, and enter a new File number or SSN. |
|  | --- |
| **II.3.B.1.b**      **Search Results** | After selecting the **Search** button, a new section will appear.   1. Search results allow the user to access record of the beneficiary’s past utilized education benefits by selecting the **View Report** button.     *Figure: Search results example*   2. Multiple results may be presented. The beneficiary and/or their dependents may be displayed with separate and distinct payee codes. The user should select the record which matches their adjudicative need.   An error message will display if the search has more than 100 results requesting the user to refine the search further.  A message will be displayed if no results are returned.      *Figure: Error message when no results are returned*    An error message will appear if the user does not have the required sensitivity level to access the record.      *Figure: Error message when user lacks sensitivity level to access the report*    Upon clicking **Ok** the **View Report** button will disappear and be replaced with the message *Do not have permission to view.*    If system connections are down or intermittent, the following error message may be encountered.      *Figure: Example of error message when system is down* |

| **II.3.B.2.a**      **General** | When the identified beneficiary has been successfully located and the user clicks **View Report**, the following options may be presented, based on the beneficiary’s past benefit usage.     1. Chapter 35, 2. Chapter 30, 3. Chapter 1606, 4. Chapter 1607, 5. Chapter 32, or 6. Chapter 34.       *Figure: View of reports provided during claimant search*    To return to the search screen of BDN Historical Reporting, the user must click the **Back to search results** button at the top.      *Figure: Back to search results button example*   --- |
| **II.3.B.2.b**    **Claimant Details Banner** | Upon expanding any of the reports provided, a claimant details banner will be presented at the top (in blue), containing the following information:     1. Name & Address,    1. For a beneficiary with a domestic residence,       1. Displayed as First name, Middle initial, Last name, Street Address/PO Box, City, State    2. For a beneficiary with a foreign residence, and       1. Displayed as First name, Middle initial, Last name, Street Address/PO Box, City, Province, Country.    3. For a beneficiary with a military overseas residence.       1. Displayed as First name, Middle initial, Last name, Street Address/PO Box, Military PO, and Geographic area of station. 2. Zip code,    1. For a beneficiary with a domestic residence,       1. Displays the first five digits.    2. For a beneficiary with a foreign residence,       1. Displays up to five digits.    3. For a beneficiary with a military overseas residence,       1. Displays the first five digits.     *Note*: If an “FZ code” was used in BDN, only the succeeding 3 digits will be displayed, (i.e., “FZ953” in BDN will display as “953” in the report).     3. Date of birth,    1. Displayed in the format of MM/DD/YYYY. 4. Date of death,    1. If deceased, will be displayed in the format of MM/DD/YYYY, and 5. Gender,    1. Display will include either:       1. M for Male, or       2. F for Female. 6. Station,    1. Indicates the last RO/RPO to process the record, and    2. Displays as RPOXXX. 7. Branch of service,    1. Active-duty branches, and    2. Reserve/guard units.     Note: Field will NOT be utilized or displayed on Chapter 35 records.     8. File number,    1. If a non-SSN VA identifier is provided, this field will display all alphanumeric values, in capitalized form. For example, for a “C” type file, the user would see *C12345678* without any dashes or separation,    2. If the beneficiary is only identified by their SSN, their SSN will appear in this field, without dashes, and    3. If the beneficiary does not have an SSN, then a system-generated identifier would be produced. 9. SSN,    1. If an SSN exists for the beneficiary, it will display as nine (9) numeric digits with dashes, and    2. If no SSN exists for the beneficiary, the field will display a system-generated identifier. 10. Date last paid,     1. Represents the most recent “none” or no-pay date for the beneficiary, and     2. Displays in the format of MM/DD/YYYY. 11. Six-credit hour exclusion, and     1. If previously utilized, the field will display **USED**, and     2. If not previously utilized, the field will display **UNUSED**. 12. Master record status.     1. Displays the most recent Master record status from BDN as per Rescinded: M22-4, Part II, 9.28.     *Note*: If a field does not include a value to present user from BDN, the field will show **N/A**.        *Figure: Example of a claimant details banner* |
