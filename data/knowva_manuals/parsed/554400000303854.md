| Topic 1: Cost of Living Adjustment (COLA) | |
|  | --- |
| **Introduction** | This topic provides information about the Cost of Living Adjustment (COLA) functionality in BM. |
|  | --- |

|  |  |
| --- | --- |
| **Change Date** | June 27, 2025 |
|  | --- |

|  |  |
| --- | --- |
| **II.3.K.1.a**  **General** | Benefits Manager contains functionality to automatically adjust previously adjudicated rates for Cost of Living Adjustments (COLA). This adjustment takes place using an automated batch process.  The effective date of a COLA adjustment will generally be October 1. |
|  | --- |

|  |  |
| --- | --- |
| **II.3.K.1.b**  **Identification criteria** | Beneficiaries with awards which span the effective date of the COLA will be identified for adjustment by a system query. |
|  | --- |

|  |  |
| --- | --- |
| **II.3.K.1.c**  **Adjudication of increase** | After beneficiaries have been identified by the system query detailed in [Identification criteria](#3.K.1.b), claims will be created by system automation which will re-adjudicate the claim to update award rates on or after the effective date of the COLA. The system will attempt to authorize these claims through automation.  The created claim will have a claim type of “Legislative Rate Adjustment”. |
|  | --- |
| **II.3.K.1.d**  **Claim workflow** | The Manual Transaction Reconciliation follows the claim workflow detailed in M22-4, Part II, 3.C. |
|  | --- |

|  |  |
| --- | --- |
| **II.3.K.1.e**  **Creation and authorization issues** | There are circumstances which may prevent creation of the COLA claim or prevent successful authorization of the COLA claim. Circumstances which may prevent creation of a claim include, but are not limited to:   1. An existing claim is in-progress at time of attempted creation of COLA claim 2. The most recently authorized claim is a Claim-0 3. The most recently authorized claim is a Claim-R 4. No original claim exists for the benefit type     Circumstances which may prevent successful authorization include, but are not limited to:   5. Erroneous or no net awards generated in the COLA claim 6. Unavailability of integrated partner systems (eMPWR, MPI, VA Profile, etc.) |
|  | --- |

|  |  |
| --- | --- |
| **II.3.K.1.f**  **Resolution of authorization issues** | If one of the issues detailed in [Creation and authorization issues](#II.3.K.1.e) is encountered during the COLA claim workflow, an actionable drop scan document will be created by Benefits Manager and placed in the beneficiary’s TIMS folder. This document will contain the beneficiary’s name, file number, station of jurisdiction, facility code of affected award, and failure reason.  To resolve the issues, use the following workflow:   * Failure reason is existing claim in-progress   + Continue processing of in-progress claim. Updated rates should automatically propagate. * Failure reason is Claim-0   + Process a Claim-R and supplemental claim. Updated rates should automatically propagate. * Failure reason is Claim-R   + Process a supplemental claim. Updates rates should automatically propagate. * Failure reason is no original claim for benefit type   + Review for other benefit usage and create a Claim-R and/or supplemental claim as necessary. Updates rates should automatically propagate. If no other benefit usage exists, record was identified in error. * Failure reason is erroneous or no net award generated   + Record may have been identified in error. If an erroneous net award, or no net award is generated and should be generated, a Detailed Issue File (DIF) may be required. * Failure reason is unavailability of integrated partner system   + If there is a known outage of a partner system, retry authorization after notification of restoration of service. If the outage is not known, follow local procedures for reporting service outages. After restoration of service, retry authorization. |
|  | --- |

|  |  |
| --- | --- |
| **II.3.K.1.g**  **Letters generated by COLA claim** | A COLA claim will generate a payment letter with an alternate introduction paragraph to explain the purpose of the notification. Outside of the altered introduction, the content of the generated letter will be identical to that of a normal payment letter. Updated rates will be shown for applicable awards in the “Payments Made to You” and “What You Are Due” tables. |
